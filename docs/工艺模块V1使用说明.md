# 工艺模块 V1 使用说明（工段 + 小工序 + 产品多模板）

## 1. 模块范围

工艺模块 V1 为独立侧边栏 `craft`，仅包含 2 个页面：

1. `process_management`（工序管理）
2. `production_process_config`（生产工序配置数据）

本版本聚焦生产流程建模，不包含消息中心、缺陷/报废/返修扩展。

## 2. 角色可见性

默认可见角色：

1. `system_admin`
2. `production_admin`

默认不可见角色：

1. `quality_admin`
2. `operator`

## 3. 核心模型说明

1. 工段（大工序）：`mes_process_stage`
2. 小工序：`mes_process`（必须归属一个工段）
3. 产品工序模板：`mes_product_process_template`
4. 模板步骤：`mes_product_process_template_step`

订单中保存的是“工段 + 小工序快照”，后续改名不影响历史订单展示。

## 4. 工序管理页（工段/小工序）

可执行操作：

1. 新增/编辑/启停/删除工段
2. 新增/编辑/启停/删除小工序

规则：

1. 先建工段，再建小工序
2. 小工序必须归属工段
3. 被模板或未完成订单引用的数据不可删除
4. 推荐优先“停用”，避免历史引用中断

## 5. 生产工序配置页（模板）

可执行操作：

1. 按产品创建多套工序模板
2. 维护模板步骤顺序（工段 -> 小工序）
3. 设置默认模板
4. 更新模板并可选同步未完成订单

模板同步规则：

1. `pending` 订单：整条流程重建
2. `in_progress` 订单：仅重建当前工序之后
3. 同步冲突返回 `409`，可查看 `total/synced/skipped/reasons`

## 6. 订单管理联动

在生产订单管理页，新建/编辑订单支持：

1. 先选产品，再选工序模板
2. 在模板基础上调整步骤
3. 可“另存为新模板”，并可设为默认模板

兼容输入：

1. 后端仍兼容 `process_codes` 一个版本周期
2. 新逻辑优先 `template_id + process_steps`

## 7. 用户与审批联动

1. 操作员工序分配改为“小工序多选”
2. 用户管理与注册审批页面均按工段分组展示小工序
3. 操作员必须至少分配 1 个小工序

## 8. 保养模块联动

1. 保养计划“执行工序”改为工段口径（`execution_process_code` 表示工段 code）
2. 执行权限按“用户绑定小工序所属工段集合”判定
3. 前端不再依赖固定 4 个工序常量

## 9. 数据迁移说明

本次工艺升级采用“清空后重建”策略：

1. 生产域及工艺域历史数据不保留
2. 产品、设备、保养主数据保留
3. 需由管理员重新配置工段、小工序、产品模板
