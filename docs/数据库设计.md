# MES 数据库设计（PostgreSQL）

## 1. 设计原则

- 主键统一使用 `bigserial` 自增整数。
- 全表保留审计字段：`created_at`、`updated_at`（通过 `TimestampMixin` 实现）。
- 关键业务表保留 `status` 字段实现状态流转控制。
- 查询高频字段必须建立索引。
- 外键约束使用 `ondelete` 策略（CASCADE、RESTRICT、SET NULL）。

## 2. 核心表清单

### 2.1 用户与权限

- `sys_user`：用户表
- `sys_role`：角色表
- `sys_user_roles`：用户角色关联表
- `mes_process`：工序表
- `sys_user_processes`：用户工序关联表

### 2.2 产品域

- `mes_product`：产品主数据
- `mes_product_parameter`：产品参数表
- `mes_product_parameter_history`：产品参数变更历史

### 2.3 设备域

- `mes_equipment`：设备台账表

### 2.4 保养域

- `mes_maintenance_item`：保养项目表
- `mes_maintenance_plan`：保养计划表
- `mes_maintenance_work_order`：保养工单表
- `mes_maintenance_record`：保养记录表

### 2.5 页面可见性

- `mes_page_visibility`：页面可见性配置表

### 2.6 注册审批（规划中）

- `mes_registration_request`：用户注册申请表

## 4. 关系与约束

- `mes_product_parameter.product_id -> mes_product.id`（外键，ondelete="CASCADE"）。
- `mes_product_parameter_history.product_id -> mes_product.id`（外键，ondelete="CASCADE"）。
- `mes_maintenance_plan.equipment_id -> mes_equipment.id`（外键，ondelete="RESTRICT"）。
- `mes_maintenance_plan.item_id -> mes_maintenance_item.id`（外键，ondelete="RESTRICT"）。
- `mes_maintenance_plan.default_executor_user_id -> sys_user.id`（外键，ondelete="SET NULL"）。
- `mes_maintenance_work_order.plan_id -> mes_maintenance_plan.id`（外键）。
- `mes_maintenance_work_order.equipment_id -> mes_equipment.id`（外键）。
- `mes_maintenance_work_order.assigned_user_id -> sys_user.id`（外键）。
- `mes_maintenance_record.work_order_id -> mes_maintenance_work_order.id`（外键）。

### 唯一约束

- `sys_user.username`：用户名唯一。
- `sys_role.code`：角色编码唯一。
- `mes_product.name`：产品名称唯一。
- `mes_equipment.code`：设备编码唯一。
- `mes_product_parameter(product_id, param_key)`：产品参数键唯一。
- `mes_maintenance_plan(equipment_id, item_id)`：设备保养项目组合唯一。

### 检查约束

- `mes_product_parameter.param_type IN ('Text', 'Link')`：参数类型限制。
- `mes_maintenance_plan.cycle_days > 0`：周期天数必须大于 0。
- `mes_maintenance_plan.estimated_duration_minutes IS NULL OR > 0`：预计耗时非负。
- `mes_maintenance_plan.execution_process_code IN ('laser_marking', 'product_testing', 'product_assembly', 'product_packaging')`：工序编码限制。

### 索引建议

- 用户查询：`sys_user(username)`、`sys_user_roles(user_id, role_id)`。
- 产品查询：`mes_product(name)`、`mes_product_parameter(product_id)`。
- 设备查询：`mes_equipment(code)`、`mes_equipment(name)`。
- 保养计划查询：`mes_maintenance_plan(equipment_id)`、`mes_maintenance_plan(next_due_date)`。
- 保养工单查询：`mes_maintenance_work_order(status)`、`mes_maintenance_work_order(assigned_user_id)`。

## 5. 数据迁移管理

- 使用 Alembic 管理结构迁移。
- 每次结构调整必须附带回滚脚本。
- 生产变更遵循"先兼容后切换再清理"原则，避免停机。
- 启动时自动执行 `alembic upgrade head`。
