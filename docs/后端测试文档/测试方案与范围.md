# 后端测试方案与范围

## 1. 目标
- 覆盖后端核心业务、接口封装、权限依赖、数据模型初始化路径。
- 所有测试统一放在 `test/backend` 目录。
- 覆盖率统计口径为 `backend/app` 全量代码，目标 >= 80%。

## 2. 测试范围
- 核心层：`app/core/*`
- 服务层：`app/services/*`
- API 层：`app/api/*` 与 `app/api/v1/endpoints/*`
- 启动与依赖：`app/main.py`、`app/bootstrap/*`、`app/db/*`
- Schema 与 Model：`app/schemas/*`、`app/models/*`

## 3. 不在本次范围
- 前端代码（`mes_client/*`）
- Alembic 历史迁移脚本（`backend/alembic/*`）
- 真实 PostgreSQL 启动引导链路（测试环境使用 SQLite 内存/临时文件）

## 4. 测试策略
- 使用 `pytest` + `pytest-cov`。
- 使用统一 `conftest.py` 提供：
  - `db_engine`（SQLite）
  - `db`（Session）
  - `factory`（标准化造数，覆盖用户/角色/工段/工序/产品/模板/订单/保养对象）
- 采用“服务集成测试 + 接口层直调测试”组合：
  - 服务层直接调用，验证业务规则、状态机与异常分支。
  - 接口层直调 endpoint 函数，验证封装、参数透传和异常映射。

## 5. 目录结构
- `test/backend/conftest.py`：测试基座与数据工厂
- `test/backend/test_core_and_small_services.py`
- `test/backend/test_role_process_page_visibility_services.py`
- `test/backend/test_user_service.py`
- `test/backend/test_product_and_craft_services.py`
- `test/backend/test_production_order_and_execution_services.py`
- `test/backend/test_quality_and_statistics_services.py`
- `test/backend/test_equipment_service.py`
- `test/backend/test_api_and_bootstrap_smoke.py`
- `test/backend/test_endpoints_basic.py`

## 6. 质量门禁
- 所有用例必须通过。
- 覆盖率按 `backend/app` 全量统计，需 >= 80%。
