# 后端测试用例清单

## 1. 核心与基础服务
- security: 密码哈希与 token 编解码。
- equipment_process/page_catalog: 枚举映射与默认可见性。
- process_code_rule: 编码规则、工段匹配、唯一性校验。
- online_status_service: 在线状态写入、快照、过期清理。
- maintenance_scheduler_service: 时间解析、时区回退、下一次触发计算。

## 2. 角色/工序/页面可见性
- role_service: 创建、查询、分页、更新、缺失码返回。
- process_service: 创建、更新、分页、非法编码拦截。
- page_visibility_service: 默认配置初始化、更新、非法输入拦截。
- production_event_log_service: 事件写入和查询顺序。

## 3. 用户域
- create_user: 角色合法性、operator 工序约束、成功路径。
- update_user: 重名冲突、角色与工序协同约束。
- registration: 提交、审批、驳回闭环。
- admin 保障：`ensure_admin_account`、`normalize_users_to_single_role`。

## 4. 产品与工艺模板
- craft_service: 工段/工序 CRUD、引用保护。
- 系统母版：创建、更新、解析（有效/无效分支）。
- 产品模板：创建、更新、默认模板逻辑。
- product_service: 新建产品、参数模板初始化、参数更新、历史记录。

## 5. 生产域
- production_order_service: 建单、改单、删单、手工完工。
- production_execution_service: 首件提交、报工、状态推进、错误分支。
- list_my_orders: 管理员/操作员视角。

## 6. 质量与统计
- quality_service: 首件列表、概览、工序统计、人员统计。
- production_statistics_service: 订单总览、工序维度、人员维度统计。

## 7. 设备与保养域
- equipment_service: 设备/保养项 CRUD、启停、删除保护。
- maintenance plan: 创建、更新、生成工单、删除限制。
- work order: 列表过滤、权限、开工、完工、记录落表。

## 8. API 与启动链路
- deps: 当前用户解析、角色校验。
- main/api router: 路由装载与基础健康检查。
- bootstrap_seed_service: 初始化角色与管理员。
- endpoint 直调：auth/users/roles/processes/ui/products/craft/production/quality/equipment 关键路径。

## 9. 文件位置
- 全部后端测试文件位于：`test/backend`
- 测试文档位于：`docs/测试文档`
