# 设备模块深度检查

## 1. 总体判断

- 结论：设备保养主闭环已初步打通，但详情页、规则页、附件、导出体验和若干确定性缺陷仍使模块无法按需求完整验收。
- 核心风险：存在前端请求体语法问题、规则启停接口字段不一致、追溯展示未优先使用快照字段。

## 2. 已满足项

- 设备台账已支持列表、关键字搜索、状态筛选、新增、编辑、启停、删除、详情查看、导出。证据：`frontend/lib/pages/equipment_ledger_page.dart:119`、`backend/app/api/v1/endpoints/equipment.py:215`
- 已实现设备编号唯一校验、删除前计划引用校验、未完成工单校验。证据：`backend/app/services/equipment_service.py:184`、`backend/app/services/equipment_service.py:253`
- 保养项目已支持列表、搜索、状态筛选、新增、编辑、启停、删除、导出，并校验项目名称唯一与周期大于 0。证据：`frontend/lib/pages/maintenance_item_page.dart:122`、`backend/app/services/equipment_service.py:379`
- 保养计划已支持设备/项目/状态筛选、新增、编辑、启停、删除、手工生成工单、导出，也已支持默认执行人和预计时长录入。证据：`frontend/lib/pages/maintenance_plan_page.dart:154`、`backend/app/services/equipment_service.py:503`
- 自动生成工单链路较完整，已支持到期扫描、防重、更新下次到期日和写日志。证据：`backend/app/services/equipment_service.py:769`、`backend/app/services/maintenance_scheduler_service.py:58`
- 保养执行已支持状态筛选、仅看我的任务、到期日期范围、工段筛选、开始执行、完成执行、取消、详情查看、导出。证据：`frontend/lib/pages/maintenance_execution_page.dart:165`、`backend/app/services/equipment_service.py:843`
- 保养记录已支持列表、时间范围、结果筛选、设备筛选、详情、导出，且后端已保留来源计划/设备/项目快照字段。证据：`frontend/lib/pages/maintenance_record_page.dart:157`、`backend/app/models/maintenance_record.py:15`

## 3. 部分满足项

- 设备详情页只是弹窗式轻量详情，只展示基础信息、启用计划数、待执行工单数、最近记录，缺少关联计划列表、未完成工单列表、风险提示和快速跳转。证据：`frontend/lib/pages/equipment_ledger_page.dart:306`、`backend/app/schemas/equipment.py:175`
- 保养执行详情也只是简化版查看，缺少来源设备快照、来源计划完整快照、附件点击查看/下载与记录互跳。证据：`frontend/lib/pages/maintenance_execution_page.dart:372`、`backend/app/schemas/equipment.py:191`
- 保养记录页虽支持关键筛选，但“执行人筛选”未做成独立条件，前端只是把执行人文本并入关键字。证据：`frontend/lib/pages/maintenance_record_page.dart:115`、`frontend/lib/services/equipment_service.dart:579`
- 保养项目页已支持分类字段，但缺少“项目分类筛选”。证据：`frontend/lib/pages/maintenance_item_page.dart:505`、`backend/app/api/v1/endpoints/equipment.py:357`
- 保养计划页缺少执行工段筛选、默认执行人筛选，列表也未完整展示预计时长、创建时间、更新时间。证据：`frontend/lib/pages/maintenance_plan_page.dart:626`、`backend/app/api/v1/endpoints/equipment.py:475`

## 4. 未满足项

- 设备台账的“位置筛选、负责人筛选”未实现。证据：`frontend/lib/pages/equipment_ledger_page.dart:577`、`backend/app/api/v1/endpoints/equipment.py:215`
- 设备规则 / 运行参数页目前只算基础雏形，未完整实现规则编码、适用设备/类型、参数默认值、启停状态、绑定范围等核心能力。证据：`backend/app/models/equipment_rule.py:10`、`backend/app/models/equipment_runtime_parameter.py:10`、`frontend/lib/pages/equipment_rule_parameter_page.dart:143`
- 保养记录列表未展示附件列。证据：`frontend/lib/pages/maintenance_record_page.dart:463`
- 前端虽然提供导出入口，但更多是把 CSV 文本弹窗展示，导出体验与需求中的正式导出仍有差距。证据：`frontend/lib/pages/equipment_ledger_page.dart:498`

## 5. 明显偏差

- `frontend/lib/services/equipment_service.dart` 中保养计划请求体存在可疑语法写法，属于确定性缺陷风险。证据：`frontend/lib/services/equipment_service.dart:388`、`frontend/lib/services/equipment_service.dart:423`
- 设备规则启停接口前后端字段名不一致：前端传 `is_enabled`，后端 Schema 定义 `enabled`，接口实现又读取不存在字段。证据：`frontend/lib/services/equipment_service.dart:731`、`backend/app/schemas/equipment.py:84`、`backend/app/api/v1/endpoints/equipment.py:1052`
- 工单详情/列表优先使用当前关联对象名，而不是快照字段，不符合追溯优先快照的原则。证据：`backend/app/api/v1/endpoints/equipment.py:158`
- 设备台账表单与需求的必填规则不一致：需求要求位置必填、负责人可选，当前实现更接近位置可空、负责人必填。证据：`frontend/lib/pages/equipment_ledger_page.dart:188`、`frontend/lib/pages/equipment_ledger_page.dart:196`
- 执行导出后端逻辑存在“非全量角色时可能直接导出空集”的风险。证据：`backend/app/services/equipment_service.py:1310`、`backend/app/services/equipment_service.py:868`

## 6. 优先级建议

1. 先修复前端请求体与启停接口字段不一致等确定性缺陷。
2. 再补设备详情、执行详情、规则/参数页的完整能力与快照追溯。
