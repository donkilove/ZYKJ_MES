# 功能规划 V1 深度审查总览

## 1. 审查目标

- 对照 `docs/功能规划V1` 下 7 个模块需求说明，静态核查当前前端与后端实现的符合度。
- 本次审查仅输出检查结论，不直接改动业务代码。
- 结果按模块拆分存放，便于后续逐项整改与复核。

## 2. 审查范围

- 需求基线：`docs/功能规划V1/用户模块/用户模块需求说明.md`、`docs/功能规划V1/产品模块/产品模块需求说明.md`、`docs/功能规划V1/工艺模块/工艺模块需求说明.md`、`docs/功能规划V1/设备模块/设备模块需求说明.md`、`docs/功能规划V1/品质模块/品质模块需求说明.md`、`docs/功能规划V1/生产模块/生产模块需求说明.md`、`docs/功能规划V1/消息模块/消息模块需求说明.md`
- 后端范围：`backend/app/api/v1/endpoints/`、`backend/app/services/`、`backend/app/schemas/`、`backend/app/models/`
- 前端范围：`frontend/lib/pages/`、`frontend/lib/services/`、`frontend/lib/models/`

## 3. 审查方法

- 采用“需求条目 -> 页面/接口 -> 服务/模型 -> 关键业务规则”的静态对照方式。
- 核查重点覆盖页面清单、字段、状态流转、权限、导出、追溯、跨模块依赖、实时能力。
- 本次未执行联调、构建与自动化测试，结论以仓库现状代码为准。

## 4. 总体判断

| 模块 | 总体判断 | 结论摘要 |
| --- | --- | --- |
| 用户模块 | 主体具备，关键规则未闭环 | 7 个页面和主流程基本齐全，但自定义角色不可分配、权限配置缺少系统管理员保底规则。 |
| 产品模块 | 核心模型偏离需求 | 页面和接口较多已存在，但“参数绑定版本、已生效只读、按生效版本查询”未真正成立。 |
| 工艺模块 | 主体具备，存在高风险缺口 | 模板、发布、回滚、引用分析和看板已较完整，但轻量查询、删除保护、已发布模板约束仍不足。 |
| 设备模块 | 闭环初步打通，细节与稳定性不足 | 台账、项目、计划、执行、记录已串联，但详情页、规则页、附件、导出体验和若干确定性缺陷未闭合。 |
| 品质模块 | P0 能力较多已落地 | 首件、品质统计、报废、维修主流程已具备，但趋势分析、处置留痕、统计口径一致性不足。 |
| 生产模块 | 主链路可用但未达标 | 建单、执行、并行、代班、报废、维修链路基本存在，但数量口径、删除追溯、多回流分配仍有关键问题。 |
| 消息模块 | 基础版具备，验收目标未达成 | 列表、未读、角标、WS 基础链路已存在，但业务跳转、实时同步、公告与规则配置明显不足。 |

## 5. 跨模块高优先级问题

### 5.1 P0 级问题

1. 产品模块的数据模型仍以“当前参数表”为核心，未实现“参数归属具体版本”，导致版本、生效、查询、导出、历史追溯均与需求存在系统性偏差。详见 `docs/功能规划V1深度审查-20260318/11-产品模块.md`。
2. 用户模块允许创建自定义角色，但用户新建、编辑、注册审批环节无法实际分配这些角色，形成业务闭环断裂。详见 `docs/功能规划V1深度审查-20260318/10-用户模块.md`。
3. 生产模块报工时未把异常数量一并纳入可生产数量校验，数量口径不闭环。详见 `docs/功能规划V1深度审查-20260318/15-生产模块.md`。
4. 品质模块导出报废统计会直接把“待处理”更新为“已申请”，实现语义与需求严重不符。详见 `docs/功能规划V1深度审查-20260318/14-品质模块.md`。
5. 消息模块的跳转编码与前端导航机制不匹配，部分消息无法正确落到目标页签或业务上下文。详见 `docs/功能规划V1深度审查-20260318/16-消息模块.md`。

### 5.2 P1 级问题

1. 工艺模块已发布模板仍可直接编辑当前实体，删除保护未完整覆盖历史版本引用。详见 `docs/功能规划V1深度审查-20260318/12-工艺模块.md`。
2. 设备模块存在前端请求体语法问题、规则启停接口字段不一致等确定性缺陷，且详情页、规则页与附件能力明显缩水。详见 `docs/功能规划V1深度审查-20260318/13-设备模块.md`。
3. 用户权限配置缺少“至少保留一个可进入权限配置页的系统管理员账号”的保底控制。详见 `docs/功能规划V1深度审查-20260318/10-用户模块.md`。
4. 生产模块删除订单虽写事件日志，但日志外键使用级联删除，最终无法满足删除追溯要求。详见 `docs/功能规划V1深度审查-20260318/15-生产模块.md`。
5. 品质趋势与产品统计中的报废/维修聚合未严格应用相同筛选条件，统计口径不一致。详见 `docs/功能规划V1深度审查-20260318/14-品质模块.md`。

## 7. 结论

- 当前仓库并非“普遍缺页”状态，绝大多数模块都已有前后端页面与接口骨架。
- 主要问题集中在：关键业务规则未闭环、统计口径不一致、导出与筛选不一致、追溯链断裂、实时/跳转承接不足。
- 若按需求说明进行正式验收，产品、用户、生产、品质、消息 5 个模块仍存在足以阻断通过的关键问题；工艺、设备模块也需要补齐若干高风险缺口后再复核。
