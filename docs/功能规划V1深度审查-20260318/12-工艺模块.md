# 工艺模块深度检查

## 1. 总体判断

- 结论：主流程覆盖度较高，但删除保护、历史引用、轻量接口和已发布模板约束仍存在明显缺口。
- 核心风险：已发布模板仍可直接编辑当前实体；历史版本引用未完整纳入删除校验与引用分析。

## 2. 已满足项

- 工段/工序管理已支持列表查询、搜索、状态筛选、按工段筛工序、新建、编辑、启停、删除、导出。证据：`backend/app/api/v1/endpoints/craft.py:385`、`frontend/lib/pages/process_management_page.dart:1080`
- 删除前可查看引用分析，后端也已对用户、模板、系统母版、订单等现存引用做保护。证据：`backend/app/services/craft_service.py:281`、`frontend/lib/pages/process_management_page.dart:1013`
- 工艺模板配置能力较完整，已支持系统母版、产品模板、新建、编辑、复制、跨产品复制、从系统母版套版、发布、版本记录、版本对比、回滚、归档/取消归档。证据：`backend/app/api/v1/endpoints/craft.py:569`、`frontend/lib/pages/process_configuration_page.dart:1641`
- 发布链路已有版本快照、回滚链路也有 `source_revision_id` 追溯。证据：`backend/app/models/product_process_template_revision.py:23`、`backend/app/services/craft_service.py:1307`
- 引用分析页面已覆盖工段、工序、模板、产品四类对象，并展示引用类型、风险等级、风险说明。证据：`backend/app/services/craft_service.py:2214`、`frontend/lib/pages/craft_reference_analysis_page.dart:593`
- 工艺看板已支持产品、时间范围、工段、工序筛选，以及工时、产能和导出。证据：`backend/app/services/craft_service.py:1560`、`frontend/lib/pages/craft_kanban_page.dart:255`
- 模板 JSON 导入与批量导出已存在。证据：`backend/app/api/v1/endpoints/craft.py:728`、`frontend/lib/pages/process_configuration_page.dart:2276`

## 3. 部分满足项

- 工序默认排序只近似满足需求，后端缺少独立“工序顺序”字段，排序主要依赖 `id` 和前端 `code`。证据：`backend/app/services/craft_service.py:320`、`frontend/lib/pages/process_management_page.dart:189`
- 系统母版区具备版本、编辑、历史能力，但未直接在主区域展示完整步骤列表。证据：`frontend/lib/pages/process_configuration_page.dart:2747`
- 发布前有影响分析，回滚前则只有确认表单，没有同等强度的影响分析结果展示。证据：`frontend/lib/pages/process_configuration_page.dart:1732`、`frontend/lib/pages/process_configuration_page.dart:1886`
- 模板导出主要以 JSON 预览/复制形式存在，缺少更完整的单模板或版本级导出体验。证据：`frontend/lib/pages/process_configuration_page.dart:2276`
- 引用分析页展示了 `jump_target`，但前端并未真正完成业务跳转。证据：`frontend/lib/pages/craft_reference_analysis_page.dart:421`

## 4. 未满足项

- 未发现对外开放的工段/工序轻量查询接口，难以满足用户、设备、生产等模块统一复用的需求。证据：`backend/app/api/v1/endpoints/craft.py`、`backend/app/services/craft_service.py:2111`
- 历史版本引用未完整纳入工段/工序删除保护，也未完整进入引用分析。证据：`backend/app/services/craft_service.py:413`、`backend/app/models/product_process_template_revision_step.py:23`、`backend/app/models/craft_system_master_template_revision_step.py:23`
- 模板复制/套版未记录来源模板或来源母版信息。证据：`backend/app/models/product_process_template.py:18`、`backend/app/services/craft_service.py:1909`
- 需求建议的步骤扩展字段 `标准工时`、`关键工序标识`、`步骤说明` 尚未落地。证据：`backend/app/models/product_process_template_step.py:17`、`frontend/lib/models/craft_models.dart:94`
- 未见单模板导出与模板版本导出能力。证据：`backend/app/api/v1/endpoints/craft.py:728`、`backend/app/api/v1/endpoints/craft.py:928`

## 5. 明显偏差

- 已发布模板仍可直接编辑当前模板行，需求要求则是“已发布模板不允许直接覆盖编辑”。证据：`backend/app/services/craft_service.py:1152`、`frontend/lib/pages/process_configuration_page.dart:3061`
- 模板删除只重点阻止未完工订单引用，对已完成订单和历史业务引用保护不足。证据：`backend/app/services/craft_service.py:1892`、`backend/app/models/production_order.py:22`
- 模板列表默认排序与需求不完全一致，后端不是纯更新时间倒序。证据：`backend/app/services/craft_service.py:583`
- 当前 `frontend/lib/services/craft_service.dart` 存在明显的可疑语法写法，属于确定性风险点。证据：`frontend/lib/services/craft_service.dart:100`、`frontend/lib/services/craft_service.dart:210`

## 6. 优先级建议

1. 先补齐“已发布模板不可直接编辑”“历史版本引用纳入删除保护/影响分析”两项规则。
2. 再补工段/工序轻量接口、来源记录、模板版本导出和步骤扩展字段。
