# MES 环境搭建与启动指南

## 1. 开发环境要求

- Python 3.11+
- Flutter 3.24+（含 Dart SDK）
- PostgreSQL 14+
- Git 2.40+

## 2. 后端环境搭建（FastAPI）

### 2.1 项目结构

```
ZYKJ_MES/
├── backend/                    # 后端代码目录
│   ├── app/
│   │   ├── api/              # API 路由层
│   │   ├── core/             # 核心配置与安全
│   │   ├── db/               # 数据库连接
│   │   ├── models/           # ORM 模型
│   │   ├── schemas/          # Pydantic 数据模型
│   │   ├── services/         # 业务逻辑层
│   │   └── main.py           # 应用入口
│   ├── alembic/              # 数据库迁移
│   ├── scripts/              # 脚本目录
│   ├── requirements.txt      # 依赖包
│   ├── .env                  # 环境变量配置
│   └── .env.example          # 环境变量示例
├── mes_client/               # 前端 Flutter 项目
└── docs/                     # 文档目录
```

### 2.2 环境准备

```powershell
# 1) 克隆项目
git clone <repository-url>
cd ZYKJ_MES

# 2) 创建虚拟环境
cd backend
python -m venv .venv

# 3) 激活虚拟环境（PowerShell）
.\.venv\Scripts\Activate.ps1

# 4) 安装依赖
pip install -r requirements.txt
```

### 2.3 PostgreSQL 初始化

```sql
CREATE DATABASE mes_db;
CREATE USER mes_user WITH PASSWORD 'mes_password';
GRANT ALL PRIVILEGES ON DATABASE mes_db TO mes_user;
```

### 2.4 后端配置

创建 `.env` 文件：

```env
APP_NAME=ZYKJ MES API
APP_ENV=dev
APP_HOST=0.0.0.0
APP_PORT=8000
API_V1_PREFIX=/api/v1

DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=mes_db
DB_USER=mes_user
DB_PASSWORD=mes_password

BOOTSTRAP_ON_STARTUP=true
DB_BOOTSTRAP_HOST=127.0.0.1
DB_BOOTSTRAP_PORT=5432
DB_BOOTSTRAP_USER=postgres
DB_BOOTSTRAP_PASSWORD=your_postgres_password
DB_BOOTSTRAP_MAINTENANCE_DB=postgres

BOOTSTRAP_ADMIN_USERNAME=admin
BOOTSTRAP_ADMIN_PASSWORD=Admin@123456
ONLINE_STATUS_TTL_SECONDS=90
MAINTENANCE_AUTO_GENERATE_ENABLED=true
MAINTENANCE_AUTO_GENERATE_TIME=00:05
MAINTENANCE_AUTO_GENERATE_TIMEZONE=Asia/Shanghai

JWT_SECRET_KEY=your_strong_secret_here
JWT_ALGORITHM=HS256
JWT_EXPIRE_MINUTES=120
```

### 2.5 数据迁移与启动

```powershell
# 启动服务（自动执行数据库初始化和迁移）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 或使用启动脚本
cd ..
python start_backend.py
```

### 2.6 接口文档地址

- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`

## 3. 前端环境搭建（Flutter）

### 3.1 环境准备

```powershell
# 检查 Flutter 环境
flutter doctor

# 进入前端项目目录
cd mes_client

# 安装依赖
flutter pub get
```

### 3.2 配置 API 地址

在 Flutter 项目的配置文件中设置后端 API 地址，例如：

```dart
const String kApiBaseUrl = 'http://127.0.0.1:8000/api/v1';
```

### 3.3 启动前端

```powershell
# 启动 Web 应用
flutter run -d chrome

# 启动 Windows 桌面应用
flutter run -d windows

# 构建应用
flutter build web
flutter build windows
```

## 4. 项目启动流程

### 4.1 完整启动步骤

1. **启动数据库**：确保 PostgreSQL 服务运行
2. **启动后端**：
   ```powershell
   cd backend
   .\.venv\Scripts\Activate.ps1
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```
3. **启动前端**：
   ```powershell
   cd mes_client
   flutter run -d chrome
   ```

### 4.2 自动化启动脚本

项目根目录提供启动脚本：

```powershell
# 启动后端
python start_backend.py

# 启动前端
python start_frontend.py
```

## 5. 默认账号

## 6. 常见问题

- **连接数据库失败**：检查 PostgreSQL 监听地址、端口和账号权限。
- **Flutter 运行失败**：先执行 `flutter doctor` 并按提示修复。
- **跨域问题**：后端已启用 CORS 中间件，允许所有来源。
- **首次启动**：系统会自动初始化角色、工序和管理员账号。
- **权限错误**：确保操作员用户分配了至少一个工序。
