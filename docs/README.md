# MES 项目文档总览

本目录用于存放项目初创与迭代阶段的核心文档，目标是统一团队认知、明确边界、降低沟通成本。

## 文档清单

1. [项目规划](./项目规划.md)
2. [开发计划](./开发计划.md)
3. [技术方案](./技术方案.md)
4. [环境搭建](./环境搭建.md)
5. [数据库设计](./数据库设计.md)
6. [API规划](./API规划.md)
7. [项目管理规范](./项目管理规范.md)
8. [执行进度](./执行进度.md)

## 建议阅读顺序

1. 先读《项目规划》，确认目标与范围。
2. 再读《技术方案》《数据库设计》《API规划》，统一实现口径。
3. 开发前阅读《环境搭建》《项目管理规范》。
4. 每周按《执行进度》与《开发计划》同步实际状态。

## 文档维护约定

- 功能边界变化时，优先更新《项目规划》。
- 数据结构变化时，同步更新《数据库设计》和《API规划》。
- 每个迭代结束后，更新《执行进度》和《开发计划》。
- 所有文档修改走 PR 流程，避免口头约定与文档脱节。
