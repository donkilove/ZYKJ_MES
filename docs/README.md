## 建议阅读顺序
1. 项目与方案：`项目规划 -> 技术方案 -> 数据库设计 -> API规划`
2. 研发落地：`环境搭建 -> 开发计划 -> 执行进度`
3. 域专题：按业务进入对应目录（生产流转、生产迁移蓝图等）
4. 质量保障：阅读前后端测试文档与开发规范

## 维护规则
- 新文档优先放专题目录，避免根目录继续膨胀。
- 被替代文档不直接删除，移动到 `docs/归档` 并标注“历史版本”。
- 涉及接口/数据结构变更时，同步更新 `API规划.md` 与 `数据库设计.md`。
- 每次迭代结束后更新 `执行进度.md`，并在 `文档索引与状态.md` 维护状态。
