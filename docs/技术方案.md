# MES 技术方案

## 1. 技术选型

- **前端**：Flutter（支持 Web/桌面/移动统一代码基线）。
- **后端**：Python + FastAPI（高性能异步接口框架）。
- **数据库**：PostgreSQL（事务一致性、索引能力强）。
- **ORM**：SQLAlchemy + Alembic（模型与迁移管理）。
- **鉴权**：JWT + RBAC（角色权限控制）。
- **密码加密**：bcrypt。
- **配置管理**：pydantic-settings（.env 环境变量）。

## 2. 总体架构

```text
Flutter Client
    |
HTTP/HTTPS + JSON
    |
FastAPI Service
    |-- Auth & RBAC
    |-- User Module
    |-- Role Module
    |-- Process Module
    |-- Product Module
    |-- Product Parameter Module
    |-- Equipment Module
    |-- Maintenance Plan Module
    |-- Maintenance Work Order Module
    |-- Maintenance Record Module
    |-- Page Visibility Module
    |
PostgreSQL
```

## 3. 后端分层

- `app/api/v1/endpoints`：路由层，参数接收与响应格式统一。
- `app/services`：业务逻辑层，承载流程编排与规则校验。
- `app/models`：ORM 模型定义。
- `app/schemas`：Pydantic 请求/响应模型。
- `app/core`：配置、鉴权、中间件、安全。
- `app/db`：数据库会话管理。
- `app/bootstrap`：启动引导逻辑。

### 3.1 核心服务

- `bootstrap_seed_service`：初始化角色、工序、管理员账号。
- `user_service`：用户管理业务逻辑。
- `role_service`：角色管理业务逻辑。
- `process_service`：工序管理业务逻辑。
- `product_service`：产品与参数管理业务逻辑。
- `equipment_service`：设备管理业务逻辑。
- `maintenance_scheduler_service`：保养工单自动生成调度。
- `online_status_service`：设备在线状态跟踪。
- `page_catalog_service`：页面目录查询；页面可见性由权限快照派生，不再单独配置。

## 4. 前端分层（Flutter）

- `lib/pages`：页面层（登录、用户管理、产品管理、设备管理等）。
- `lib/widgets`：复用组件（自适应表格容器等）。
- `lib/models`：数据模型。
- `lib/services`：API 调用封装（认证、用户、产品、设备服务）。
- `lib/main.dart`：应用入口与路由配置。

## 5. 关键设计点

- **统一响应格式**：`code`、`message`、`data`，使用泛型 `ApiResponse[T]`。
- **审计字段统一**：`created_by`、`created_at`、`updated_by`、`updated_at`（通过 `TimestampMixin` 实现）。
- **角色权限控制**：
  - 4 个固定角色：`system_admin`、`production_admin`、`quality_admin`、`operator`
  - 使用 `require_role_codes` 依赖注入实现接口级权限控制
  - 操作员必须分配至少一个工序
- **产品参数模板**：
  - 支持 `Text` 和 `Link` 两种参数类型
  - 参数变更自动记录历史（`ProductParameterHistory`）
  - 支持预设参数和自定义参数
- **保养工单自动生成**：
  - 后台任务定期扫描保养计划
  - 根据周期和到期日期自动生成工单
  - 支持按工序分配执行人员
- **设备在线状态**：
  - 基于 TTL 的在线状态跟踪
  - 定期心跳更新状态

## 6. 安全与权限

- 登录后签发 JWT Token（有效期可配置，默认 120 分钟）。
- 后端按接口声明角色码，前端根据角色控制菜单与按钮。
- 所有写操作记录审计日志，含操作者与操作时间。
- 密码使用 bcrypt 加密存储。
- 删除敏感操作需要验证当前用户密码。

## 7. 数据库迁移与初始化

- 使用 Alembic 管理数据库结构迁移。
- 启动时自动执行：
  - 数据库创建（如不存在）
  - `alembic upgrade head`
  - 角色、工序、管理员账号初始化
- 支持手动初始化脚本：`python -m scripts.init_admin`

## 8. 部署建议（初期）

- 环境划分：`dev`、`test`、`prod`。
- 后端采用 `uvicorn` 启动，生产建议加 `gunicorn` 管理进程。
- PostgreSQL 独立部署并启用定时备份。
- 后续可引入 Docker Compose 简化环境一致性。
