# MES 项目规划（初版）

## 1. 项目目标

建设一套面向生产现场的 MES 系统，支持从产品资料到生产订单执行，再到首件管理的闭环管理，满足“可追溯、可查询、可审计”。

## 2. 功能范围

### 2.1 本期范围（MVP）

- 用户管理：用户、角色、权限、账号启停。
- 产品管理：产品基础资料维护。
- 产品参数管理：产品参数模板与参数项维护。
- 产品查询：按产品编码、名称、状态查询。
- 生产订单管理：订单创建、下发、状态流转。
- 生产订单查询：按订单号、产品、时间、状态查询。
- 首件管理：首件报检、审核、结论记录、历史追溯。

### 2.2 暂不纳入本期

- 设备直连（PLC/OPC UA）。
- 自动排产与高级计划。
- 质量统计大屏与 BI 报表平台。
- 多工厂多租户架构。

## 3. 用户角色

- 系统管理员：账号、角色、基础配置维护。
- 工艺/产品工程师：产品与参数维护。
- 计划员：生产订单创建与下发。
- 产线操作员：订单执行、首件提报。
- 质检员：首件审核与结论记录。
- 车间主管：生产与首件状态查询。

## 4. 核心业务流程

1. 维护产品与参数模板。
2. 计划员创建生产订单并下发。
3. 产线操作员执行订单并提报首件。
4. 质检员审核首件，结果为通过或驳回。
5. 管理人员基于订单与首件记录进行查询追溯。

## 5. 非功能要求

- 安全：基于 JWT 的认证与 RBAC 权限控制。
- 性能：常规查询接口在 1 秒内响应（95 分位，1 万级数据量）。
- 可用性：关键业务操作保留审计日志。
- 可维护性：前后端分离，接口契约清晰。

## 6. 分期建议

### 阶段一（MVP，建议 8-10 周）

- 完成本期 7 个核心功能模块。
- 提供基础权限、审计日志、错误码体系。

### 阶段二（增强，建议 4-6 周）

- 引入报表中心、消息通知、导出能力。
- 首件流程增加多级审批与电子签名。

### 阶段三（扩展）

- 对接设备数据、实现工序级数据采集。
- 逐步建设 OEE、良率等分析能力。

## 7. 成功标准（MVP）

- 用户可独立完成：账号配置、产品维护、订单下发、首件审核。
- 订单与首件可按条件准确查询并追溯历史。
- 关键接口稳定可用，具备基本测试与发布流程。
