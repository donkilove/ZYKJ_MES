# MES 项目规划

## 1. 项目目标

建设一套面向生产现场的 MES 系统，支持产品管理、产品参数管理、设备管理、设备保养计划与工单执行，满足"可追溯、可查询、可审计"。

## 2. 功能范围

### 2.1 已实现功能（当前版本）

- **用户与权限管理**：
  - 基于 JWT 的认证与 RBAC 权限控制
  - 4 个固定角色：系统管理员、生产管理员、品质管理员、操作员
  - 用户管理：查询、新增、更新、删除
  - 操作员可分配工序（激光打标、产品测试、产品组装、产品包装）
  - 非操作员角色不允许绑定工序

- **产品管理**：
  - 产品基础资料维护（产品名称）
  - 产品参数模板管理（支持 Text/Link 类型）
  - 产品参数值维护与历史追溯
  - 参数变更审计（记录操作人、变更内容、备注）

- **设备管理**：
  - 设备台账管理（编码、名称、型号、位置、负责人）
  - 设备启用/禁用状态管理
  - 设备在线状态跟踪

- **保养管理**：
  - 保养项目管理
  - 保养计划管理（周期、执行工序、计划开始日期）
  - 保养工单自动生成与执行
  - 保养记录管理

- **页面可见性配置**：
  - 支持按角色配置页面可见性
  - 动态控制前端菜单展示

### 2.2 规划中功能

- 生产订单管理：订单创建、下发、状态流转。
- 首件管理：首件报检、审核、结论记录、历史追溯。
- 注册审批流程：用户注册申请与审批。

### 2.3 暂不纳入本期

- 设备直连（PLC/OPC UA）。
- 自动排产与高级计划。
- 质量统计大屏与 BI 报表平台。
- 多工厂多租户架构。

## 3. 用户角色

- **系统管理员（system_admin）**：账号、角色、基础配置维护，拥有最高权限。
- **生产管理员（production_admin）**：产品管理、设备管理、保养计划与工单管理。
- **品质管理员（quality_admin）**：产品参数管理、保养记录查询、质量管理。
- **操作员（operator）**：设备保养工单执行、产品参数查询，必须分配至少一个工序。

## 4. 核心业务流程

1. **用户与权限管理流程**：
   - 系统初始化自动创建 4 个固定角色和默认工序
   - 管理员创建用户并分配角色
   - 操作员用户必须分配至少一个工序

2. **产品与参数管理流程**：
   - 创建产品（产品名称唯一）
   - 系统自动初始化产品参数模板
   - 维护产品参数值（支持 Text/Link 类型）
   - 所有参数变更自动记录历史

3. **设备保养管理流程**：
   - 创建设备台账
   - 创建保养项目
   - 为设备制定保养计划（周期、执行工序）
   - 系统自动生成保养工单
   - 操作员执行工单并记录保养结果

4. **页面可见性控制流程**：
   - 管理员配置各角色的页面可见性
   - 前端根据配置动态展示菜单

## 5. 非功能要求

- **安全**：基于 JWT 的认证与 RBAC 权限控制。
- **性能**：常规查询接口在 1 秒内响应（95 分位，1 万级数据量）。
- **可用性**：关键业务操作保留审计日志。
- **可维护性**：前后端分离，接口契约清晰。
- **自动化**：数据库迁移、初始化自动执行。

## 6. 分期建议

### 阶段一（已完成）

- 完成用户与权限管理、产品管理、设备管理、保养管理核心模块。
- 提供基础权限、审计日志、错误码体系。
- 实现保养工单自动生成与执行。

### 阶段二（规划中）

- 完成生产订单管理与首件管理。
- 实现注册审批流程。
- 引入报表中心、消息通知、导出能力。

### 阶段三（扩展）

- 对接设备数据、实现工序级数据采集。
- 逐步建设 OEE、良率等分析能力。

## 7. 成功标准（当前版本）

- 用户可独立完成：账号配置、产品维护、设备管理、保养计划与执行。
- 产品参数可按条件准确查询并追溯历史。
- 保养工单自动生成与执行闭环。
- 关键接口稳定可用，具备基本测试与发布流程。
