# 插件开发指南总览与阅读顺序

## 1. 目的

- 本目录是当前仓库插件体系的正式开发指南。
- 本目录用于统一宿主维护者与插件开发者的开发口径。
- 本目录只描述当前仓库已经采用的实现，不描述未来愿景或未落地方案。

## 2. 适用范围

适用对象：

1. 维护 Flutter 插件宿主的人
2. 在 `plugins/` 下新增或维护 Python 插件的人
3. 需要定位插件目录、解释器、启动协议和 UI 承载规则的人

不适用对象：

1. 最终使用插件中心的业务用户
2. 只关心 MES 业务模块而不接触插件体系的开发者

## 3. 与其他分册关系

- 本文档是阅读入口，先读它，再按角色进入后续分册。
- `10-宿主约定与目录结构.md` 负责解释宿主如何定位插件与解释器。
- `20-插件包结构与清单规范.md` 负责解释插件目录结构与 `manifest.json`。
- `30-Python入口与依赖加载.md` 负责解释 `launcher.py`、环境变量和 `sys.path`。
- `40-UI承载与前后端接口.md` 负责解释插件 UI 如何被宿主内嵌承载。
- `50-开发调试测试与交付.md` 负责解释开发流程、自测、联调和交付检查。
- `60-串口助手示例拆解.md` 负责用当前样板插件演示完整链路。

## 4. 当前插件体系边界

当前仓库中的插件体系采用以下固定边界：

1. 插件根目录固定为仓库根下的 `plugins/`
2. Python 运行时固定为 `plugins/runtime/python312/python.exe`
3. 插件以独立 Python 程序包存在，并自带业务逻辑与 Web UI
4. 宿主负责扫描插件、启动插件、停止插件、判断状态，并在主程序内部承载插件页面
5. 插件私有第三方依赖由插件自己携带，默认放在插件目录下的 `vendor/`
6. 插件启动后必须输出结构化 `ready` 消息，宿主根据该消息拿到 `entry_url` 与 `heartbeat_url`

这意味着：

1. 宿主不渲染插件内部业务 UI
2. 插件不决定解释器绝对路径
3. 插件不依赖系统已安装的 Python
4. 插件开发要求必须与当前 `serial_assistant` 样板保持一致

## 5. 当前目录总览

```text
plugins/
  runtime/
    python312/
      python.exe
  serial_assistant/
    manifest.json
    launcher.py
    app/
    vendor/
    web/
    tests/
```

各部分职责：

1. `plugins/runtime/python312/`：宿主统一提供的 Python 3.12 运行时
2. `plugins/<plugin_id>/manifest.json`：宿主读取的插件清单入口
3. `plugins/<plugin_id>/launcher.py`：插件 Python 启动入口
4. `plugins/<plugin_id>/app/`：插件业务代码
5. `plugins/<plugin_id>/vendor/`：插件自带第三方依赖
6. `plugins/<plugin_id>/web/`：插件 Web UI 静态资源
7. `plugins/<plugin_id>/tests/`：插件自身测试

## 7. 当前标准样板

当前仓库中已经存在一个标准样板插件：

- `plugins/serial_assistant/`

阅读和开发时，应默认把它视为以下事项的参照实现：

1. `manifest.json` 怎么写
2. `launcher.py` 如何读取环境变量并组装 `sys.path`
3. 本地 HTTP 服务如何提供 `entry_url` 和 `heartbeat_url`
4. `web/` 页面与 `app/` 逻辑如何配合
5. 插件自带依赖如何放进 `vendor/`

## 8. 规则用语

为了避免文档口径模糊，本目录统一采用以下表述：

1. `必须`：当前实现和交付口径中的硬约束
2. `应该`：推荐做法，通常也是最稳妥做法
3. `可以`：在不破坏当前协议前提下允许的可选做法

## 9. 常见误解

1. 误以为插件可以依赖系统 Python
   - 当前体系不允许这样做，解释器由宿主统一提供
2. 误以为宿主会渲染插件内部页面
   - 当前体系不是协议式 UI，而是插件自带 Web UI
3. 误以为环境变量覆盖就是正式口径
   - 不是，正式口径始终是仓库内固定路径
4. 误以为插件可以跳过 `ready` 消息
   - 不可以，宿主需要它来拿到页面入口和心跳地址
