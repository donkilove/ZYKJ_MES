# 宿主约定与目录结构

## 1. 目的

- 本文档说明宿主如何定位插件目录与 Python 运行时。
- 本文档说明插件中心在目录扫描和启动阶段采用的正式口径。

## 2. 适用范围

适用对象：

1. 宿主维护者
2. 需要判断插件为什么未被宿主发现的人
3. 需要核对解释器、插件目录和调试覆盖的人

## 3. 与其他分册关系

- 本文档解释宿主边界，不解释插件内部代码。
- `20-插件包结构与清单规范.md` 继续解释单个插件目录应该长什么样。
- `30-Python入口与依赖加载.md` 继续解释插件进程启动后如何装配依赖。
- `40-UI承载与前后端接口.md` 继续解释宿主拿到 `entry_url` 后如何承载页面。

## 4. 正式目录口径

宿主当前采用以下固定目录结构：

```text
plugins/
  runtime/
    python312/
      python.exe
  serial_assistant/
    manifest.json
    launcher.py
    app/
    vendor/
    web/
```

正式口径说明：

1. 宿主必须默认从仓库根下的 `plugins/` 扫描插件
2. 宿主必须默认从 `plugins/runtime/python312/python.exe` 定位解释器
3. 正式交付时不依赖 `frontend/build/.../runtime/python/`

## 5. 宿主定位规则

### 5.1 插件根目录

宿主默认插件根目录是：

```text
<repo>/plugins
```

当前实现中：

1. 优先读取 `MES_PLUGIN_ROOT`
2. 若未提供，则从 `executablePath` 或 `currentDirectory` 祖先链中寻找仓库根
3. 找到仓库根后，拼接 `plugins`
4. 若仓库根无法识别，则退回到可执行文件同级的 `plugins`

正式文档口径：

1. `plugins/` 是默认正式口径
2. `MES_PLUGIN_ROOT` 只是调试覆盖，不是正式交付路径

### 5.2 Python 解释器

宿主默认解释器路径是：

```text
<repo>/plugins/runtime/python312/python.exe
```

当前实现中：

1. 优先读取 `MES_PYTHON_RUNTIME_DIR`
2. 若未提供，则拼接 `plugins/runtime/python312/python.exe`

正式文档口径：

1. `plugins/runtime/python312/python.exe` 是默认正式口径
2. `MES_PYTHON_RUNTIME_DIR` 只是调试覆盖，不是正式交付路径

## 6. 宿主扫描规则

当前宿主扫描流程如下：

1. 解析插件根目录
2. 枚举插件根目录下的一级子目录
3. 只处理包含 `manifest.json` 的子目录
4. 成功解析 `manifest.json` 的目录标记为 `ready`
5. 解析失败的目录标记为 `invalid`

扫描阶段不会做的事：

1. 不会递归扫描多层目录
2. 不会自动安装插件依赖
3. 不会在扫描阶段启动插件进程

## 7. 宿主启动前检查

在点击“打开插件”后，宿主首先检查：

1. 插件目录是否存在
2. Python 解释器是否存在

当前错误语义：

1. 插件目录缺失
2. Python 运行时缺失

宿主应该把这类问题翻译成可理解的错误提示，而不是原始 `ProcessException`。

## 8. 示例

### 8.1 运行时定位器示意

当前宿主的核心定位规则可以概括为：

```dart
String resolvePythonExecutable() {
  final externalRuntimeDir = environment['MES_PYTHON_RUNTIME_DIR']?.trim();
  if (externalRuntimeDir != null && externalRuntimeDir.isNotEmpty) {
    return p.join(externalRuntimeDir, 'python.exe');
  }
  return p.join(resolvePluginRoot(), 'runtime', 'python312', 'python.exe');
}
```

### 8.2 调试覆盖示例

调试时可以临时覆盖：

```powershell
$env:MES_PLUGIN_ROOT = 'C:\temp\plugins'
$env:MES_PYTHON_RUNTIME_DIR = 'C:\temp\plugins\runtime\python312'
python start_frontend.py
```

但这不改变正式口径仍然是仓库内固定路径。

## 9. 常见错误

1. 把解释器重新部署回 `frontend/build/.../runtime/python/`
   - 这已经不是当前正式口径
2. 让插件在 `manifest.json` 里声明解释器绝对路径
   - 当前体系中解释器路径由宿主决定，不由插件决定
3. 在正式文档里把 `MES_PLUGIN_ROOT` 写成默认方案
   - 这会误导后续开发者
4. 在插件根目录下再套一层二级目录
   - 当前扫描器只扫描一级子目录
