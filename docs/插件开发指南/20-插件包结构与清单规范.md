# 插件包结构与清单规范

## 1. 目的

- 本文档说明单个插件目录应如何组织。
- 本文档说明宿主会从 `manifest.json` 中读取哪些字段。

## 2. 适用范围

适用对象：

1. 新建插件的开发者
2. 维护已有插件目录结构的人
3. 需要判断 `manifest.json` 为什么无法通过宿主扫描的人

## 3. 与其他分册关系

- 本文档只负责解释“插件长什么样”。
- `30-Python入口与依赖加载.md` 继续解释 `launcher.py` 如何启动插件。
- `40-UI承载与前后端接口.md` 继续解释插件启动后如何对外提供页面入口。
- `60-串口助手示例拆解.md` 提供真实样板目录的逐文件说明。

## 4. 标准目录结构

当前插件建议采用以下结构：

```text
plugins/<plugin_id>/
  manifest.json
  launcher.py
  app/
  vendor/
  web/
  tests/
```

各目录职责：

1. `manifest.json`：宿主读取的唯一清单入口
2. `launcher.py`：插件启动入口
3. `app/`：插件业务逻辑
4. `vendor/`：插件自带第三方依赖
5. `web/`：插件前端静态资源
6. `tests/`：插件自身测试

可以存在但不应当提交的内容：

1. `__pycache__/`
2. 临时日志
3. 本地调试临时文件

## 5. `manifest.json` 的当前结构

当前样板插件使用的清单如下：

```json
{
  "id": "serial_assistant",
  "name": "串口助手",
  "version": "0.1.0",
  "entry": {
    "type": "python",
    "script": "launcher.py"
  },
  "ui": {
    "type": "web",
    "mode": "embedded"
  },
  "runtime": {
    "python": "3.12",
    "arch": "win_amd64"
  },
  "dependencies": {
    "mode": "plugin_local",
    "paths": ["vendor", "app"]
  },
  "permissions": ["serial", "filesystem"],
  "lifecycle": {
    "startup_timeout_sec": 15,
    "heartbeat_interval_sec": 5
  }
}
```

## 6. 必填字段

当前宿主与样板插件口径下，以下字段应视为必填：

1. `id`
2. `name`
3. `version`
4. `entry.type`
5. `entry.script`
6. `ui.type`
7. `ui.mode`
8. `runtime.python`
9. `runtime.arch`
10. `dependencies.mode`
11. `dependencies.paths`
12. `lifecycle.startup_timeout_sec`
13. `lifecycle.heartbeat_interval_sec`

## 7. 字段约定

### 7.1 `id`

- 必须与插件目录名保持一致
- 应该使用小写字母、数字和下划线

### 7.2 `entry.script`

- 当前正式口径下应该是 `launcher.py`
- 不应该写成宿主绝对路径

### 7.3 `ui`

- 当前样板使用：
  - `type: web`
  - `mode: embedded`

这代表：

1. 插件页面由插件自己的 Web UI 提供
2. 宿主只负责内嵌承载

### 7.4 `dependencies.paths`

- 当前样板使用 `["vendor", "app"]`
- 这些路径必须是插件目录下的相对路径

### 7.5 `permissions`

- 当前只做声明式展示，不是强安全沙箱
- 应该如实声明插件会用到的能力

## 8. 最小合法样例

下面是一个最小合法插件的清单示例：

```json
{
  "id": "demo_tool",
  "name": "示例工具",
  "version": "0.1.0",
  "entry": {
    "type": "python",
    "script": "launcher.py"
  },
  "ui": {
    "type": "web",
    "mode": "embedded"
  },
  "runtime": {
    "python": "3.12",
    "arch": "win_amd64"
  },
  "dependencies": {
    "mode": "plugin_local",
    "paths": ["vendor", "app"]
  },
  "permissions": [],
  "lifecycle": {
    "startup_timeout_sec": 15,
    "heartbeat_interval_sec": 5
  }
}
```

## 9. 宿主视角下的扫描结果

当前扫描器会把插件目录分成两类：

1. `ready`
   - `manifest.json` 成功解析
2. `invalid`
   - `manifest.json` 缺失、字段错误或 JSON 非法

因此，`manifest.json` 出问题时，插件可能不会被正常打开。

## 10. 常见错误

1. `id` 与插件目录名不一致
2. `entry.script` 不是 `launcher.py`
3. `dependencies.paths` 写成绝对路径
4. `runtime.python` 漏写或与当前宿主解释器不一致
5. 把 `vendor/` 放到宿主公共目录，而不是插件目录内
