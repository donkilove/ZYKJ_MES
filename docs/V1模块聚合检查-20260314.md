# 功能规划 V1 模块聚合检查（2026-03-14）

- 适用范围：`docs/功能规划V1` 下 7 个模块（用户、产品、工艺、生产、品质、设备、消息）
- 目的：汇总各模块相对 V1 需求说明的实现状态，便于后续由 Claude 按优先级继续推进功能完成度
- 汇总依据：各模块需求说明 + 各模块最新检查结果文档
- 核查方式：以现有检查结果为主进行聚合整理；原始检查大多基于静态代码核查，部分模块附带自动化验证，整体仍未替代最终联调验收

---

## 1. 聚合结论

当前 `7` 个模块中，**无模块达到“完全满足 V1 需求说明”**。

- `2` 个模块已完成 P0 修复，可进入联调验收：`用户模块`、`消息模块`
- `4` 个模块可按”有条件验收/补缺后复检”推进：`工艺模块`、`生产模块`、`品质模块`、`设备模块`
- `1` 个模块可内部联调但不建议直接正式验收：`产品模块`

整体判断：

1. 系统主体骨架已成型，主业务链路在多数模块中已经可跑通。
2. 当前主要短板不是“完全没做”，而是集中在 `P0 稳定性问题`、`验收字段/筛选/导出缺口`、`审计留痕不足`、`跨模块联动不完整`。
3. 若要让 Claude 继续推进，建议优先解决“阻断可用性/稳定性”的问题，再处理“验收完备度”问题，最后补分析增强项。

## 3. 按模块核实结论

### 3.1 用户模块

- 当前状态：`不满足验收`
- 结论来源：`docs/功能规划V1/用户模块/检查结果-20260313.md`
- 核心问题：
  - `backend/app/api/v1/endpoints/auth.py:348` 存在语法错误，属于基础可用性阻断项
  - 注册审批通过时“管理员必须设置初始密码”的规则未严格落地
  - 首次登录强制改密只有后端标记，前端登录流程未形成强制跳转闭环
  - 系统管理员相关权限规则未完全收敛，存在越权风险
- 结论：用户模块必须先修 P0，再谈联调验收

### 3.2 产品模块

- 当前状态：`可内部联调，不建议按 V1 直接验收`
- 结论来源：`docs/功能规划V1/产品模块/检查结果-20260313.md`
- 核心问题：
  - 产品删除仍为物理删除，不符合“逻辑删除”要求
  - 产品列表导出、版本参数导出、参数查询导出均缺失
  - 产品详情抽屉/查看详情入口缺失
  - 参数说明字段编辑链路未贯通
  - Link 类型参数缺后端格式校验
- 结论：产品模块主链路已可用，但验收缺口较集中，适合作为 Claude 的专项补齐模块

### 3.3 工艺模块

- 当前状态：`部分满足 / 有条件验收`
- 结论来源：`docs/功能规划V1/工艺模块/检查结果-20260313.md`
- 核心问题：
  - 工艺引用分析仅覆盖工段/工序，缺模板/产品维度与风险字段
  - 未明确看到“同一工段下工序名称唯一”规则
  - 模板复制仅支持同产品复制，缺系统母版套版与跨产品复制
  - 多个页面缺搜索、筛选、导出完备度
  - 归档、复制等关键动作审计证据不足
- 结论：工艺模块核心链路较完整，后续重点是补“验收完整度”和“风险分析能力”

### 3.4 生产模块

- 当前状态：`部分满足 / 有条件验收`
- 结论来源：`docs/功能规划V1/生产模块/检查结果-20260313.md`
- 核心问题：
  - 订单管理缺开始日期范围/交期范围筛选
  - 前端未展示产品版本字段
  - 缺少“并行实例追踪页面”和入口
  - 订单查询列表字段与筛选项不完整
  - 代班审批语义与前端文案不一致
- 结论：生产主业务可跑通，但还没达到需求文档要求的页面完备度

### 3.5 品质模块

- 当前状态：`部分满足 / 有条件验收`
- 结论来源：`docs/功能规划V1/品质模块/检查结果-20260313.md`
- 核心问题：
  - 品质入口下复用生产页面时，报废导出、维修导出、完成维修被禁用
  - 首件处置存在“无痕覆盖”风险，审计链路不足
  - 每日首件、品质数据页面筛选维度不足
  - 趋势分析仍偏基础统计，缺 Top 缺陷、不良分布、产品对比等能力
  - 报废/维修详情追溯字段不完整
- 结论：品质模块比之前明显收敛，但距离完整验收还差“归口能力可执行 + 审计追溯 + 分析深度”

### 3.6 设备模块

- 当前状态：`部分满足 / 有条件验收`
- 结论来源：`docs/功能规划V1/设备模块/检查结果-20260314.md`
- 核心问题：
  - 自动生成调度按 4 个值解包服务返回，服务实际默认返回 3 个值，存在运行时异常风险
  - 设备规则 / 运行参数页面未实现
  - 台账、项目、计划、记录导出能力缺失
  - 多处筛选控件与列表字段不完整
- 结论：设备保养闭环可运行，但应先修调度 P0 稳定性问题

### 3.7 消息模块

- 当前状态：`可进入联调，不建议正式验收`
- 结论来源：`docs/功能规划V1/消息模块/检查结果-20260314.md`
- 核心问题：
  - 实时推送协议命名存在不一致风险：`message_created` vs `new_message`
  - 生产、品质来源尚未接入消息服务
  - 跳转缺少“无权限”与“来源对象失效”提示
  - 单条已读/全部已读未写入审计日志
- 结论：消息模块骨架完整，适合作为跨模块联动补齐的中枢模块推进

## 4. 共性缺口归纳

跨模块看，当前最影响 V1 完成度的不是单点页面，而是以下 5 类共性问题：

1. **P0 可用性 / 稳定性问题**
   - 用户模块语法错误
   - 设备模块调度解包缺陷
   - 消息模块实时协议不一致

2. **验收完备度不足**
   - 多模块存在筛选条件缺失、列表字段不全、详情页缺口、导出能力缺失
   - 典型模块：产品、工艺、生产、设备、品质

3. **审计与追溯不完整**
   - 用户权限特殊规则闭环不足
   - 产品关键动作审计不足
   - 工艺归档/复制等动作审计不足
   - 品质首件处置存在覆盖历史风险
   - 消息已读操作缺审计留痕

4. **跨模块联动不完整**
   - 消息模块尚未接入生产、品质关键事件
   - 工艺引用分析对产品/模板维度覆盖不足
   - 品质与生产之间的报废/维修追溯链路仍不够完整

5. **需求语义未完全收敛**
   - 用户模块审批与强改密规则
   - 生产模块代班审批语义
   - 产品模块“参数按版本绑定”的实现语义

## 5. 建议 Claude 的推进顺序

建议不要平均推进 7 个模块，而是按“先止血、再补验收、最后做增强”的顺序推进。

### 第一阶段：先修阻断项（必须优先）

1. 用户模块
   - 修复 `auth.py` 语法错误
   - 收紧审批通过初始密码规则
   - 补齐首次登录强制改密前端闭环
   - 收敛系统管理员权限分配规则

2. 设备模块
   - 修复自动生成调度返回值解包问题
   - 回归验证自动生成、消息推送、异常分支日志

3. 消息模块
   - 统一 WebSocket 事件协议
   - 确保消息创建后实时推送与未读数刷新链路一致

### 第二阶段：补验收缺口（最影响完成度）

4. 产品模块
   - 改为逻辑删除
   - 补齐三类导出
   - 补“查看详情/详情抽屉”
   - 打通参数说明写入链路
   - 增加 Link 格式校验

5. 生产模块
   - 补订单管理筛选与产品版本列
   - 新增并行实例追踪页与入口
   - 统一代班审批语义、接口、文案

6. 品质模块
   - 在品质入口开放报废/维修实际执行能力
   - 补齐首件处置不可无痕覆盖的审计链路
   - 补每日首件与品质数据多维筛选

7. 工艺模块
   - 补模板/产品维度引用分析
   - 落实工序名称唯一规则
   - 增强模板复制/套版范围

### 第三阶段：补跨模块联动和增强项

8. 消息模块接入生产、品质关键事件
9. 产品/工艺/品质/设备统一补关键操作审计日志
10. 对筛选、导出、详情、追溯字段做一次跨模块一致性收口
