# 02_用户_认证_会话_在线状态

## 1. 模块职责与边界
### 1.1 职责
- 用户增删改查（含人类账号/虚拟账号）
- 注册申请审批
- 登录、登出、会话 token 管理
- 在线状态心跳与超时折算
- 口令校验能力（用于高风险操作二次确认）

### 1.2 边界
- 不处理订单流转业务（见 06/07/08）
- 不处理参数规则（见 05）

## 2. 入口与调用链
- 登录入口：`src/ui/login_window.py`
- 用户管理入口：`src/ui/son_page/user_management_page.py`
- 控制器：`auth_controller.py`、`user_controller.py`
- 服务层：`user_service.py`
- 实现层：`user_impl.py`

调用链：
`UI -> user/auth_controller -> user_service -> user_impl -> users/user_status/register_requests`

## 3. 业务对象与状态
### 3.1 对象
- `users`
  - `username`, `password`, `role`
  - `account_type`, `owner_user_id`, `device_code`, `max_parallel_sub_orders`
- `user_status`
  - `status`, `last_updated`, `session_token`
- `register_requests`
  - 待审批注册请求

### 3.2 状态
- 在线状态：`在线` / `离线`
- 注册流程：`待处理` ->（审批）入 users /（拒绝）删除

## 4. 核心流程
### 4.1 登录
1. 读取 users 中的哈希口令。
2. `_verify_password` 校验。
3. 生成新 session token。
4. 更新 `user_status` 为在线。
5. 若旧 token 在线且心跳有效，形成顶号语义（旧端在会话检查中被踢）。

### 4.2 登出
1. 主窗口退出或显式退出触发 `user_service.logout` / `update_user_status(离线)`。
2. 若传入 `session_token`，按 token 精确更新，避免误影响其他会话。

### 4.3 在线心跳
- 主窗口每 15 秒调用 `update_user_status(在线)`。
- 用户列表查询时通过 `last_updated` 与 30 秒阈值计算“在线/离线”。

### 4.4 注册审批
- 注册写入 `register_requests`。
- 审批通过：将请求转入 `users`，并删除请求。
- 审批拒绝：直接删除请求。

## 6. 数据库交互
- 查询用户列表：`users LEFT JOIN user_status`
- 登录写会话：`UPDATE/INSERT user_status`
- 注册审批：`SELECT register_requests` + `INSERT users` + `DELETE register_requests`

## 7. 权限与可见性
- 系统管理员默认可见用户管理页。
- 其他角色通常通过注册申请进入系统。
- 删除/编辑用户由页面和角色共同约束。

## 8. 并发与一致性
- 会话 token 防止误踢其他会话。
- 在线状态采用主动心跳 + 被动超时双机制。
- 用户缓存与全局 DB 缓存在写操作后清理。

## 9. 失败路径
- 用户不存在/密码错误：登录失败。
- 用户名重复：新增失败。
- 会话失效：主窗口强制退出。
- 配置读取失败：回退默认可见性。

## 10. 耦合点
- 与主窗口耦合：会话守卫与在线心跳。
- 与权限模块耦合：角色可见性。
- 与订单模块耦合：`max_parallel_sub_orders` 影响并发生产。

## 11. 回归测试（10条）
1. 正确口令登录成功。
2. 错误口令登录失败。
3. 同账号重复登录触发顶号。
4. 退出后用户状态及时离线。
5. 新增用户校验唯一性。
6. 更新用户角色后菜单可见性变化。
7. 注册申请审批成功转正。
8. 注册申请拒绝后请求消失。
9. `verify_user_password` 可用于二次确认。
10. 会话失效时旧端自动回登录页。
