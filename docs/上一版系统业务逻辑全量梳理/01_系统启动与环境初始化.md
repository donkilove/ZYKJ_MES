# 01_系统启动与环境初始化

## 1. 启动入口
- 入口文件：`src/main.py`
- 主函数：`main()`
- 关键对象：`SystemInitTool`、`LoginWindow`、`MainWindow`

## 2. 启动步骤（源码事实）
1. 设置 Qt 插件路径与应用图标环境。
2. 初始化 UI 服务：`ui_service.initialize_app()`。
3. 显示加载页：`ui_service.show_window('loading')`。
4. 执行环境初始化：`SystemInitTool.init_environment()`。
5. 初始化完成后显示登录页并绑定回调。
6. 登录成功后创建 `MainWindow`，连接 `logout_signal`。
7. 程序退出时通过 `atexit` 清理数据库连接。

## 3. `SystemInitTool` 细节
文件：`src/utils/system_init_tool.py`

### 3.1 `init_environment()`
- 检查目录：resources/logs/data
- 设置环境变量：`PYTHONIOENCODING=utf-8`
- 检查 Python 版本（>=3.8）
- 检查管理员账号，不存在则自动创建

### 3.2 管理员兜底创建
- 角色：`系统管理员`
- 默认口令：`123456`（PBKDF2 哈希）
- 用户名优先序：`admin` → `系统管理员` → `admin{n}`

## 6. 失败路径与处理
1. 初始化异常
- 行为：加载页弹 `MessageBox`，程序退出。

2. 数据库不可达
- 行为：初始化中断，登录页不会进入。

3. 管理员创建失败
- 行为：抛异常阻断启动，要求修复数据库权限或连接。

4. 退出清理失败
- 行为：仅记日志，不阻断进程退出。

## 7. 验收建议
1. 空库首次启动可自动生成管理员。
2. 二次启动不会重复创建管理员。
3. 登录后主窗口定时器正常启动。
4. 退出登录后可回到登录页。
5. 托盘退出可真正终止进程并释放连接。
