# 16_异常场景_边界条件_已知风险

## 1. 模块职责与边界
- 职责：汇总旧系统各模块常见异常、边界触发条件、失败后果、补偿策略和风险等级。
- 范围：用户认证、会话、权限、订单流转、首件、子订单并发、维修闭环、消息、导出、插件。

## 2. 异常分类总览
1. 输入与参数异常（用户名、密码、日期、数量、文件路径）。
2. 权限异常（角色不可见、工序角色不匹配、审批权限不足）。
3. 状态机异常（非法状态跳转、重复提交、重复审批）。
4. 一致性异常（并发报工、缓存陈旧、事件延迟）。
5. 资源异常（数据库连接失败、插件文件丢失、导出路径不可写）。

## 3. 高优先级异常清单
| 分类 | 触发条件 | 典型报错/现象 | 影响范围 | 处理策略 |
|---|---|---|---|---|
| 会话 | 同账号多端登录 | 旧端被顶号/强制下线 | 用户管理、主窗口 | 以新 token 为准，旧 token 心跳被拒绝 |
| 订单状态机 | 未首件直接开工 | 开工失败/按钮不可用 | 生产执行 | 先完成首件记录再开工 |
| 并发报工 | 多人同工序同时结束生产 | 数量不一致风险 | 订单/子订单 | 事务+process_status 校正+可见量重算 |
| 维修闭环 | 原因数量总和≠送修数量 | 维修完成失败 | 维修页、品质统计 | 强校验并回滚 |
| 导出 | 路径无权限/文件占用 | 导出失败提示 | 查询页 | 提示后重试，不影响主流程 |

## 4. 用户与会话异常
### 4.1 顶号登录
- 触发：同用户名在心跳有效期内再次登录。
- 行为：新会话写入新 `session_token`，旧端 `is_session_valid` 失败。
- 风险：旧端短时间仍显示在线（直到下一轮守护检查）。

### 4.2 退出状态不同步
- 触发：异常退出（崩溃/断电）。
- 行为：无法主动写离线，依赖在线超时判断变离线。
- 建议：保留主动 logout + 心跳超时双机制。

## 5. 权限与可见性异常
### 5.1 visibility 配置损坏
- 行为：`load_visibility_config` 解析失败时回落默认映射。
- 风险：角色页面可见性回到系统默认，不是最近人工配置。

### 5.2 页面可见但动作无权限
- 典型：能进入页面但按钮后端校验失败（例如非工序角色发起协助）。
- 建议：UI 前置禁用按钮 + 后端兜底校验同时保留。

## 6. 订单流转异常
1. 订单不存在：所有操作应快速失败。
2. 状态不匹配：例如对 `生产完成` 再开工应拒绝。
3. 删除订单中途失败：需验证事务是否完整回滚。
4. 兼容状态别名（`已完成/生产完成`）导致统计偏差。

## 7. 子订单并发异常
1. 子订单隐藏后未恢复：造成“有量不可见”。
2. 授权审批重复提交：状态冲突。
3. 代班人与申请人相同：业务语义冲突应拒绝。
4. 并发结束生产后 `completed_quantity` 与 `assigned_quantity` 关系异常。

## 8. 首件与校验码异常
1. 校验码为空、错误、过期：应拒绝首件提交。
2. 工序首件记录缺失但状态已进行中：属于不一致脏数据。
3. 再次首件流程与普通首件流程混用导致按钮状态错乱。

## 9. 维修与返修异常
1. 维修单不存在/已完成再次提交。
2. 原因项为空或总量不守恒。
3. 回流工序名不合法，导致回流写入失败。
4. 报废补充逻辑漏刷新上游目标量。

## 10. 消息与通知异常
1. 角色消息无匹配用户：消息未投递。
2. 弹窗显示失败：用户感知延迟，但消息仍在未读列表。
3. 已读回写失败：重复提醒。

## 11. 导出与插件异常
### 11.1 导出
- 路径不存在、无权限、文件被占用会导致失败。
- 导出失败通常不影响查询主流程。

### 11.2 插件
- `plugin_code` 非法（非 Python 标识符）。
- 插件文件缺失。
- 插件未启用或无 `execute` 入口。

## 12. 风险等级矩阵
| 风险等级 | 问题 | 建议 |
|---|---|---|
| 高 | 生产数量不守恒、维修回流不一致 | 全链路事务 + 单元测试 + 审计日志 |
| 高 | 状态机非法跳转 | 统一状态机层封装 |
| 中 | 消息重复/延迟 | 事件+轮询双通道，增强幂等 |
| 中 | 缓存短时陈旧 | 写后清缓存 + 版本号校验 |
| 低 | 导出样式差异 | 与业务无关，可延后优化 |

## 13. 观测与排障指引
1. 先看前端错误提示和 `LogTool` 日志时间线。
2. 对照 `附录_B` 找到函数入口。
3. 抽样检查关键表：`orders/order_processes/process_status/order_sub_orders/repair_orders/messages/user_status`。
4. 检查是否发出 `notify_change(scope, action)`。

## 14. 回归测试清单（10条）
1. 顶号登录后旧端 3 秒内被下线。
2. 配置文件损坏后能回落默认可见性。
3. 首件错误校验码无法放行。
4. 并发报工后总量守恒。
5. 协助授权重复审批被拒绝。
6. 维修原因总量不守恒被拒绝。
7. 回流后工序目标量及时刷新。
8. 消息已读回写后不再重复弹窗。
9. 插件非法 code 被拒绝。
10. 导出失败仅提示，不导致页面崩溃。
