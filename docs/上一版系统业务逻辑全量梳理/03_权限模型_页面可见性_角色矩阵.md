# 03_权限模型_页面可见性_角色矩阵

## 1. 模块职责与边界
- 职责：根据角色决定主窗口菜单、快捷入口和页面可访问性。
- 边界：不处理接口级鉴权；仅做桌面端 UI 可见性控制。

## 2. 实现路径
- 配置源：`resources/json/visibility_config.json`
- 读取函数：`UserImpl.load_visibility_config()`
- 对外接口：`ui_service.get_user_visible_pages(role)`
- 页面挂载点：`MainWindow.add_navigation_items()`

## 3. 核心规则
1. 若角色不在配置中，最小可见页为 `product_parameter_query`。
2. `product_parameter_query` 对所有角色强制开放（代码硬规则）。
3. 主窗口导航与首页快捷按钮均使用同一套可见页面列表。

  - `setup_shortcut_buttons`

## 6. 页面挂载策略
1. 登录成功拿到 `user_info.role`。
2. 调用 `get_user_visible_pages(role)` 获取 code 列表。
3. 对每个 code 判断后实例化页面并 `addSubInterface`。
4. 首页快捷按钮按同样条件生成，保证一致。

## 7. 异常与兜底
- 配置文件缺失：使用 `generate_default_visibility_config()`。
- JSON 解析失败：记录 warning，回退默认配置。
- 未知角色：只保留公共查询页，避免空白主界面。

## 8. 回归测试（10条）
1. 系统管理员菜单完整。
2. 生产管理员无用户管理菜单。
3. 品质管理员可见质量查询和首件管理。
4. 维修角色可见维修查询。
5. 任意角色可见产品参数查询。
6. 快捷入口与侧边导航一致。
7. 配置文件删除后仍可登录并显示默认菜单。
8. 配置文件损坏后系统不中断。
9. 角色改动后重新登录菜单即时生效。
10. 未知角色账号不会导致主窗口崩溃。
