# 执行总则

## 1. 定位与适用范围

- 本分册承接原统一手册第 1-4 章的执行总则内容。
- 本分册用于说明项目范围、优先级、核心原则与必须遵守的基础约束。
- 本分册适用于仓库全部目录，除非子目录另有更严格的局部约束。
- 只要任务涉及项目规则、指挥官流程、工具门禁、留痕模板或规则文档本身，均按本分册与根 `AGENTS.md` 执行。

## 2. 优先级与冲突处理

### 2.1 优先级

1. 系统级与开发者级强制指令。
2. 用户显式指令。
3. 根 `AGENTS.md`。
4. 本分册及其余 `docs/AGENTS/*.md` 正文。
5. 支撑文档与辅助说明文档。

### 2.2 冲突处理

- 若用户显式指令与本规则集冲突，优先执行用户指令，并在前置说明与任务日志中记录偏差原因。
- 若本规则集对同一事项在不同位置存在表述差异，按“更严格、更新、可执行”的口径处理。
- 若不同分册之间出现冲突，以 `docs/AGENTS/00-导航与装配说明.md` 中定义的顺序判定。

## 3. 核心原则

1. 强制优先：规则一旦命中，默认必须执行，不以个人偏好覆盖。
2. 结果导向：以可交付、可验证、可审计为完成标准，不交付占位方案。
3. 可追溯：关键输入、假设、证据、验证、失败重试与迁移口径必须留痕。
4. 最小变更：在满足目标前提下控制改动边界，避免无关扩散。
5. 工具优先：研究、实现、验证优先使用既定工具链，不凭空主观判断。
6. 先验证后通过：无独立验证结论，不得标记完成。
7. 默认不中断：除硬阻塞外，不因普通实现分歧暂停等待。

## 4. 强制约束

### 4.1 通用执行约束

- 所有沟通、代码注释、文档与新增文件默认使用中文，编码为 UTF-8（无 BOM）。
- 每次任务开始与结束都要更新 `evidence/` 或等效项目日志。
- 每次对外回复开头提供“前置说明”；若默认优先使用的当前会话宿主工具或 MCP 服务存在缺失、未配置、不可达、权限不足，或受更高优先级指令约束无法使用，必须在前置说明中写明缺失工具、原因、替代工具与影响范围；宿主命令若出现“命令可解析但二进制不可执行、进程创建被拒绝、命中受限安装路径”等情况，按不可用处理；若已实施修复，需补充至少一条真实命令复测结果；若存在外部调用，在末尾补“工具调用简报”。
- 盘点或汇报当前会话可用工具、MCP、技能时，必须同时核对工具命名空间、MCP tools、MCP resources/templates 与至少一次可行的只读真实调用；不得因 `list_mcp_resources` 或 `list_mcp_resource_templates` 为空，就判定 Docker Toolkit / `MCP_DOCKER` 或其他已暴露 MCP tools 不可用。
- 编码前必须完成当前会话可用的 `Sequential Thinking` 或等效任务拆解分析；若相关工具不可用，必须记录降级原因、触发时间、替代动作、影响范围与残余风险，并在 `evidence/` 中同步留痕。
- 工作流状态统一使用 `update_plan` 或 `TodoWrite` 维护；若工具不可用，允许在任务日志中维护步骤、状态、验收标准与更新时间。
- 仅运行安全命令，禁止破坏性清理、泄露密钥、令牌或内部敏感链接。
- 所有 git 提交信息必须使用中文，不得使用英文或中英混杂标题作为默认提交口径。
- 默认采取破坏性改动并拒绝向后兼容，但必须在交付中明确迁移口径；若无迁移需求，统一写“无迁移，直接替换”。

### 4.2 测试与质量基线

- 后端默认测试基线：`pytest`。
- 前端默认测试基线：`flutter test` 与 `integration_test`。
- `desktop_tests/flaui/` 仅作为历史保留或按需 fallback，不是默认测试主线，且必须串行执行。
- 构建、编译、静态检查必须零报错；关键路径应有真实验证证据。
- 若任务命中供应链或发布前审计，需保留 SBOM、扫描摘要或等效证据。

### 4.3 用户模块与同步收敛

- 只要命中用户模块行为、入口、筛选、状态流转、文案边角分支等变更，必须在同一轮任务内同步收敛：
  - 后端/API
  - Flutter
  - `integration_test`
  - `evidence/`
  - 执行命令口径
- 禁止只改一端后将其余部分留待下轮补齐。

### 4.4 安全与退避

- 遇 HTTP `429` 固定退避 20 秒。
- 遇 HTTP `5xx` 或超时，退避 2 秒后最多一次重试。
- 若重试后仍失败，需提供保守离线结论，并明确局限与下一步建议。
- 默认仅维持最低安全基线，不额外引入超出任务目标的安全设计。
