# 执行总则

## 1. 定位与适用范围

- 本分册承接原统一手册第 1-4 章的执行总则内容。
- 本分册用于说明项目范围、优先级、核心原则与必须遵守的基础约束。
- 本分册适用于仓库全部目录，除非子目录另有更严格的局部约束。
- 只要任务涉及项目规则、轻量工作流、工具建议、留痕模板或规则文档本身，均按本分册与根 `AGENTS.md` 执行。

## 2. 优先级与冲突处理

### 2.1 优先级

1. 系统级与开发者级强制指令。
2. 用户显式指令。
3. 根 `AGENTS.md`。
4. 本分册及其余 `docs/AGENTS/*.md` 正文。
5. 支撑文档与辅助说明文档。

### 2.2 冲突处理

- 若用户显式指令与本规则集冲突，优先执行用户指令；涉及安全、数据丢失或不可逆操作时，先说明风险再执行。
- 若本规则集对同一事项在不同位置存在表述差异，按“用户当前目标优先、更新、可执行、少约束”的口径处理。
- 若不同分册之间出现冲突，以 `docs/AGENTS/00-导航与装配说明.md` 中定义的顺序判定。

## 3. 核心原则

1. 结果导向：以可交付、可验证为完成标准，不交付占位方案。
2. 最小必要：规则、工具、留痕和计划只服务于当前目标，不把小任务做成重流程。
3. 上下文优先：改动前读相关代码、文档和历史结论，沿用现有模式。
4. 真实验证：完成前运行与改动匹配的检查；不能验证就说明原因和风险。
5. 可追溯：复杂、高风险、跨模块、规则变更和提交前保留必要记录。
6. 默认不中断：除硬阻塞外，不因普通实现分歧暂停等待。

## 4. 强制约束

### 4.1 通用执行约束

- 所有沟通、代码注释、文档与新增文件默认使用中文，编码为 UTF-8（无 BOM）。
- 小任务不强制 `evidence/`、计划工具或 `Sequential Thinking`；复杂、跨模块、高风险、规则变更、排障超过 15 分钟或提交前，需使用当前可用的计划/拆解方式，并按需更新 `evidence/`、`progress.md` 或等效项目日志。
- 对外回复不强制固定“前置说明”格式；当工具缺失、降级、外部调用失败、验证无法执行或影响结论可靠性时，需在回复中说明原因、替代动作与影响范围。
- 盘点或汇报当前会话可用工具、MCP、技能时，需同时核对工具命名空间、MCP tools、MCP resources/templates 与至少一次可行的只读真实调用；不得因 `list_mcp_resources` 或 `list_mcp_resource_templates` 为空，就判定 Docker Toolkit / `MCP_DOCKER` 或其他已暴露 MCP tools 不可用。
- 若仓库存在 Graphify 项目图谱（`graphify-out/GRAPH_REPORT.md` 或 `graphify-out/graph.json`），架构理解、跨模块排查、入口定位与关系追踪时优先作为结构导航；只有形成长期复用价值的稳定项目事实时，才写入 `docs/project-memory/`，并按需记录 Graphify 刷新或待语义更新状态。Graphify 结论不得覆盖源码、测试、接口契约、本规则集与真实验证。
- Memory MCP 只记录长期偏好、稳定约定和可复用项目事实；不记录密钥、隐私、临时流水或未验证猜测。
- 仅运行安全命令，禁止破坏性清理、泄露密钥、令牌或内部敏感链接；需要停止进程、删除文件、重写历史或影响外部环境时，先确认目标与影响范围。
- 所有 git 提交信息必须使用中文，不得使用英文或中英混杂标题作为默认提交口径。
- 提交前至少执行 `git status --short --untracked-files=all` 与 `git diff --check`，并确认提交范围不包含无关或敏感文件。
- 涉及迁移或兼容性取舍时，按用户目标选择直接替换或兼容保留，并在交付中说明迁移口径；无迁移需求时可省略固定表述。

### 4.2 测试与质量基线

- 后端常用测试基线：`pytest`。
- 前端常用测试基线：`flutter test` 与按需 `integration_test`。
- `desktop_tests/flaui/` 仅作为历史保留或按需 fallback，不是默认测试主线，且必须串行执行。
- 构建、编译、静态检查在被改动触发或作为发布/提交门槛时需零报错；关键路径应有真实验证证据。
- 若任务命中供应链或发布前审计，需保留 SBOM、扫描摘要或等效证据。

### 4.3 用户模块与同步收敛

- 涉及接口契约、真实用户流程、用户模块入口、筛选、状态流转或文案边角分支时，同步检查并按需收敛：
  - 后端/API
  - Flutter
  - `integration_test`
  - 执行命令口径
- 不涉及这些边界的小改动不强制全链路收敛。

### 4.4 安全与退避

- 遇 HTTP `429` 固定退避 20 秒。
- 遇 HTTP `5xx` 或超时，退避 2 秒后最多一次重试。
- 若重试后仍失败，需提供保守离线结论，并明确局限与下一步建议。
- 默认仅维持最低安全基线，不额外引入超出任务目标的安全设计。
