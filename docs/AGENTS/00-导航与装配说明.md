# AGENTS 导航与装配说明

## 1. 目的

- 本文件定义根 `AGENTS.md` 与 `docs/AGENTS/*.md` 的装配关系。
- 目标是把单文件高内聚规则拆为多文件正文，同时保持“单一规则集”而非“并列规则源”。

## 2. 装配规则

- 根 `AGENTS.md` 负责：
  - 默认读取入口
  - 适用范围与优先级
  - 必须立即生效的硬约束
  - 正文分册阅读顺序
- `docs/AGENTS/*.md` 负责：
  - 规则正文细则
  - 表格、模板与索引
  - 对应专题的边界、门禁和执行细节
- 任何新增规则都必须先判断归属：
  - 入口级硬约束，放根 `AGENTS.md`
  - 细则与操作口径，放分册正文

## 3. 阅读顺序

1. `00-导航与装配说明.md`
2. `10-执行总则.md`
3. `20-指挥官模式与工作流.md`
4. `30-工具治理与验证门禁.md`
5. `40-质量交付与留痕.md`
6. `50-模板与索引.md`

## 4. 冲突处理

- 判定顺序：
  1. 系统级与开发者级强制指令
  2. 用户显式指令
  3. 根 `AGENTS.md`
  4. `10-执行总则.md`
  5. `20-指挥官模式与工作流.md`
  6. `30-工具治理与验证门禁.md`
  7. `40-质量交付与留痕.md`
  8. `50-模板与索引.md`
  9. 其他支撑文档与历史材料
- 同层内容冲突时，按“更严格、更新、可执行”的口径处理。

## 5. 维护边界

- 不得在多个分册重复定义同一条规则，只允许在一个地方正式定义，其他地方以引用或摘要方式提及。
- 根 `AGENTS.md` 不再承载完整模板、长表格和历史收口说明。
- 分册正文若修改影响入口级硬约束，必须同步更新根 `AGENTS.md`。
- 历史 `evidence/` 允许保留旧路径与旧语境，不因本轮分册化批量回写。

## 6. 当前分册职责

- `10-执行总则.md`：适用范围、优先级、核心原则、强制约束与基础质量基线
- `20-指挥官模式与工作流.md`：角色边界、触发条件、最小闭环、拆分标准与收尾要求
- `30-工具治理与验证门禁.md`：工具口径、分类矩阵、验证门禁、降级规则
- `40-质量交付与留痕.md`：质量门槛、交付要求、evidence 要求
- `50-模板与索引.md`：日志模板、验证模板、支撑索引与适用结论
