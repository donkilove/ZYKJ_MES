# 工具参考与验证建议

## 1. 工具使用口径

- 优先使用当前会话实际可用、与任务匹配的宿主工具与 MCP 服务；小任务可直接使用最短可靠路径。
- 宿主命令类工具的“可用”判定，建议同时满足“命令可解析”与“最小真实调用可成功”；若仅能被 `Get-Command`、`where` 或 PATH 命中，但实际启动时报权限错误、进程创建失败或命中受限安装路径，仍按不可用处理。
- 当前 Codex 会话若已接入 Docker Toolkit / `MCP_DOCKER` 聚合网关，则该聚合 MCP 可作为常用工具来源之一。
- 盘点可用工具、MCP、技能时，区分四类对象：当前会话工具命名空间、MCP tools、MCP resources、MCP resource templates；其中 resources/templates 为空只表示未暴露资源列表，不代表 tools 不可用。
- `list_mcp_resources`、`list_mcp_resource_templates` 为空时，不直接判定 Docker Toolkit / `MCP_DOCKER` 不可用；需继续结合当前会话 server 识别结果、工具命名空间列表、`mcp_exec` 或等效真实调用结果判断。
- 若会话中已出现 `mcp__MCP_DOCKER__.*` 工具，视为 Docker MCP tools 已被宿主暴露；除非真实只读调用失败，否则不要在结论里写成“未看到 Docker MCP 工具”。
- `Sequential Thinking` 或等效分析工具：用于复杂、跨模块、高风险、规则变更或失败重试前复盘；小任务不强制。
- `update_plan` 或 `TodoWrite`：用于需要持续跟踪的多步骤任务；小任务不强制。
- 结构化检索工具：代码与文档结构定位、引用追踪。
- 项目结构记忆工具（如 `Graphify`）：用于项目级结构图谱、跨模块关系导航、入口定位与图谱问答；默认产物为 `graphify-out/GRAPH_REPORT.md`、`graphify-out/graph.json`、`graphify-out/graph.html`，默认忽略配置为 `.graphifyignore`。
- 文档检索工具：外部官方文档与规范补证。
- 浏览器自动化工具：页面联调、交互验证与页面证据采集。
- 数据库工具：数据库连接、查询与抽检。
- 接口契约工具：契约读取、OpenAPI 校验、接口补证。
- 代码托管工具：仓库状态、差异、远端协作与结果核对。
- 网页抓取工具：在前述能力不足时补充网页抓取。
- 文件系统工具：目录访问、本地文件读取与受控写入；若当前会话受更高优先级文件工具约束，则按更高优先级执行。
- 记忆类工具（如 `Memory`）：用于跨任务稳定偏好、稳定约定与长期上下文补充，但不能替代 `AGENTS.md` 与仓库文档。
- 宿主安全命令与宿主文件工具：作为本地补偿或降级手段，用于安全命令执行、本地文本检索、文件读写、测试与验证。
- 文本检索优先命令如 `rg`，若命中受限安装路径导致不可执行，可切换到 `Select-String` 等替代检索；若恢复可用，按需记录恢复动作与残余风险。

### 1.1 Graphify 项目结构记忆口径

- Graphify 是项目级结构记忆层，用于回答“模块如何连接、入口在哪里、跨模块关系如何追踪”等问题；Memory MCP 只保存跨会话稳定偏好与少量长期约定，二者不得互相替代。
- 若存在 `graphify-out/GRAPH_REPORT.md`，涉及架构理解、跨模块排查、入口定位、影响面评估或大范围检索前，可先读取其中的结构摘要，再回到源码和测试做最终验证。
- 若存在 `graphify-out/graph.json`，关系型问题可优先使用 `graphify query "<问题>"`、`graphify path "<A>" "<B>"`、`graphify explain "<节点>"` 获取结构线索；回答时要标明其为图谱线索，并用源码、测试或契约补证后才能作为结论。
- 只有形成长期复用价值的稳定项目事实时，才写入 `docs/project-memory/`；临时任务流水仍写入 `evidence/` 或 `progress.md`，不要为了进图谱复制噪音。
- 稳定项目记忆包括：长期有效的架构决策、模块关系、入口约定、运行环境约定、工具链约定、跨模块业务流程事实；未验证猜测、一次性调试记录、敏感信息和临时命令输出不得写入 `docs/project-memory/`。
- 代码变更后如本任务依赖图谱，建议运行 `graphify update .` 或等效 AST-only 刷新；若只改文档、图片或资料，可按需记录是否需要语义重抽取。
- 新增或修改 `docs/project-memory/*.md` 后，若本轮未执行 Graphify 语义更新，可在 `evidence/`、`progress.md` 或最终回复中标记“Graphify 文档语义待更新”；若执行语义更新，记录命令、范围、输出产物与成本或限制。
- 全量语义抽取涉及 LLM 成本，先确认 `.graphifyignore` 已排除敏感与噪音文件，并优先限定子目录；不要把密钥、令牌、内部敏感链接、环境文件、数据库 dump、日志、构建产物、依赖缓存、临时目录纳入图谱输入。
- 不要在本仓库直接运行会自动改写规则入口的 `graphify codex install` 或同类安装命令；如需接入，按“根入口薄规则 + 分册细则”的现有结构手工合并。
- Graphify 输出是辅助索引，不是事实源；当图谱与源码、测试、接口契约、`AGENTS.md` 或 `evidence/` 冲突时，以后者及真实验证为准，并记录图谱可能过期。

## 3. 验证检查清单

以下检查项用于复杂任务、提交前或复盘场景；小任务按需选用。

| 检查项 | 通过标准 |
| --- | --- |
| 范围清楚 | 已知道主要文件、接口、数据或页面边界 |
| 工具合适 | 使用了当前可用且匹配任务的工具 |
| 真实验证 | 有真实命令、页面结果、抓包、查询或扫描证据 |
| 降级透明 | 不可用工具、影响范围、替代动作与残余风险已说明 |
| 迁移明确 | 涉及迁移或兼容性取舍时已说明口径 |

## 4. 工具降级规则

- 所需工具缺失、未配置、不可达、权限不足，或受更高优先级指令约束无法使用：在影响结论可靠性时说明缺失工具、原因、替代工具与影响范围；复杂任务同步记录到 `evidence/` 或 `progress.md`。
- 宿主命令若出现“可解析但不可执行”，建议记录：命中路径、报错原文摘要、是否属于受限安装路径、替代命令，以及对检索或验证深度的影响。
- Docker Toolkit / `MCP_DOCKER` 已配置但当前会话不可用：先检查 `docker mcp client ls --global` 中 `codex` 是否为 `connected`；若未连接，优先执行连接修复并重启会话，再决定是否进入降级。
- Docker Toolkit / `MCP_DOCKER` 修复后仍需重新验证：至少补一条真实工具调用证据，例如 `mcp_exec` 调用 `list_allowed_directories`、`search` 或其他最小只读工具。
- `Sequential Thinking` 或等效分析工具不可用：复杂任务以书面拆解或计划工具代替；小任务无需额外记录。
- 结构化检索工具不可用：退回宿主安全文本检索、宿主文件工具与最小 patch 编辑，并按需记录无法做结构化或语义级定位的影响。
- 文档检索工具、网页抓取工具不可用：使用仓库既有文档、离线经验结论或宿主安全网页检索补偿，并标注时效与不确定性。
- Graphify 不可用或图谱缺失或过期：退回 `rg`、结构化检索、源码阅读与测试验证；若任务依赖项目结构记忆，说明图谱不可用原因、替代动作、影响范围和残余风险。
- 文件系统工具不可用：退回宿主文件工具，并明确当次目录访问范围与写权限边界；若当前会话本就受更高优先级文件工具约束，则继续按该约束执行。
- 记忆类工具不可用：退回 `AGENTS.md`、`evidence/` 与仓库文档补齐上下文；正式规则和交付结论不得由记忆类工具替代。
- 数据库工具、浏览器自动化工具、接口契约工具、代码托管工具等验证类工具不可用：退回宿主安全命令、宿主文件工具、本地包装命令或人工步骤脚本，但要说明未完成的真实验证边界。
