# 工具治理与验证门禁

## 1. 默认工具口径

- 默认强制优先使用当前会话实际可用的宿主工具与 MCP 服务；若所需工具缺失、未配置、不可达、权限不足，或受更高优先级指令约束无法使用，方可进入降级与补偿流程。
- 宿主命令类工具的“可用”判定，必须同时满足“命令可解析”与“最小真实调用可成功”；若仅能被 `Get-Command`、`where` 或 PATH 命中，但实际启动时报权限错误、进程创建失败或命中受限安装路径，仍按不可用处理。
- 当前 Codex 会话若已接入 Docker Toolkit / `MCP_DOCKER` 聚合网关，则该聚合 MCP 视为默认主链之一；其工具能力可覆盖 `sequentialthinking`、`filesystem`、`git`、`github`、`playwright`、`fetch`、`memory`、`duckduckgo/search` 等常用类别。
- Docker Toolkit / `MCP_DOCKER` 的可用性判定以“当前会话可识别 + 至少一次真实工具调用成功”为准，不能只看本地配置文件、客户端 UI 或 `connected` 文案。
- 盘点可用工具、MCP、技能时，必须区分四类对象：当前会话工具命名空间、MCP tools、MCP resources、MCP resource templates；其中 resources/templates 为空只表示未暴露资源列表，不代表 tools 不可用。
- `list_mcp_resources`、`list_mcp_resource_templates` 为空时，不得直接判定 Docker Toolkit / `MCP_DOCKER` 不可用；需继续结合当前会话 server 识别结果、工具命名空间列表、`mcp_exec` 或等效真实调用结果判断。
- 若会话中已出现 `mcp__MCP_DOCKER__.*` 工具，必须视为 Docker MCP tools 已被宿主暴露；除非真实只读调用失败，否则不得在前置说明或交付结论中写成“未看到 Docker MCP 工具”。
- `Sequential Thinking` 或等效分析工具：任务拆解、边界澄清、失败重试前复盘。
- `update_plan` 或 `TodoWrite`：维护步骤、状态与验收标准。
- 结构化检索工具（如 `ast-grep`、Serena 符号工具等）：代码与文档结构定位、引用追踪。
- 项目结构记忆工具（如 `Graphify`）：用于项目级结构图谱、跨模块关系导航、入口定位与图谱问答；默认产物为 `graphify-out/GRAPH_REPORT.md`、`graphify-out/graph.json`、`graphify-out/graph.html`，默认忽略配置为 `.graphifyignore`。
- 文档检索工具（如 `Context7`）：外部官方文档与规范补证。
- 浏览器自动化工具（如 `Playwright` / Browser MCP）：页面联调、交互验证与页面证据采集。
- 数据库工具（如 `database-server` / PostgreSQL MCP）：数据库连接、查询与抽检。
- 接口契约工具（如 OpenAPI 相关工具）：契约读取、OpenAPI 校验、接口补证。
- 代码托管工具（如 `Git` / `GitHub`）：仓库状态、差异、远端协作与结果核对。
- 网页抓取工具（如 `Fetch`）：在前述能力不足时补充网页抓取。
- 文件系统工具（如 `Filesystem`、宿主文件工具）：用于目录访问、本地文件读取与受控写入；若当前会话受更高优先级文件工具约束，则按更高优先级执行。
- 记忆类工具（如 `Memory`）：用于跨任务稳定偏好、模块关系与长期上下文补充，但不能替代 `AGENTS.md`、`evidence/` 与仓库文档。
- 宿主安全命令与宿主文件工具：仅作为本地补偿/降级手段，用于安全命令执行、本地文本检索、文件读写、测试与验证。
- 文本检索优先命令如 `rg`，若命中 `WindowsApps`、应用打包资源目录或其他受限安装路径导致不可执行，可先切换到 `Select-String` 等替代检索；若确认只是命中路径受限而非二进制损坏，允许通过用户可控目录中的等效副本、显式路径改道或 PATH 优先级调整恢复，但必须记录恢复动作与残余风险。

### 1.1 Graphify 项目结构记忆口径

- Graphify 是项目级结构记忆层，用于回答“模块如何连接、入口在哪里、跨模块关系如何追踪”等问题；Memory MCP 只保存跨会话稳定偏好与少量长期约定，二者不得互相替代。
- 若存在 `graphify-out/GRAPH_REPORT.md`，涉及架构理解、跨模块排查、入口定位、影响面评估或大范围检索前，先读取其中的 God Nodes、社区结构、Surprising Connections 与 Suggested Questions，再回到源码和测试做最终验证。
- 若存在 `graphify-out/graph.json`，关系型问题可优先使用 `graphify query "<问题>"`、`graphify path "<A>" "<B>"`、`graphify explain "<节点>"` 获取结构线索；回答必须标明其为图谱线索，并用源码、测试或契约补证后才能作为结论。
- 代码变更后如已存在图谱或本任务依赖图谱，应运行 `graphify update .` 或等效 AST-only 刷新；若只改文档、图片或论文资料，需明确是否需要语义重抽取，不能静默假装图谱已完整更新。
- 全量语义抽取涉及 LLM 语义处理和潜在成本，必须先确认 `.graphifyignore` 已排除敏感与噪音文件，并优先限定子目录；不得把密钥、令牌、内部敏感链接、环境文件、数据库 dump、日志、构建产物、依赖缓存、临时目录纳入图谱输入。
- 不得在本仓库直接运行会自动改写规则入口的 `graphify codex install` 或同类安装命令；如需接入，必须按“根入口薄规则 + 分册细则”的现有结构手工合并。
- Graphify 输出是辅助索引，不是事实源；当图谱与源码、测试、接口契约、`AGENTS.md` 或 `evidence/` 冲突时，以后者及真实验证为准，并记录图谱可能过期。

## 3. 验证门禁

| 门禁 | 检查项 | 通过标准 |
| --- | --- | --- |
| G1 | 任务分类已判定 | 已映射到主分类，且有触发依据 |
| G2 | 工具触发有依据 | 已记录默认触发、补充触发或降级原因 |
| G3 | 执行与验证分离 | 存在独立验证子 agent，或存在等效降级补偿 |
| G4 | 真实验证已执行 | 有真实命令、页面结果、抓包、查询或扫描证据 |
| G5 | evidence 已闭环 | 能串起“触发 -> 执行 -> 验证 -> 重试 -> 收口” |
| G6 | 降级已代偿 | 不可用工具、影响范围、替代动作与残余风险齐全 |
| G7 | 迁移口径明确 | 已声明迁移步骤，或写明“无迁移，直接替换” |

## 4. 工具降级规则

- 所需默认工具缺失、未配置、不可达、权限不足，或受更高优先级指令约束无法使用：先在前置说明写明缺失工具、原因、替代工具与影响范围，再在 `evidence/` 中同步记录降级与补偿。
- 宿主命令若出现“可解析但不可执行”，必须同时记录：命中路径、报错原文摘要、是否属于受限安装路径、替代命令，以及对检索/验证深度的影响。
- Docker Toolkit / `MCP_DOCKER` 已配置但当前会话不可用：先检查 `docker mcp client ls --global` 中 `codex` 是否为 `connected`；若未连接，优先执行连接修复并重启会话，再决定是否进入降级。
- Docker Toolkit / `MCP_DOCKER` 修复后仍需重新验证：至少补一条真实工具调用证据，例如 `mcp_exec` 调用 `list_allowed_directories`、`search` 或其他最小只读工具，不能仅以重启完成或连接状态代替验证。
- `Sequential Thinking` 或等效分析工具不可用：以书面拆解代替，并写入 `evidence/`。
- 结构化检索工具不可用：退回宿主安全文本检索、宿主文件工具与最小 patch 编辑，并记录无法做结构化/语义级定位的影响。
- 宿主命令类工具完成修复后，必须补至少一条同类真实命令复测；仅看到 PATH 顺序变化、`Get-Command` 命中变化或文件存在，不足以判定恢复成功。
- 文档检索工具、网页抓取工具不可用：使用仓库既有文档、离线经验结论或宿主安全网页检索补偿，并标注时效与不确定性。
- Graphify 不可用或图谱缺失/过期：退回 `rg`、结构化检索、源码阅读与测试验证；若任务依赖项目结构记忆，需在 `evidence/` 中记录图谱不可用原因、替代动作、影响范围和残余风险。
- 文件系统工具不可用：退回宿主文件工具，并明确当次目录访问范围与写权限边界；若当前会话本就受更高优先级文件工具约束，则继续按该约束执行。
- 记忆类工具不可用：退回 `AGENTS.md`、`evidence/` 与仓库文档补齐上下文；正式规则、过程留痕与交付结论不得由记忆类工具替代。
- 数据库工具、浏览器自动化工具、接口契约工具、代码托管工具等验证类工具不可用：退回宿主安全命令、宿主文件工具、本地包装命令或人工步骤脚本，但必须说明未完成的真实验证边界。
