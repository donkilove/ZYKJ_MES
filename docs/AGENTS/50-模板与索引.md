```markdown
## 5. 验证与交付
- 验证命令：<命令与结果>

- 残余风险：<若无可删>
- 是否满足任务目标：是 / 否
- 迁移说明：<仅涉及迁移或兼容性取舍时填写>
```

## 3. 当前有效索引

- 根 `AGENTS.md`：默认入口与硬约束装配点。
- `docs/AGENTS/00-导航与装配说明.md`：规则集装配与冲突处理。
- `docs/AGENTS/10-执行总则.md`：适用范围、核心原则与基础约束。
- `docs/AGENTS/20-指挥官模式与工作流.md`：轻量工作流；指挥官模式已废止。
- `docs/AGENTS/30-工具治理与验证门禁.md`：工具参考、验证建议与降级规则。
- `docs/AGENTS/40-质量交付与留痕.md`：交付门槛与触发式 evidence 要求。
- `.graphifyignore`：Graphify 项目结构记忆的输入边界与敏感文件排除清单。
- `docs/project-memory/`：Graphify 可摄取的稳定项目记忆入口，用于长期架构事实、模块关系、运行约定和工具约定。
- `graphify-out/GRAPH_REPORT.md`：Graphify 图谱报告；存在时作为架构理解、跨模块排查与入口定位的优先结构导航。
- `graphify-out/graph.json`：Graphify 原始图谱数据；存在时可用于 `graphify query/path/explain` 关系追踪。
- `evidence/指挥官任务日志模板.md`：历史模板副本，仅供追溯旧记录，不再作为现行规则。
- `evidence/指挥官工具化验证模板.md`：历史验证模板副本，仅供追溯旧记录，不再作为现行规则。

## 4. 历史收口说明

- 本轮保留根 `AGENTS.md` 与 `docs/AGENTS/*.md` 的单一规则集装配方式，但现行规则已改为轻量默认、重流程按需触发。
- 旧的“单文件唯一正文”口径已废止，改为“根入口 + 分册正文”的单一规则集装配方式。
- 旧索引 `docs/opencode_tooling_bundle.md` 与 `docs/host_tooling_bundle.md` 当前仓库未检出，不再作为有效支撑路径引用。

## 5. 适用结论

- 本规则集生效后，项目规则、轻量工作流、工具建议与留痕模板统一由根 `AGENTS.md` 与 `docs/AGENTS/*.md` 共同承载。
- 执行中若出现歧义，先按根 `AGENTS.md`、再按分册顺序判定；仍无法消解时，回到更高层级指令处理。
