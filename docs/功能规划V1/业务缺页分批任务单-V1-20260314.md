- 编写日期：2026-03-14

- 目标：把业务缺页实施清单进一步拆成可逐批执行、可逐批验收的任务单
- 使用方式：每次只让 Claude 执行一个批次；完成后先自查，再跑仓库要求的验证命令，再回填结果

---

## 1. 执行总原则

1. 先做“已实现但未纳管”的页面，再做真正新增页面。
2. 每一批都必须同时处理：
   - 前端 page catalog / 页签顺序
   - 后端 page catalog / 默认可见
   - 权限目录 / 权限层级
   - 必要的页面分发与访问控制
3. 不做与本批无关的重构。
4. 若发现某页面其实已部分纳管，只补缺口，不推倒重做。
5. 完成代码改动后，必须执行仓库要求的乱码检查与测试命令，并在结果中明确报告。

## 9. 最终建议

推荐你实际推动时按这个顺序下发：

1. 先让 Claude 做 `Batch 1`
2. 验证通过后再做 `Batch 2`
3. `Batch 3` 和 `Batch 4` 建议分开做，不要并行

原因很简单：

- Batch 1 最容易成功，能快速看见成果；
- Batch 2 会牵涉一点接口和页面模式调整；
- Batch 3、4 都是新功能页，边界和实现成本明显更高。
