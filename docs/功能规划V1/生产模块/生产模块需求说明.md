### 2.1 模块边界说明

- **以上主页面与子页面均纳入本期规划范围。**
- 生产模块负责生产订单创建、订单执行、首件、报工、并行模式、代班协同、生产统计、报废统计、维修回流与订单事件追溯。

- 产品主数据来源于产品模块。
- 工段、工序、工艺模板来源于工艺模块。
- 用户、角色、组织身份来源于用户模块。
- 消息中心页面和消息投递链路**不纳入本期**，统一以订单事件日志作为追溯落点。

## 4. 核心对象定义

### 4.1 生产订单

生产订单是生产模块的核心业务对象，用于承载某一产品在某一生产任务下的生产数量、工艺路线、进度状态和执行结果。

### 4.2 订单工序快照

订单工序快照用于固化订单创建时所使用的工段、工序、顺序和可见数量等信息。  
即使后续工艺模板变更，也不应影响既有订单的历史追溯。

### 4.3 子订单

子订单用于表达某道工序分配给具体操作员的可执行任务，用于支持可见量、代班、并行生产与个人执行视角。

### 4.4 生产记录

生产记录用于记录首件、报工等执行动作，是统计分析、异常追溯和维修闭环的数据来源。

### 4.5 代班授权

代班授权用于表达在某订单、某工序、某目标操作员场景下，允许其他人员代为执行首件或报工的授权关系。

### 4.6 并行实例

并行实例用于记录并行生产模式下，某子订单在某条并行链路中的激活状态、实例编号和执行顺序。

### 4.7 报废统计

报废统计用于汇总生产过程中的异常报废数量、报废原因和处理进度，是维修闭环的重要输入。

### 4.8 维修订单

维修订单用于承载生产异常后的送修、原因归类、报废判定、回流分配和维修完成结果。

### 4.9 订单事件日志

订单事件日志用于统一记录订单关键动作，包括建单、更新、并行配置、代班、首件、报工、维修完成等事件，用于审计和业务追溯。

## 5. 核心流程概述

本模块本期需支持以下核心流程：

1. 建单
2. 订单分配与可见量形成
3. 首件检验
4. 报工
5. 并行生产
6. 代班申请与执行
7. 异常送修
8. 报废统计
9. 维修完成与回流分配
10. 订单完工

### 5.1 主链路说明

- 生产管理员创建订单。
- 系统根据工艺模板或手工工艺步骤生成订单工序快照。
- 第一工序先进行首件，之后进入报工。
- 若开启并行模式，满足条件的下一工序可提前开始。
- 若执行人无法作业，可申请代班或由授权人员代为执行。
- 若报工中出现异常，可触发手工送修或后续维修闭环。
- 维修完成后，根据报废与回流分配结果更新相关工序可见量。
- 所有工序完成后，订单状态流转为“生产完成”。

## 6.4 列表业务规则

- 订单号全局唯一。
- 仅待生产状态订单允许编辑和删除。
- 已生产中的订单不允许重建工艺路线。
- 已完工订单不允许再次编辑或删除。
- 订单手工完工属于关键动作，必须写入订单事件日志。
- 创建、编辑、删除、手工完工、并行模式设置均需记录操作日志。

## 6.5 新建/编辑订单（子页面）

## 6.5.1 页面目标

用于录入订单基础信息、选择工艺模板或手工工艺步骤，并在必要时将本次路线保存为模板。

## 6.5.2 表单字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 订单号 | 输入框 | 是 | 全局唯一 |
| 产品 | 下拉选择 | 是 | 仅可选启用产品 |
| 数量 | 数字输入 | 是 | 必须大于 0 |
| 开始日期 | 日期选择 | 否 | 计划开始日期 |
| 交期 | 日期选择 | 否 | 计划交付日期 |
| 备注 | 输入框 | 否 | 订单补充说明 |
| 工艺模板 | 下拉选择 | 否 | 可直接引用产品已发布模板 |
| 手工工艺步骤 | 结构化列表 | 否 | 当不选模板时，支持按工段/工序手工配置 |
| 保存为模板 | 开关 | 否 | 是否将本次路线另存为模板 |
| 新模板名称 | 输入框 | 否 | 当“保存为模板”开启时可填写 |
| 设为默认模板 | 开关 | 否 | 当保存为模板时可选 |

### 6.5.3 业务规则

- 产品必须为启用状态。
- 数量必须大于 0。
- 订单必须存在可执行的工艺路线，来源可为模板或手工工艺步骤。
- 手工工艺步骤的步骤顺序不可重复。
- 手工工艺步骤中的工序必须与工段匹配。
- 若同时选择模板又手工调整步骤，系统应明确以哪种方式生效；默认建议“手工调整优先，且不再视为原模板完全匹配”。
- 若勾选“保存为模板”，需同步校验模板名称规则。

## 7.3 工单执行详情页面（子页面）

## 7.3.1 页面目标

用于在执行视角下查看工单完整上下文，并完成首件、报工、送修、代班申请等动作。

## 7.3.2 页面内容

至少包含以下区域：

- 当前视角说明区
- 执行动作区
- 基础信息区
- 工序页签
- 子订单页签
- 记录页签
- 事件页签

### 7.3.3 执行动作区

支持以下按钮：

- 首件
- 报工
- 手工送修建单
- 发起代班

### 7.3.4 首件业务规则

- 首件仅允许在当前工序满足开始条件时执行。
- 首件需输入当日校验码。
- 若当前为代班执行，必须带有效授权。
- 首件成功后，工序状态流转为“进行中”。

### 7.3.5 报工业务规则

- 报工数量必须大于 0。
- 报工数量不得超过当前可生产数量。
- 报工时允许录入异常现象和数量。
- 若当前为代班执行，必须带有效授权。
- 报工后需刷新可见量、工序状态、订单状态和下游工序可见性。

### 7.3.6 手工送修建单规则

- 送修数量来源于本次异常数量汇总。
- 手工送修建单后，应生成维修订单并写入订单事件日志。
- 送修仅针对当前工单、当前工序的异常数据生效。

## 13. 通用业务规则

## 13.1 状态定义

### 13.1.1 订单状态

| 状态 | 含义 |
| --- | --- |
| 待生产 | 订单已创建，尚未进入正式执行 |
| 生产中 | 至少一道工序已进入执行 |
| 生产完成 | 全部工序完成 |

### 13.1.2 工序状态

| 状态 | 含义 |
| --- | --- |
| 待生产 | 工序尚未执行 |
| 进行中 | 当前工序已首件并正在执行 |
| 部分完成 | 当前工序已完成部分数量，仍可继续执行 |
| 生产完成 | 当前工序全部完成 |

### 13.1.3 子订单状态

| 状态 | 含义 |
| --- | --- |
| 待执行 | 子订单已分配但尚未执行 |
| 执行中 | 子订单正在执行 |
| 已完成 | 子订单执行完成 |

### 13.1.4 代班状态

| 状态 | 含义 |
| --- | --- |
| 待审批 | 已发起申请，等待审批 |
| 已审批 | 审批通过，可执行代班动作 |
| 已拒绝 | 审批驳回，不可执行 |
| 已消耗 | 已用于执行首件或报工 |

### 13.1.5 维修状态

| 状态 | 含义 |
| --- | --- |
| 维修中 | 维修单已创建，尚未完成处理 |
| 已完成 | 已完成原因归类、报废、回流分配 |

### 13.1.6 报废处理进度

| 状态 | 含义 |
| --- | --- |
| 待处理 | 已形成报废统计，但尚未完成处理归档 |
| 已处理 | 已完成处理或已纳入后续闭环 |

---

## 13.2 可见性与数量规则

- 第一道工序默认可见数量为订单总量。
- 后续工序的可见数量取决于上一工序完成数量。
- 若启用并行模式，满足并行边的下一工序可在上一工序部分完成后提前可见。
- 当工序完成时，该工序角色默认不再继续看到该工序的可执行任务。
- 统计、详情和按钮态必须基于同一份可见性规则计算。

## 13.3 并行模式规则

- 并行模式仅允许在满足条件的订单上开启。
- 开启并行后，应明确配置参与并行的工序。
- 并行模式下的首件/报工按钮态必须与后端校验一致。
- 并行实例必须可追溯，不允许重复实例或越序执行。

## 13.4 代班与代理规则

- 代班仅允许在订单未完工、工序未完成的情况下发起。
- 代班执行首件和报工必须留痕到有效执行人和实际操作人。
- 生产管理员可使用代理视角查看指定操作员工单。
- 代理视角只影响查看和代操作边界，不改变订单归属关系。

## 13.5 报废与维修规则

- 报工异常数据应能沉淀为维修输入或报废统计输入。
- 维修完成必须满足数量守恒：报废 + 回流 = 送修数量。
- 回流目标工序必须合法，且不能越过当前工序之后的下游工序。
- 报废统计口径必须与维修结果口径一致。

## 13.6 日志与追溯规则

- 建单、编辑、删除、手工完工、并行配置、代班、首件、报工、送修、维修完成等关键动作均需写入订单事件日志。
- 日志需统一保留事件类型、事件标题、事件详情、操作者、时间和结构化负载。
- 页面应支持通过详情或关联入口查看事件记录。

## 16. 评审确认结论

以下内容建议在本版评审中固化：

1. 生产模块本期以“订单主链路 + 协同控制 + 数据统计 + 异常闭环”为整体边界。
2. 订单管理、订单查询、代班记录、生产数据、报废统计、维修订单 6 个主页面全部纳入本期。
3. 订单新建/编辑、订单详情、工单执行详情作为正式子页面纳入需求，不再视为临时跳转页。
4. 并行实例追踪页面和维修订单详情页面作为建议新增页面纳入本期规划。
5. 代班能力默认按“审批流”定义；若项目采用“发起即生效”，必须作为业务裁剪单独说明。
6. 生产数据页面建议补齐工序统计与人员统计子视图，避免后端已有接口长期无前端承接。
7. 生产后段闭环需明确满足“报废统计 -> 维修完成 -> 回流分配 -> 状态更新”的完整关系。
