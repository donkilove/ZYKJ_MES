# 工艺模块需求说明（最终可评审稿）

## 1. 文档信息

- 模块名称：工艺模块
- 文档版本：V1.0
- 文档状态：可评审
- 目标：完成工段、工序、系统母版、产品工艺模板、模板版本发布、引用分析与工艺看板等基础能力建设，满足 MES 场景下工艺主数据维护、跨模块引用和工艺变更追溯需求。

---

## 2. 本期范围

本模块本期包含以下页面：

1. 工段/工序管理页面
2. 工艺模板配置页面
3. 工艺引用分析页面
4. 工艺看板页面

### 2.1 模块边界说明

- **以上 1～4 页面均纳入本期上线范围。**
- 工艺模块负责维护工段、工序、系统母版、产品工艺模板、模板版本、模板发布和模板引用分析。
- 工艺模块中涉及产品、生产订单、设备、用户、质量标准等内容时，默认作为**关联信息/引用信息**使用，不在本模块内做主数据归口维护。
- 用户模块中涉及的“工段”数据来源于工艺模块，用户模块仅做引用，不提供工段新增、编辑、删除能力。
- 产品模块中涉及的“工艺模板/工艺路线”由工艺模块维护，产品模块仅做绑定与展示。
- 生产模块使用已发布的工艺模板生成订单工艺快照，工艺模块不直接负责订单执行。

### 2.2 本期建议新增页面与功能

在现有系统能力基础上，本需求说明**额外纳入以下新增建议**：

- 新增**工艺引用分析页面**，用于在停用、删除、发布、回滚前查看工段/工序/模板的跨模块引用关系。
- 在工艺模板配置页面补充**模板复制/套版**功能，降低相似产品建模成本。
- 在工艺模板配置页面补充**模板归档/取消归档**功能，形成模板生命周期闭环。
- 在工艺模板配置页面补充**发布记录查看**能力，增强模板版本变更追溯。
- 补充标准化的**工段/工序轻量查询接口**，供用户、设备、生产等模块复用。

## 4. 核心对象定义

### 4.1 工段

工段是工艺模块的一级基础对象，用于表达生产流程中的大阶段。  
工段既服务于工艺模板编排，也为用户模块中的操作员归属提供引用数据。

### 4.2 工序

工序是归属于某工段的二级基础对象，用于表达实际生产中的具体操作步骤。  
工序必须绑定工段，且在模板步骤中按顺序编排。

### 4.3 系统母版

系统母版是平台级默认工艺模板，用于在新建产品或初始化模板时快速生成基础工艺路径。  
系统母版不直接参与生产执行，但可作为产品模板的来源。

### 4.4 产品工艺模板

产品工艺模板是面向具体产品维护的工艺路线配置对象。  
模板承载某产品可用于生产的工段/工序顺序，并支持版本化管理、发布、回滚与影响分析。

### 4.5 模板版本

模板版本用于记录模板在不同时点的配置快照。  
版本是模板变更审计、发布控制、回滚和生产引用追溯的核心对象。

### 4.6 引用关系

引用关系用于描述工段、工序、系统母版、产品模板在其他模块中的使用情况，例如：

- 用户模块引用工段
- 产品模块绑定工艺模板
- 生产订单持有模板版本与工序快照
- 历史订单、设备规则、统计口径等引用工序或工段

用于统一管理工段与工序基础资料，支持查询、新建、编辑、启停、删除和引用校验，为用户、产品、生产等模块提供稳定的工艺基础数据来源。

## 6. 工艺模板配置页面

## 6.1 页面目标

用于维护系统母版与产品工艺模板，支持模板创建、编辑、复制、发布、回滚、归档、导入导出和版本追溯，确保生产工艺配置可控、可复用、可审计。

## 6.2 页面组成

页面由以下 4 个区域组成：

1. 系统母版区
2. 产品模板筛选区
3. 模板列表区
4. 模板详情/步骤配置区

### 6.2.1 页面范围说明

- 系统母版不单独拆独立页面，纳入工艺模板配置页面统一管理。
- 模板版本对比、发布记录、回滚结果可通过弹窗、抽屉或侧边页呈现，不强制拆独立页面。
- 本期建议在本页面内增加“复制模板”“归档模板”“发布记录查看”等增强能力。

### 6.2.2 搜索/筛选功能

支持以下筛选条件：

- 产品名称搜索
- 产品分类筛选
- 模板名称搜索

- 是否默认模板筛选
- 模板状态筛选（启用 / 停用）
- 更新时间范围筛选

### 6.2.3 顶部操作区

支持以下按钮：

- 新建模板
- 从系统母版套版
- 从已有模板复制
- 导出模板
- 导入模板

## 6.3 系统母版管理

### 6.3.1 页面目标

用于维护平台默认工艺流程，作为新建产品模板和新建产品时的基础套版来源。

### 6.3.2 展示内容

至少展示以下内容：

- 当前系统母版版本号
- 母版步骤列表
- 最近更新时间
- 最近更新人

### 6.3.3 支持操作

- 新建系统母版
- 编辑系统母版
- 查看历史版本
- 复制为产品模板

### 6.3.4 业务规则

- 系统母版至少包含 1 条步骤。
- 步骤顺序不可重复。
- 系统母版步骤只能引用启用状态的工段与工序。
- 修改系统母版不会自动覆盖已存在的产品模板。
- 新建产品时，若系统启用了“自动套版”策略，则默认从当前系统母版生成产品默认模板。

## 6.5 新建/编辑模板

### 6.5.1 表单字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 所属产品 | 下拉选择 | 是 | 选择模板归属产品 |
| 模板名称 | 输入框 | 是 | 同一产品下唯一 |
| 是否默认模板 | 单选 | 否 | 默认否 |
| 生命周期 | 单选 | 否 | 默认草稿 |
| 模板备注 | 输入框 | 否 | 便于记录用途 |

### 6.5.2 步骤配置字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 步骤顺序 | 数字输入 | 是 | 从 1 开始递增 |
| 工段 | 下拉选择 | 是 | 仅可选启用工段 |
| 工序 | 下拉选择 | 是 | 仅可选所选工段下的启用工序 |
| 标准工时 | 数字输入 | 否 | 建议新增字段，用于后续统计与优化 |
| 关键工序标识 | 开关 | 否 | 建议新增字段，用于异常关注 |
| 步骤说明 | 输入框 | 否 | 便于维护说明 |

> 说明：  
> `标准工时`、`关键工序标识`、`步骤说明` 为本需求说明建议新增字段；若本期无法一次性落地，可列入 P1，但页面结构应预留扩展能力。

### 6.5.3 业务规则

- 模板至少包含 1 条步骤。
- 步骤顺序不可重复。
- 同一模板内同一顺序仅允许一个步骤。
- 模板步骤中的工序必须与所选工段匹配。
- 仅启用状态的工段和工序可被新增到模板中。
- 同一产品下模板名称必须唯一。
- 同一产品同一时刻仅允许一个默认模板处于启用状态。
- 草稿模板允许持续编辑。
- 已发布模板不允许直接覆盖编辑；如需变更，应生成新版本或回滚后重新发布。

## 6.6 模板复制/套版功能

### 6.6.1 功能目标

减少相似产品、相似工艺场景下的重复建模成本。

### 6.6.2 支持方式

- 从系统母版复制为产品模板
- 从同产品已有模板复制出新草稿版本
- 跨产品复制模板

### 6.6.3 业务规则

- 复制后默认生成草稿版本。
- 复制后允许修改模板名称。
- 复制后需重新校验工段、工序仍为启用状态。
- 复制产生的新模板需记录来源模板或来源母版。

---

## 6.7 模板发布功能

### 6.7.1 发布前检查

发布前至少校验：

- 模板状态为启用
- 模板生命周期为草稿或可发布状态
- 模板至少包含 1 条有效步骤
- 步骤顺序连续且无重复
- 模板所用工段、工序均为启用状态

### 6.7.2 发布能力

支持以下能力：

- 查看待发布模板
- 提交发布
- 查看影响分析结果
- 选择是否同步未完工订单
- 填写发布说明
- 查看发布结果

### 6.7.3 发布规则

- 发布成功后生成新版本记录。
- 发布成功后，模板生命周期更新为“已发布”。
- 若同一产品下已有默认已发布模板，新发布模板为默认模板时，原默认模板自动失去默认标识。
- 发布前若发现模板引用的工段或工序已停用，应禁止发布。
- 若用户确认发布时系统发现模板版本已被他人修改，应终止本次发布并提示刷新后重试。

---

## 6.8 影响分析、回滚与归档

### 6.8.1 影响分析功能

发布、回滚、停用、删除前支持查看影响分析，至少识别以下对象：

- 未完工生产订单
- 已下发但未开工订单
- 用户模块对工段的引用
- 其他模板对工段/工序的复用关系

### 6.8.2 回滚功能

支持以下能力：

- 查看历史版本
- 选择目标版本回滚
- 查看版本差异
- 选择是否同步未完工订单
- 查看回滚结果

### 6.8.3 归档功能

支持以下能力：

- 归档模板
- 取消归档
- 查询已归档模板

### 6.8.4 业务规则

- 已归档模板默认不可继续用于新建生产订单。
- 已归档模板保留历史追溯，不允许直接删除已被业务引用的版本。
- 回滚本质上生成新的可发布草稿或新版本，不建议直接篡改历史版本数据。
- 所有发布、回滚、归档操作均需写入操作日志。

---

## 6.9 模板导入/导出功能

### 6.9.1 导出能力

支持以下导出内容：

- 单个模板导出
- 当前筛选结果导出
- 模板版本导出

### 6.9.2 导入能力

支持以下导入方式：

- JSON 导入
- 结构化 Excel 导入（建议 P1）

### 6.9.3 导入规则

- 导入前需校验工段、工序是否存在且处于启用状态。
- 导入失败时需返回逐条错误信息。
- 导入成功后默认生成草稿模板。
- 不允许导入覆盖已发布版本的历史记录。

## 7. 工艺引用分析页面

## 7.1 页面目标

用于在停用、删除、发布、回滚前查看工段、工序、系统母版、产品模板在其他模块中的引用情况，避免误操作造成跨模块数据断链。

## 7.2 页面功能

### 7.2.1 查询维度

支持以下维度：

- 按工段查询引用
- 按工序查询引用
- 按模板查询引用
- 按产品查询模板引用

### 7.2.2 展示范围

至少展示以下引用来源：

- 用户模块中的用户工段绑定
- 产品模块中的产品模板绑定关系
- 生产模块中的订单模板版本、工段/工序快照
- 其他工艺模板中的步骤引用
- 设备、统计或规则配置中的工段/工序引用（若存在）

### 7.2.3 表头字段

| 表头 | 说明 |
| --- | --- |
| 引用对象类型 | 用户 / 产品 / 模板 / 订单 / 设备等 |
| 引用对象编码 | 被引用对象编码 |
| 引用对象名称 | 被引用对象名称 |
| 引用状态 | 正在使用 / 历史引用 / 可同步 / 不可同步 |
| 风险说明 | 如“存在未完工订单，不可删除” |
| 跳转操作 | 查看详情或跳转到来源模块 |

### 7.2.4 业务规则

- 查看引用分析不改变业务数据。
- 删除、停用、发布、回滚等关键操作前，系统应支持从本页面或弹窗中查看引用分析结果。
- 若存在阻断级引用，应明确提示禁止执行对应操作。

## 8. 工艺看板页面

## 8.1 页面目标

用于查看按产品和工序维度汇总的工艺过程指标，帮助识别节拍异常、产能瓶颈和工艺优化机会。

## 8.2 页面功能

### 8.2.1 查询条件

支持以下筛选条件：

- 产品选择
- 时间范围
- 工段筛选（建议新增）
- 工序筛选（建议新增）

### 8.2.2 展示内容

至少展示以下内容：

- 各工序工时柱状图
- 各工序产能折线图
- 样本数量
- 统计时间范围

### 8.2.3 建议增强能力

建议增加以下功能：

- 工段维度切换
- 工序趋势对比
- 工时异常高亮
- 数据导出

### 8.2.4 业务规则

- 看板数据默认基于已完成工序记录统计。
- 无样本数据时展示空状态，不报错。
- 统计口径需在页面内明确说明。
- 看板仅用于分析展示，不直接修改模板数据。

## 9. 通用业务规则

## 9.1 编码规则

- 工段编码全局唯一。
- 工序编码全局唯一。
- 工序编码建议采用 `工段编码-两位序号` 格式。
- 模板名称在同一产品下唯一。

## 9.2 启停规则

- 停用工段后，默认不可继续用于用户绑定和模板新增。
- 停用工序后，默认不可继续用于模板新增。
- 已被历史订单引用的工段/工序允许查看，但不允许无引用分析直接删除。
- 已归档模板默认不可继续用于新建订单。

## 9.3 删除规则

- 工段、工序、模板删除前需二次确认。
- 已被用户、模板、订单、历史版本引用的工段/工序不做物理删除。
- 已被业务引用的模板或模板版本不做物理删除，建议采用停用或归档方式处理。

## 9.4 版本与发布规则

- 模板发布必须形成版本记录。
- 发布、回滚、复制、归档均需记录操作日志。
- 回滚应保留历史版本链路，不覆盖既有历史版本。
- 系统中同一产品允许存在多个模板，但应限制默认模板唯一。

## 9.5 跨模块联动规则

- 用户模块中的工段选择数据实时来源于工艺模块。
- 产品模块中的工艺模板引用数据来源于工艺模块。
- 生产模块创建订单时应使用已发布模板，并固化工段/工序快照，避免后续模板变更影响历史订单追溯。
- 当模板发布或回滚时，应支持识别对未完工订单的影响范围。

## 12. 评审确认结论

以下内容建议在本版评审中固化：

1. 工段管理明确归属工艺模块。
2. 工艺模块中的基础主数据以“工段 + 工序”为核心，不再拆成多个零散子模块。
3. 系统母版不单独拆页面，纳入工艺模板配置页面统一维护。
4. 产品工艺模板必须支持版本管理、发布、回滚和影响分析。
5. 新增工艺引用分析页面，作为跨模块安全变更的前置能力。
6. 模板复制/套版、模板归档/取消归档作为本期建议增强功能纳入范围。
7. 工艺看板继续保留，并补充时间范围、工段、工序等筛选能力。
