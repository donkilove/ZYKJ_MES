## 各页面详细检查

### 1. 每日首件（P0）— ✅ 基本完成

**后端实现：**
- `quality.py`：首件检验记录列表（分页/按产品/工序/结果/时间范围筛选）、首件详情、首件导出、首件处置提交
- 后端模型：`first_article_record.py`、`first_article_disposition.py`、`first_article_disposition_history.py`

**前端实现：**
- `quality_page.dart`：品质模块入口（含 6 个 Tab）
- `daily_first_article_page.dart`：每日首件主页面
- `first_article_disposition_page.dart`：首件详情与处置弹窗

**缺失功能点：**
- 处置历史日志查询接口未暴露（后端有 `FirstArticleDispositionHistory` 模型但无查询 API）
- 前端未展示处置历史记录列表

### 2. 品质数据（P0）— ✅ 完成

**后端实现：**
- `quality.py`：品质概览统计（`/stats/overview`）、工序品质统计（`/stats/processes`）、操作员品质统计（`/stats/operators`）、产品品质统计（`/stats/products`）、品质统计导出

**前端实现：**
- `quality_data_page.dart`：品质数据查询、总览卡片区 + 按工序/按人员/按产品三个 Tab + 导出 + 全部筛选条件

**缺失功能点：无**

### 3. 报废统计（P1）— ✅ 基本完成（复用生产模块）

**后端实现：**
- `production.py`：报废统计查询、详情（含关联维修单）、导出

**前端实现：**
- `production_scrap_statistics_page.dart`：报废数据统计（品质模块 Tab 中嵌入）

**缺失功能点：**
- 缺少独立的产品/工序下拉筛选（当前仅通过关键字搜索覆盖）
- 报废详情弹窗未展示"关联维修单"信息（后端已返回 `related_repair_orders`，前端未使用）

### 4. 返修工单（P1）— ✅ 基本完成（复用生产模块）

**后端实现：**
- `production.py`：返修工单列表、创建、完成、导出、详情（含缺陷现象、维修原因、回流分配、事件日志）

**前端实现：**
- `production_repair_orders_page.dart`：返修工单管理（品质模块 Tab 中嵌入）

**缺失功能点：**
- 列表字段中缺少"已修复数量"和"补投产"列
- 完成维修表单中的"回流分配"仅支持单一目标工序，需求要求支持多条回流分配

### 5. 不良分析（P1）— ✅ 基本完成

**后端实现：**
- `quality.py`：不良分析查询（`/defect-analysis`，含 Top 缺陷现象、按工序/按产品分布）、不良分析导出

**前端实现：**
- `quality_defect_analysis_page.dart`：不良品分析（不良总数卡片、Top 缺陷柱状图、按工序/按产品分布表格）

**缺失功能点：**
- 缺少"操作员"和"不良类型"筛选条件
- 帕累托图（累计占比曲线）未实现，当前为普通柱状图
- 导出按钮 UI 未实现（后端接口已有）

### 6. 品质趋势（P1）— ✅ 基本完成

**后端实现：**
- `quality.py`：品质趋势查询（`/trend`，按日统计首件通过/不通过/报废/维修数量）

**前端实现：**
- `quality_trend_page.dart`：品质趋势折线图（通过/不通过/报废三条线）+ 数据表格 + 筛选条件

**缺失功能点：**
- 缺少"首件通过率趋势线"（当前图表展示绝对数量而非通过率曲线）
- 缺少导出功能
- 缺少"产品质量对比图"（需求建议新增）

## 待改进项

1. **首件处置历史**：需新增后端查询接口并在前端展示处置历史日志
2. **报废详情关联维修单**：前端需渲染后端已返回的 `related_repair_orders` 数据
3. **不良分析增强**：补充操作员/不良类型筛选、帕累托累计占比线、导出按钮
4. **返修工单增强**：列表补充"已修复数量"列、回流分配支持多条
5. **报废统计筛选**：增加独立的产品/工序下拉筛选
6. **品质趋势增强**：增加首件通过率趋势线、导出功能、产品质量对比图
