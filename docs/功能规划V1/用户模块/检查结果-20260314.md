## 各页面详细检查

### 1. 用户管理（P0）— ✅ 完成

**后端实现：**
- `users.py`：用户列表（分页/搜索/按角色/工段/在线状态/账号状态筛选）、新增、编辑、启用、禁用、重置密码、逻辑删除、CSV/Excel 导出
- `user_service.py`：`list_users`、`create_user`、`update_user`、`set_user_active`、`delete_user`、`reset_user_password`，含角色分配、工段/工序绑定

**前端实现：**
- `user_management_page.dart`：用户列表、搜索、角色/工段/在线状态/账号状态筛选、新增/编辑对话框、禁用/启用、重置密码、删除、导出、5秒自动刷新在线状态

## 通用规则检查（第12节）

| 规则 | 状态 | 说明 |
|------|------|------|
| 用户名唯一 | ✅ | 创建/编辑/审批均校验 |
| 用户名2-10位 | ✅ | Schema 层 `min_length=2, max_length=10` |

| 不支持与已有用户相同密码 | ✅ | 后端 `validate_password` 遍历比对 |
| 在线状态规则 | ✅ | `online_status_service.py` TTL 机制 |
| 操作员必须绑定工段 | ✅ | 创建/编辑/审批均校验 |
| 非操作员不可绑定工段 | ✅ | 角色变更时自动清空 |
| 逻辑删除 | ✅ | `is_deleted` + `deleted_at` |
| 至少保留一个系统管理员 | ✅ | 权限配置保存时校验 |
