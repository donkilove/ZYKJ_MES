# 用户模块需求说明（最终可评审稿）

## 1. 文档信息

- 模块名称：用户模块
- 文档版本：V1.0
- 文档状态：可评审
- 目标：完成用户、注册审批、角色、权限、个人中心、登录日志/在线会话及审计相关基础能力建设，满足系统后台用户管理与权限控制需求。

---

## 2. 本期范围

本模块本期包含以下页面：

1. 用户管理页面
2. 注册审批页面
3. 角色管理页面
4. 功能权限配置页面
5. 操作审计日志页面
6. 个人中心/账号设置页面
7. 登录日志/在线会话页面

### 2.1 模块边界说明

- **以上 1～7 页面均纳入本期上线范围。**
- **工段管理页面不属于用户模块，归属工艺模块。**
- 用户模块中涉及“工段”的地方，仅作为**引用字段/下拉选项**使用，数据来源于工艺模块。
- 用户模块不提供工段新增、编辑、删除能力。

---

## 4. 角色定义

### 4.1 系统内置角色

系统默认内置以下角色：

- 系统管理员
- 生产管理员
- 品质管理员
- 操作员
- 维修员

### 4.2 角色规则

- 系统内置角色默认存在。
- 系统内置角色不允许删除。
- 系统内置角色不允许修改角色名称。
- 系统内置角色允许配置权限。
- 系统支持新增自定义角色。
- 已绑定用户的角色不允许直接删除，建议先解除用户绑定或停用角色。

---

## 5. 用户管理页面

## 5.1 页面目标

用于统一管理系统内全部用户，支持搜索、筛选、查看、新建、编辑、停用、删除、重置密码、导出等操作。

## 5.2 页面功能

### 5.2.1 搜索/筛选功能

支持以下筛选条件：

- 用户名搜索（模糊搜索）
- 角色筛选
- 工段筛选
- 在线状态筛选
- 账号状态筛选（启用/停用）

### 5.2.2 顶部操作区

支持以下按钮：

- 新建用户
- 进入角色管理页面
- 导出用户列表（支持 CSV / Excel）

> 说明：  
> 本期支持导出，不支持导入。

---

## 5.3 新建用户

### 5.3.1 表单字段

| 字段   | 类型   | 必填  | 说明                 |
| ---- | ---- | --- | ------------------ |
| 用户名  | 输入框  | 是   | 全局唯一               |
| 用户密码 | 密码框  | 是   | 新建时填写              |
| 用户角色 | 单选下拉 | 是   | 单选                 |
| 用户工段 | 单选下拉 | 否   | 仅操作员角色可选；数据来源于工艺模块 |
| 账号状态 | 单选   | 否   | 默认启用               |
| 备注   | 输入框  | 否   | 便于管理               |

### 5.3.2 业务规则

- 用户名必须唯一。
- 用户角色为“操作员”时，用户工段为必填。
- 用户角色为非“操作员”时，用户工段字段不可编辑，展示为空或 `/`。
- 用户工段下拉数据来源于**工艺模块的工段基础数据**。
- 工段数据采用**实时查询**方式获取。
- 新建成功后，用户进入用户列表，默认按数据库 `id` 升序展示。

---

## 5.4 用户列表展示

### 5.4.1 列表要求

- 列表需带表头。
- 列表区域支持滚动。
- 按数据库 `id` 字段排序展示，默认升序。
- 支持空数据占位展示。

### 5.4.2 表头字段

| 表头   | 说明                   |
| ---- | -------------------- |
| 用户名  | 展示用户名                |
| 角色   | 展示当前用户角色             |
| 工段   | 未分配、无法分配角色时留空或显示 `/` |
| 在线状态 | 在线显示绿色；离线显示灰色        |
| 账号状态 | 展示启用/停用              |
| 创建时间 | 展示用户创建时间             |
| 操作   | 下拉操作菜单               |

### 5.4.3 操作菜单

下拉菜单包含：

- 编辑
- 停用
- 启用
- 重置密码
- 删除

---

## 5.5 编辑用户

### 5.5.1 可编辑字段

- 用户名（仅系统管理员可修改）
- 用户角色
- 用户工段
- 账号状态
- 备注

### 5.5.2 编辑规则

- 仅系统管理员可修改用户名，普通用户不可修改用户名。
- 若角色修改为“操作员”，则必须选择工段。
- 若角色由“操作员”改为其他角色，则原工段自动清空。
- 用户工段数据来源于工艺模块，不在本模块内维护。
- 若用户已有关联业务数据，禁止物理删除，仅允许停用。

---

## 5.6 删除规则

- 用户删除采用**逻辑删除/停用**方式，不做物理删除。
- 删除前需二次确认。
- 已停用用户默认不可登录系统。
- 逻辑删除/停用操作需记录审计日志。

---

## 5.7 重置密码规则

- 管理员可对用户执行重置密码操作。
- 重置密码后，用户下次登录可正常使用新密码。
- 首次登录强制修改密码：本期支持。
- 重置密码操作需记录审计日志。

---

## 6. 注册审批页面

## 6.1 页面目标

用于处理待审批注册申请，支持审批通过、驳回。

## 6.2 页面功能

### 6.2.1 列表要求

- 展示全部待审批注册用户。
- 列表需带表头。
- 列表区域支持滚动。
- 按数据库 `id` 字段排序展示，默认升序。

### 6.2.2 表头字段

| 表头   | 说明              |
| ---- | --------------- |
| 用户名  | 注册申请用户名         |
| 密码   | 按当前确认方案展示       |
| 申请时间 | 注册申请时间          |
| 申请状态 | 待审批 / 已通过 / 已驳回 |
| 操作   | 下拉操作菜单          |

> 说明：  
> 本期按当前确认方案，审批页允许展示密码。

---

## 6.3 操作菜单

下拉菜单包含：

- 通过
- 驳回

---

## 6.4 通过操作

### 6.4.1 通过表单字段

| 字段   | 类型   | 必填  | 说明                 |
| ---- | ---- | --- | ------------------ |
| 用户名  | 输入框  | 是   | 默认带出申请用户名          |
| 用户密码 | 密码框  | 是   | 默认带出或重新设置          |
| 用户角色 | 单选下拉 | 是   | 单选                 |
| 用户工段 | 单选下拉 | 否   | 仅操作员角色可选；数据来源于工艺模块 |

### 6.4.2 通过规则

- 审批通过时需校验用户名唯一性。
- 用户角色为“操作员”时，必须选择工段。
- 用户工段下拉数据来源于**工艺模块的工段基础数据**。
- 审批通过后，注册申请状态更新为“已通过”，并生成正式用户账号。
- 审批通过操作需记录审计日志。

---

## 6.5 驳回操作

### 6.5.1 交互要求

- 点击“驳回”后弹出确认弹窗。
- 本期支持填写驳回原因。

### 6.5.2 驳回规则

- 驳回后申请状态更新为“已驳回”。
- 保留驳回记录，不直接删除申请数据。
- 驳回操作需记录审计日志。

---

## 7.1 页面目标

用于维护角色信息，管理角色的启用状态、说明及基础属性。

## 7.2 页面功能

| 表头    | 说明         |
| ----- | ---------- |

| 角色说明  | 角色说明       |

本期权限粒度包含：

- 页面查看
- 新建
- 编辑
- 删除
- 审批
- 配置

> 说明：  
> 本期**不做数据范围权限**，即不区分“仅本人 / 仅本工段 / 全部”。

---

## 8.5 特殊规则

### 8.5.1 系统管理员规则

- 系统管理员**不再固定全权限**。
- 系统管理员**固定拥有“功能权限配置页面”访问权限**。
- “功能权限配置页面”权限仅系统管理员角色必定拥有编辑权限。
- “功能权限配置页面”权限**不出现在能力包中**。
- 非系统管理员角色不可拥有该权限。

### 8.5.2 保存规则

- 修改权限后需点击“保存”生效。
- 保存成功后写入审计日志。
- 系统中必须至少保留一个可进入权限配置页面的系统管理员账号，避免系统失控。

## 10. 个人中心/账号设置页面

## 10.1 页面目标

用于当前登录用户查看本人账号信息、维护个人账号安全，并查看本人当前登录概况。  
本页面仅提供“本人信息查看”和“本人密码修改”能力，不提供角色、工段、权限、账号状态等组织属性的自助修改能力。

---

## 10.2 页面组成

页面由以下 3 个区域组成：

1. 基本信息区
2. 安全设置区
3. 会话概览区

---

## 10.3 页面功能

### 10.3.1 基本信息查看

支持查看以下本人信息：

- 当前登录账号用户名
- 当前登录账号角色
- 当前登录账号所属工段
- 当前登录账号状态
- 账号创建时间
- 最近一次登录时间
- 最近一次登录 IP
- 密码最近修改时间

> 说明：  
> 当前工段仅展示用户已绑定工段信息，工段数据来源于工艺模块。  
> 非操作员角色未绑定工段时，展示为 `/`。

### 10.3.2 本人密码修改

支持当前登录用户修改本人密码，包含以下能力：

- 输入原密码
- 输入新密码
- 输入确认新密码
- 提交修改
- 修改成功后提示重新登录
- 修改失败时提示失败原因

### 10.3.3 当前会话概览

支持查看当前登录会话的概要信息：

- 当前登录时间
- 最近活跃时间
- 当前会话状态
- 当前会话剩余有效时长
- 退出当前登录

---

| 新密码      | 安全设置区 | 是     | 修改密码时填写        |

---

## 10.5 页面交互说明

### 10.5.1 基本信息区

- 以只读表单或信息卡片形式展示。
- 用户不可在本页面修改用户名、角色、工段、账号状态。
- 当用户为非操作员角色时，工段展示为 `/`。
- 账号状态与系统中统一状态文案保持一致。

### 10.5.2 安全设置区

- 原密码、新密码、确认新密码均为密码输入框。
- 新密码输入时，页面应展示密码规则提示。
- 点击“保存”前应进行前端校验：
  - 原密码不能为空
  - 新密码不能为空
  - 确认新密码不能为空
  - 新密码与确认新密码必须一致
  - 新密码不能与原密码相同
- 提交成功后弹出成功提示，并强制当前用户重新登录。
- 提交失败时展示明确失败原因，如：
  - 原密码错误
  - 新密码不符合规则
  - 两次输入的新密码不一致

### 10.5.3 会话概览区

- 展示当前登录会话摘要信息。
- 若会话接近超时时间，可进行醒目提示。
- 支持“退出当前登录”按钮，点击后当前登录态立即失效并返回登录页。

---

## 10.6 业务规则

- 所有已登录用户均可访问个人中心/账号设置页面。
- 用户仅可查看和维护**本人**信息，不可查看其他用户个人中心内容。
- 用户不可在个人中心修改自己的角色、工段、账号状态、权限信息。
- 用户名不可在个人中心修改。
- 修改密码时必须校验原密码正确性。
- 新密码必须符合系统密码规则。
- 新密码不得与原密码相同。
- 新密码与确认新密码必须一致。
- 修改密码成功后，当前登录态立即失效，用户必须重新登录。
- 若会话已过期，页面操作应提示登录失效并跳转登录页。

---

## 10.7 安全规则

- 密码输入框默认掩码显示。
- 修改密码操作需写入操作审计日志。
- 不展示用户明文历史密码。
- 页面不得缓存用户原密码、新密码、确认新密码明文。

---

## 11. 登录日志/在线会话页面

## 11.1 页面目标

用于查看登录记录和当前在线会话。

## 11.2 页面组成

页面包含两个页签：

1. 登录日志
2. 在线会话

---

## 11.3 登录日志

用于查看账号登录历史记录。

### 11.3.2 查询条件

- 用户名

### 11.3.3 业务规则

- 登录日志记录登录成功与失败记录。

- 超过保留时长的数据按系统策略清理或归档。

## 11.4 在线会话

用于查看当前在线用户会话状态。

### 11.4.1 列表字段

| 表头     | 说明         |
| ------ | ---------- |
| 用户名    | 当前在线用户     |
| 角色     | 当前用户角色     |
| 工段     | 当前用户工段     |
| 登录时间   | 本次登录时间     |
| 最近活跃时间 | 最近一次操作时间   |
| 登录IP   | 登录来源 IP    |
| 终端信息   | 浏览器 / 设备信息 |
| 状态     | 在线 / 离线    |
| 操作     | 强制下线       |

### 11.4.2 操作说明

- 支持管理员对在线会话执行“强制下线”。
- 支持**批量强制下线**。
- 强制下线后，被下线用户当前登录状态失效，需重新登录。

### 11.4.3 业务规则

- 在线会话状态应与系统登录状态保持一致。
- 在线会话最大时长为 **1 小时**。
- 强制下线操作需记录审计日志。
- 仅**系统管理员**拥有强制下线权限。
- 普通用户不可查看全部在线会话。
- 登录日志/在线会话页面建议仅系统管理员及具备相应权限角色可查看。

## 12. 通用业务规则

## 12.1 用户名规则

- 用户名全局唯一。
- 建议长度范围为 2～10 位。
- 用户名允许中文。
- 用户名不区分大小写。

## 12.2 密码规则

- 长度不少于 6 位。
- 支持任何字符，但不做强制要求。
- 不得连续4位一样字符
- 支持首次登录强制修改密码。
- 支持管理员重置密码。
- 支持个人中心修改密码。
- 不支持与已有用户相同密码

## 12.3 在线状态规则

- 用户登录成功且会话有效时，状态为“在线”。
- 用户主动退出登录后，状态为“离线”。
- 连续超过设定时长无操作后，自动转为“离线”。
- 在线显示绿色；离线显示灰色。

## 12.4 角色与工段联动规则

- 操作员角色必须绑定工段。
- 非操作员角色不可绑定工段。
- 角色从“操作员”修改为其他角色时，工段自动清空。
- 角色修改为“操作员”时，需重新选择工段。
- 工段数据由工艺模块维护，用户模块仅负责引用。

## 12.5 删除规则

- 用户删除统一采用逻辑删除/停用。
- 已产生业务数据的用户、角色不做物理删除。
- 删除操作必须二次确认，并记录日志。

## 12.6 权限安全规则

- 至少保留一个具备权限配置能力的系统管理员账号。
- 不允许通过错误操作导致系统中无人可进入权限配置页面。
- 权限修改必须记录审计日志。

## 15. 评审确认结论

以下内容已在本版中按确认结果固化：

1. 支持新增自定义角色。
2. “新建角色功能”独立为角色管理页面，不放在用户管理页面中。
3. 用户删除统一采用逻辑删除/停用，不做物理删除。
4. 用户名仅系统管理员可修改，普通用户不可修改。
5. 注册审批页面按当前确认方案允许展示密码。
6. 操作员角色下工段为必填。
7. 用户模块调用工艺模块工段数据采用**实时查询**。
8. 本期不做批量审批、批量停用、批量重置密码；仅保留**批量强制下线**。
9. 本期支持用户列表导出（CSV/Excel），不支持导入。
10. 登录日志保留时长为 **30 天**。
11. 在线会话最大时长为 **1 小时**。
12. 强制下线权限仅系统管理员拥有。
13. 本期不做数据范围权限。
