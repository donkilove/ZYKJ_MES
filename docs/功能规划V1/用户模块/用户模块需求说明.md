# 用户模块需求说明

## 1. 文档信息

- 模块名称：用户模块
- 版本：V1.0
- 目标：完成用户、注册审批、角色、权限、个人中心、登录日志/在线会话及审计相关基础能力建设，满足系统后台用户管理与权限控制需求。

---

## 2. 模块范围

本模块本期包含以下页面：

1. 用户管理页面
2. 注册审批页面
3. 角色管理页面
4. 功能权限配置页面
5. 操作审计日志页面
6. 个人中心/账号设置页面
7. 登录日志/在线会话页面

> 说明： 
> **以上 1～7 页面均纳入本期上线范围。**  

---

## 3. 页面清单

| 页面名称        | 是否本期上线 | 说明                       |
| ----------- | ------ | ------------------------ |
| 用户管理页面      | 是      | 管理系统用户，支持查询、新建、编辑、停用/删除等 |

| 功能权限配置页面    | 是      | 为角色配置模块及功能权限             |
| 操作审计日志页面    | 是      | 记录用户、角色、权限等敏感操作日志        |

## 4. 角色定义

### 4.1 系统内置角色

系统默认内置以下角色：

- 系统管理员
- 生产管理员
- 品质管理员
- 操作员
- 维修员

### 4.2 角色规则

- 系统内置角色默认存在。
- 系统内置角色是否允许删除：**不允许删除**。
- 系统内置角色是否允许修改名称：**不允许修改名称**。
- 系统内置角色是否允许配置权限：**允许**。
- 是否支持新增自定义角色：支持。

## 5. 用户管理页面

## 5.1 页面目标

用于统一管理系统内全部用户，支持搜索、查看、新建、编辑、停用、删除等操作。

## 5.2 页面功能

### 5.2.1 搜索/筛选功能

支持以下筛选条件：

- 用户名搜索
- 角色筛选
- 工段筛选
- 账号状态筛选

> 原始需求中明确要求：  
> 
> - 搜索目标为用户名  
>   当前整理稿在此基础上补充了常用筛选项，便于后续扩展。

### 5.2.2 顶部操作区

支持以下按钮：

- 新建用户
- 进入角色管理页面
- 导出用户列表（CSV/Excel）

> 若坚持“新建角色功能”放在用户管理页中，也可保留一个“新建角色”按钮，点击后进入角色管理页面或弹窗创建角色。

## 5.3 新建用户

### 5.3.1 表单字段

| 字段   | 类型   | 必填  | 说明                 |
| ---- | ---- | --- | ------------------ |
| 用户名  | 输入框  | 是   | 全局唯一               |
| 用户密码 | 密码框  | 是   | 新建时填写              |
| 用户角色 | 单选下拉 | 是   | 单选                 |
| 用户工段 | 单选下拉 | 否   | 仅操作员角色可选；数据来源于工艺模块 |
| 账号状态 | 单选   | 否   | 默认启用               |
| 备注   | 输入框  | 否   | 便于管理               |

### 5.3.2 业务规则

- 用户名必须唯一。
- 用户角色为“操作员”时，用户工段可选，必填。
- 用户角色为非“操作员”时，用户工段字段不可编辑，展示为 `/`。
- 用户工段下拉数据来源于**工艺模块的工段基础数据**。
- 新建成功后，用户进入用户列表，默认按数据库 `id` 升序展示。

## 5.4 用户列表展示

### 5.4.1 列表要求

- 列表需带表头。
- 列表区域支持滚动。
- 按数据库 `id` 字段排序展示，默认升序。
- 支持空数据占位展示。

### 5.4.2 表头字段

| 表头   | 说明                |
| ---- | ----------------- |
| 用户名  | 展示用户名             |
| 角色   | 展示当前用户角色          |
| 工段   | 未分配、无法分配角色时显示 `/` |
| 在线状态 | 在线显示绿色；离线显示灰色     |
| 账号状态 | 展示启用/停用           |
| 操作   | 下拉操作菜单            |

### 5.4.3 操作菜单

下拉菜单包含：

- 编辑
- 停用
- 启用
- 重置密码
- 删除

> 原始需求中为“编辑、删除”。  
> 当前整理稿建议补充“停用/启用、重置密码”，以减少直接删除带来的数据风险。

## 5.5 编辑用户

### 5.5.1 可编辑字段

- 用户名（是否允许修改：可修改）
- 用户密码
- 用户角色
- 用户工段
- 账号状态
- 备注

### 5.5.2 编辑规则

- 若角色修改为“操作员”，则需选择工段。
- 若角色由“操作员”改为其他角色，则原工段自动清空。
- 用户工段数据来源于工艺模块，不在本模块内维护。
- 若用户已有关联业务数据，禁止物理删除，仅允许停用。

## 5.6 删除规则

建议如下：

- 默认采用**逻辑删除/停用**，不建议直接物理删除。
- 仅当用户未产生任何业务关联数据时，才允许物理删除。
- 删除前需二次确认。

## 6. 注册审批页面

## 6.1 页面目标

用于处理待审批注册申请，支持审批通过、驳回。

## 6.2 页面功能

### 6.2.1 列表要求

- 展示全部待审批注册用户。
- 列表需带表头。
- 列表区域支持滚动。
- 按数据库 `id` 字段排序展示，默认升序。

### 6.2.2 表头字段

| 表头   | 说明                       |
| ---- | ------------------------ |
| 用户名  | 注册申请用户名                  |
| 密码   | 原始需求为展示密码；建议改为脱敏展示或不展示明文 |
| 申请状态 | 待审批/已通过/已驳回              |
| 操作   | 下拉操作菜单                   |

## 6.3 操作菜单

下拉菜单包含：

- 通过
- 驳回

## 6.4 通过操作

### 6.4.1 通过表单字段

| 字段   | 类型   | 必填  | 说明                 |
| ---- | ---- | --- | ------------------ |
| 用户名  | 输入框  | 是   | 默认带出申请用户名          |
| 用户密码 | 密码框  | 是   | 默认带出或重新设置          |
| 用户角色 | 单选下拉 | 是   | 单选                 |
| 用户工段 | 单选下拉 | 否   | 仅操作员角色可选；数据来源于工艺模块 |

### 6.4.2 通过规则

- 审批通过时需校验用户名唯一性。
- 用户角色为“操作员”时，需选择工段。
- 用户工段下拉数据来源于**工艺模块的工段基础数据**。
- 审批通过后，注册申请状态更新为“已通过”，并生成正式用户账号。

## 6.5 驳回操作

### 6.5.1 交互要求

- 点击“驳回”后弹出确认弹窗。
- 支持填写驳回原因。

### 6.5.2 驳回规则

- 驳回后申请状态更新为“已驳回”。
- 保留驳回记录，不直接删除申请数据。

## 6.6 安全建议

- 所有审批操作需写入审计日志。

## 7. 角色管理页面

## 7.1 页面目标

用于维护角色信息，管理角色的启用状态、说明及基础属性。

## 7.2 页面功能

### 7.2.1 新建角色

支持以下字段：

| 字段   | 类型  | 必填  | 说明         |
| ---- | --- | --- | ---------- |
| 角色名称 | 输入框 | 是   | 角色名称唯一     |
| 角色说明 | 输入框 | 否   | 描述角色用途     |
| 角色类型 | 单选  | 是   | 系统内置 / 自定义 |
| 状态   | 单选  | 否   | 默认启用       |

### 7.2.2 角色列表

| 表头    | 说明        |
| ----- | --------- |
| 角色名称  | 角色名称      |
| 角色说明  | 角色说明      |
| 角色类型  | 系统内置/自定义  |
| 关联用户数 | 当前角色下用户数量 |
| 状态    | 启用/停用     |
| 操作    | 编辑、停用、删除  |

## 7.3 角色规则

- 系统内置角色不可删除。
- 系统内置角色可配置权限。
- 自定义角色是否支持新增：支持。
- 已绑定用户的角色不允许直接删除，建议先解除用户绑定或停用角色。

## 8. 功能权限配置页面

## 8.1 页面目标

用于按角色配置各业务模块的功能权限。

## 8.2 页面范围

### 8.2.1 模块选择下拉栏

支持以下模块（后期可能会增加）：

- 用户模块
- 产品模块
- 工艺模块
- 生产模块
- 品质模块
- 设备模块

### 8.2.2 角色选择下拉栏

- 支持选择需配置权限的角色（后期可能会增加）。

## 8.3 页面功能

### 8.3.1 能力包展示功能

根据所选模块与角色，展示对应能力包权限项，支持勾选保存。

### 8.3.2 本页面需去掉的功能

当前版本明确去掉以下功能：

- 搜索
- 当前版本显示
- 生效预览
- 查看变更历史
- 查看最近保存结果
- 恢复上次保存前草稿
- 套用岗位模板

## 8.4 权限粒度建议

建议权限粒度至少包含：

- 页面查看
- 新建
- 编辑
- 删除
- 审批
- 配置

> 若后续业务需要更细控制，可再扩展数据范围权限，例如：  
> 
> - 仅本人  
> - 仅本工段  
> - 全部数据

## 8.5 特殊规则

### 8.5.1 系统管理员规则

- 系统管理员**不再固定全权限**。
- 系统管理员**固定拥有“功能权限配置页面”访问权限**。
- “功能权限配置页面”权限仅系统管理员角色可拥有、可编辑。
- “功能权限配置页面”权限**不出现在能力包中**。
- 非系统管理员角色不可拥有该权限。

### 8.5.2 保存规则

- 修改权限后需点击“保存”生效。
- 保存成功后写入审计日志。
- 校验至少保留一个可进入权限配置页面的系统管理员账号，避免系统失控。

## 10. 个人中心/账号设置页面

## 10.1 页面目标

用于普通用户维护本人账号信息。

## 10.2 页面功能

- 查看当前登录账号用户名
- 查看当前登录账号角色
- 查看当前当前登录账号所属工段
- 修改本人密码
- 查看当前登录账号状态

> 说明：  
> 当前工段仅展示用户已绑定工段信息，工段数据来源于工艺模块。

## 10.3 页面字段建议

| 字段    | 是否可编辑 | 说明      |
| ----- | ----- | ------- |
| 用户名   | 否     | 仅展示     |
| 当前角色  | 否     | 仅展示     |
| 当前工段  | 否     | 仅展示     |
| 账号状态  | 否     | 仅展示     |

- 用户不可在个人中心修改自己的角色和工段。

## 11. 登录日志/在线会话页面

## 11.1 页面目标

用于查看登录记录和当前在线会话。

## 11.2 页面组成

包含两个页签：

1. 登录日志
2. 在线会话

### 11.2.1 登录日志

用于查看账号登录历史记录。

#### 列表字段建议

| 表头   | 说明      |
| ---- | ------- |
| 用户名  | 登录账号    |
| 登录时间 | 登录发生时间  |
| 登录结果 | 成功/失败   |
| 失败原因 | 登录失败时展示 |

#### 查询条件建议

- 用户名
- 登录结果
- 时间范围

### 11.2.2 在线会话

用于查看当前在线用户会话状态。

#### 列表字段建议

| 表头   | 说明     |
| ---- | ------ |
| 用户名  | 当前在线用户 |
| 角色   | 当前用户角色 |
| 工段   | 当前用户工段 |

## 11.3 业务规则

- 登录日志需记录登录成功与失败记录。
- 在线会话状态应与系统登录状态保持一致。
- 强制下线操作需记录审计日志。
- 普通用户是否可查看全部在线会话：不可。
- 仅系统管理员及具备相应权限角色可查看登录日志/在线会话页面。

## 12. 通用业务规则

## 12.1 用户名规则

- 用户名全局唯一。
- 限制长度范围，2～10 位。
- 是否允许中文：允许。
- 是否区分大小写：不区分。

## 12.2 密码规则

建议至少满足以下要求：

- 长度不少于 6 位
- 可以包含数字、字母、符号
- 支持管理员重置密码
- 支持个人中心修改密码

## 12.3 在线状态规则

建议定义如下：

- 用户登录成功且会话有效时，状态为“在线”。
- 用户主动退出登录后，状态为“离线”。
- 连续超过设定时长无操作后，自动转为“离线”。
- 在线显示绿色；离线显示灰色。

## 12.4 角色与工段联动规则

- 操作员角色必须绑定工段。
- 非操作员角色不可绑定工段。
- 角色从“操作员”修改为其他角色时，工段自动清空。
- 角色修改为“操作员”时，需重新选择工段。
- 工段数据由工艺模块维护，用户模块仅负责引用。

## 12.5 删除规则

- 优先采用逻辑删除/停用。
- 已产生业务数据的用户、角色不允许物理删除。
- 删除操作必须二次确认，并记录日志。

## 12.6 权限安全规则

- 至少保留一个具备权限配置能力的系统管理员账号。
- 不允许通过错误操作导致系统中无人可进入权限配置页面。
- 权限修改必须记录审计日志。

## 13. 模块边界说明

### 13.1 用户模块负责

- 用户基础信息管理
- 用户角色分配

- 注册审批
- 权限配置
- 个人中心/账号设置
- 登录日志/在线会话
- 审计日志

### 13.2 工艺模块负责（暂时）

- 工段基础资料管理
- 工段新增/编辑/停用/删除
- 工段数据字典维护

### 13.3 交互关系

- 用户模块中的“用户工段”字段，数据来源于工艺模块。
- 用户模块不提供工段新增、编辑、删除能力。
- 当工艺模块中的工段状态变更时，用户模块应同步使用最新可选工段数据。

## 15. 待确认项

以下事项建议在正式评审前确认：

1. 是否支持新增自定义角色。
   
   回复：支持。
2. “新建角色功能”是否保留在用户管理页面，还是独立为角色管理页面。
   回复：独立为角色管理页面。
3. 用户删除是否为物理删除，还是统一采用逻辑删除/停用。
   
   回复：采用逻辑删除/停用。
4. 用户名是否允许修改。
   回复：普通用户（非系统管理员）不得更改。
5. 注册审批页面是否允许展示明文密码。
   回复：允许。
6. 操作员角色下工段是否强制必填。
   回复：强制。
7. 用户模块调用工艺模块工段数据的方式，是实时查询还是同步缓存。
   回复：实时查询。
8. 是否需要批量操作能力（批量审批、批量停用、批量重置密码等）。
   回复：只需要批量强制下线功能。
9. 是否需要导入/导出功能。
   回复：导入/导出什么？
10. 登录日志保留时长、在线会话超时时长如何定义。
    回复：登录日志保存30天，30天前的登录日志物理删除。在线会话时长最长一个小时。
11. 强制下线权限由哪些角色拥有。
    回复：仅系统管理员
12. 是否需要数据范围权限（仅本人/仅本工段/全部）。
    回复：不需要
