### 1.2 通用业务规则实现情况

| 规则 | 状态 | 说明 |
|------|------|------|
| 用户名全局唯一 | ✅ | 数据库唯一约束 + 应用层校验 |

| 在线会话最大时长 1 小时 | ✅ | `session_max_seconds=3600` |
| 登录日志保留 30 天 | ✅ | `login_log_retention_days=30` |
| 操作员必须绑定工段 | ✅ | `_resolve_stage()` 校验 |
| 非操作员不可绑定工段 | ✅ | `_resolve_stage()` 校验 |

## 三、总结

| 维度 | 完成度 | 主要差距 |
|------|--------|---------|
| 后端 API | 96% | 密码校验规则缺失、用户名长度限制不符、缺 Excel 导出 |
| 前端页面 | 85% | 用户管理页面操作菜单不完整、缺少高级筛选和导出 |
| 整体 | 约 90% | 核心差距集中在用户管理页面的前端 UI 层 |

备注：后端 API 层已全部就绪（`disableUser`/`enableUser`/`resetUserPassword`/`exportUsers`），前端服务层也已封装好对应调用方法，缺失的仅是页面 UI 的接入。
