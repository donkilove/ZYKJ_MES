### 二、任务目标

请在现有仓库基础上，实现“消息模块”第一版，要求如下：

- 在左侧一级侧边栏新增独立入口：`消息`
- 风格必须与现有一级模块保持一致
- 实现站内消息中心页面
- 实现未读数、单条已读、全部已读
- 在侧边栏“消息”入口展示未读角标
- 后端新增消息主表与消息收件记录表
- 后端提供 HTTP API：列表、未读数、单条已读、全部已读
- 增加 WebSocket 实时提醒能力
- WebSocket 失败时，前端必须保留 HTTP 轮询兜底
- 接入第一批消息来源：
  - 注册审批待处理
  - 代班审批待处理
  - 关键生产事件
  - 设备保养到期或自动生成结果

### 三、实现边界

第一版必须做：

- 独立一级侧边栏入口 `message`
- 消息中心页面
- 消息主表 `message`
- 消息收件记录表 `message_recipient`
- 消息列表接口
- 未读数接口
- 单条已读接口
- 全部已读接口
- WebSocket 实时轻量事件推送
- 轮询兜底
- 基础权限接入

第一版不要做：

- 短信、邮件、企业微信、钉钉
- 富文本公告编辑器
- 复杂规则编排器
- 大范围重构现有壳层
- 与任务无关的代码清理

### 四、必须遵守的实现要求

1. 优先做最小且有效的改动，不要做无关重构。
2. 严格复用现有页面目录、权限快照、模块权限机制，不要额外绕开权限系统做硬编码显示。
3. 已读状态必须按“用户维度”记录，不能只在消息主表上做全局已读。
4. WebSocket 只推送轻量事件，不直接推送完整消息正文。
5. 前端全局未读状态尽量挂在 `mes_client/lib/pages/main_shell_page.dart`，确保任何页面下都能更新侧边栏角标。
6. 新增文案、文档、注释默认使用中文。
7. 不要覆盖或回退与当前任务无关的修改。
8. 所有新增或修改的文本文件必须保持 UTF-8 无 BOM、LF 行尾。
