## 1. 检查结论

当前消息模块对需求说明的满足情况：`部分满足（P0 已修复，可进入联调）`

结论摘要：

- 本轮修复了 4 项 P0 缺口：WebSocket 协议命名统一、消息创建统一触发推送、已读操作补充审计日志、跳转无权限/来源失效提示。
- 消息模块核心能力已基本到位，可进入联调验收阶段。
- 剩余缺口集中在跨模块来源接入（生产/品质）和概览统计口径优化，属于 P1 后续迭代项。

## 3. 本轮修复项

| 缺口 | 修复内容 | 修复位置 |
| --- | --- | --- |
| WebSocket 协议命名不一致（`new_message` vs `message_created`） | 前端改为监听 `message_created` 事件 | `mes_client/lib/pages/main_shell_page.dart` |
| 消息创建未统一触发推送 | `create_message` 提交后自动调度 `push_message_created` 给所有收件人 | `backend/app/services/message_service.py` |
| 已读操作缺审计留痕 | 单条已读/全部已读接口补充 `write_audit_log` | `backend/app/api/v1/endpoints/messages.py` |
| 跳转无权限提示缺失 | 跳转前校验 `_menus`，无权限时显示 SnackBar | `mes_client/lib/pages/main_shell_page.dart` |
| 来源失效提示缺失 | 消息 tile 中 `status != 'active'` 时显示"来源已失效"并隐藏跳转按钮 | `mes_client/lib/pages/message_center_page.dart` |
