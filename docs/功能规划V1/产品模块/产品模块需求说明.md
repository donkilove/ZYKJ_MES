# 产品模块需求说明（最终可评审稿）

## 1. 文档信息

- 模块名称：产品模块
- 文档版本：V1.0
- 文档状态：可评审
- 目标：完成产品主数据、产品版本、产品参数、参数追溯与生效管理等基础能力建设，满足 MES 场景下产品资料维护、版本切换和参数查询需求。

---

## 2. 本期范围

本模块本期包含以下页面：

1. 产品管理页面
2. 版本管理页面
3. 产品参数管理页面
4. 产品参数查询页面

### 2.1 模块边界说明

- **以上 1～4 页面均纳入本期上线范围。**
- 产品模块负责维护产品主数据、产品版本、产品参数及参数变更追溯。
- 产品模块中涉及工艺路线、BOM、设备、质检标准、包装规则等内容时，默认作为**关联信息/引用信息**展示或绑定，不在本模块内做主数据维护。
- 若系统已有工艺、生产、品质、设备等模块，产品模块仅提供产品维度的基础数据和生效版本给其他模块引用。

## 4. 核心对象定义

### 4.1 产品

产品是业务主对象，用于承载一个产品的基础身份信息。  
一个产品可以有多个版本，但同一时刻仅允许一个生效版本。

### 4.2 产品版本

产品版本用于承载某一阶段的完整产品参数快照。  
版本是产品参数管理和生效管理的核心对象。

### 4.3 产品参数

产品参数用于描述产品在生产、检测、包装或业务执行中需要使用的关键配置项。  
参数应绑定到具体产品版本，避免直接覆盖历史参数。

## 5. 产品管理页面

## 5.1 页面目标

用于统一管理系统内全部产品，支持搜索、筛选、查看、新建、编辑、启用、停用、版本管理、参数查看和删除等操作。

## 5.2 页面功能

### 5.2.1 搜索/筛选功能

支持以下筛选条件：

- 产品名称搜索（模糊搜索）
- 产品分类筛选
- 产品状态筛选（启用/停用）
- 生效版本状态筛选（有生效版本/无生效版本）
- 更新时间范围筛选

### 5.2.2 顶部操作区

支持以下按钮：

- 新建产品（弹窗）
- 导出产品列表

> 说明：  
> 本期支持导出，不需支持批量导入；如后续业务有批量建档需求，可在后续版本补充。

---

## 5.3 产品列表展示

### 5.3.1 列表要求

- 列表需带表头。
- 列表区域支持滚动。
- 默认按更新时间倒序展示。
- 支持空数据占位展示。

### 5.3.2 表头字段

| 表头   | 说明                                  |
| ---- | ----------------------------------- |
| 产品名称 | 产品名称                                |
| 产品分类 | 产品分类                                |
| 当前状态 | 启用 / 停用                             |
| 当前版本 | 当前可继续维护的版本；若存在草稿则显示该草稿版本，否则显示当前生效版本 |
| 生效版本 | 当前实际生效版本                            |
| 创建时间 | 产品创建时间                              |
| 更新时间 | 最近更新时间                              |
| 操作   | 下拉操作菜单                              |

### 5.3.3 操作菜单

下拉菜单包含：

- 查看详情
- 编辑产品
- 停用
- 启用
- 版本管理
- 查看参数
- 编辑参数
- 删除产品

---

## 5.4 列表业务规则

- 产品名称全局唯一。
- 产品分类为固定枚举。
- 已被其他业务单据、工单、生产记录、质检记录引用的产品，不允许逻辑删除，仅允许停用。
- 停用产品默认不可继续用于新建业务单据，但不影响历史数据展示。
- 逻辑删除与停用需区分：停用后产品仍在列表可见且可恢复启用；逻辑删除后默认不在常规列表展示，不可直接用于业务选择。
- 删除、启用、停用、版本切换等关键操作均需记录操作日志。

## 6. 产品详情查看（产品管理页内）

## 6.1 能力说明

产品详情以抽屉或侧边栏形式在产品管理页面内展示，不单独作为独立页面存在。

## 6.2 展示内容

### 6.2.1 基础信息区

至少展示以下内容：

- 产品名称
- 产品分类
- 产品状态
- 备注
- 创建时间
- 更新时间

### 6.2.2 版本概览区

至少展示以下内容：

- 当前版本号
- 当前版本状态
- 当前生效版本号
- 生效时间
- 最近一次版本变更时间

### 6.2.3 当前生效参数快照区

- 展示当前生效版本下的参数列表。
- 支持按参数名称搜索。
- 支持查看参数值、参数类型、参数分组、参数说明。

### 6.2.4 关联信息区

- 可展示关联工艺路线、适用产线、设备、质检标准、包装规则等信息。
- 若相关模块尚未建设，可展示为空或隐藏该区域。

### 6.2.5 变更记录区

- 展示最近的产品编辑、版本切换、参数修改、生效操作记录。
- 参数变更完整明细在产品参数管理页面内查看。
- 版本发布和生效结果在版本管理页面内查看。

## 7. 新增/编辑产品（产品管理页内）

## 7.1 能力说明

新建产品和编辑产品在产品管理页面内通过弹窗或抽屉完成，不单独作为独立页面存在。

## 7.2 表单字段

| 字段   | 类型   | 必填  | 说明     |
| ---- | ---- | --- | ------ |
| 产品名称 | 输入框  | 是   | 全局唯一   |
| 产品分类 | 下拉选择 | 是   | 固定枚举   |
| 默认状态 | 单选   | 否   | 默认启用   |
| 备注   | 输入框  | 否   | 便于管理说明 |

> 说明：  
> 产品分类固定枚举值为：贴片、DTU、套件。

> 补充校验建议：  
> 产品名称去除首尾空格后参与唯一性校验，长度建议为 1~100 个字符；备注长度建议不超过 500 个字符。

## 7.3 业务规则

- 新建产品时必须校验产品名称唯一。
- 产品分类仅允许选择固定枚举值。
- 编辑产品时允许修改产品名称，但修改后仍必须校验产品名称唯一。
- 产品创建成功后，默认自动生成一个初始版本草稿 `V1.0`，便于后续进入版本和参数维护。
- 产品名称为空、全空格或超长时，应在前端即时提示，后端仍需二次校验。

## 8. 版本管理页面

## 8.1 页面目标

用于管理同一产品下的多个版本，确保参数变更有版本承载、生效过程可控、历史状态可追溯。

## 8.2 页面功能

### 8.2.1 顶部操作区

支持以下按钮：

- 新建版本
- 复制版本
- 导出版本参数

### 8.2.2 版本列表字段

| 表头   | 说明                   |
| ---- | -------------------- |
| 版本号  | 如 V1.0、V1.1，由系统自动生成  |
| 版本状态 | 草稿 / 已生效 / 已失效 / 已停用 |
| 来源版本 | 复制来源版本，可为空           |
| 变更摘要 | 本次版本修改说明             |
| 创建人  | 创建该版本的用户             |
| 创建时间 | 版本创建时间               |
| 生效时间 | 实际生效时间               |
| 操作   | 下拉操作菜单               |

### 8.2.3 操作菜单

下拉菜单包含：

- 查看版本详情
- 编辑版本
- 维护参数
- 立即生效
- 停用版本
- 删除版本

### 8.2.4 发布生效功能

支持以下能力：

- 查看可发布版本
- 提交版本立即生效
- 确认立即生效
- 查看最近一次生效结果

### 8.2.5 版本状态说明

| 状态  | 含义               | 允许操作                   |
| --- | ---------------- | ---------------------- |
| 草稿  | 未生效、可继续维护的版本     | 查看详情、编辑版本、维护参数、删除、立即生效 |
| 已生效 | 当前业务实际使用中的版本     | 查看详情、复制版本、查看生效结果       |
| 已失效 | 曾生效但已被新版本替代的历史版本 | 查看详情、复制版本、在未被业务引用时停用   |
| 已停用 | 人工停用后仅保留追溯记录的版本  | 查看详情、复制版本              |

---

## 8.3 版本规则

- 同一产品同一时刻仅允许一个生效版本。
- 同一产品同一时刻仅允许一个草稿版本；存在草稿时，不允许再次新建版本，可继续编辑或删除当前草稿。
- 当前版本定义为“当前可继续维护的版本”：若存在草稿，则当前版本为草稿版本；若不存在草稿，则当前版本为当前生效版本。
- 版本号由系统自动生成，格式为 `V<主版本>.<次版本>`；新建产品默认生成 `V1.0` 草稿版本。
- 复制版本时默认基于当前最高版本号生成新草稿，规则为次版本号加 1，例如 `V1.0 -> V1.1`、`V1.1 -> V1.2`。
- 版本生效后，不允许直接修改参数；如需变更，应基于当前版本复制出新版本后再调整。
- 草稿版本允许编辑和删除。
- 已生效且已被业务引用的版本不允许物理删除。
- 设置新版本生效后，原生效版本自动转为已失效。
- 已停用版本不可直接恢复生效；如需再次启用，应复制出新草稿后重新生效。
- 立即生效前必须校验：产品状态为启用、目标版本状态为草稿、版本下至少存在一条参数记录。
- 若用户确认生效时系统发现已有其他版本抢先完成生效，应终止本次操作并提示刷新后重试，避免并发覆盖。
- 本期仅支持立即生效，计划生效列入 P1。
- 版本对比列入 P1。

| 变更原因 | 变更说明                   |

| 操作时间 | 变更时间                   |
| 操作   | 查看详情                   |

## 9.3 参数编辑内容

单个参数至少包含以下字段：

| 字段   | 类型   | 必填  | 说明            |
| ---- | ---- | --- | ------------- |
| 参数名称 | 输入框  | 是   | 同一版本内必须唯一     |
| 参数分组 | 下拉选择 | 是   | 固定枚举          |
| 参数类型 | 下拉选择 | 是   | 固定选项：文本、Link  |
| 参数值  | 输入框  | 是   | 文本内容或 Link 地址 |
| 参数说明 | 输入框  | 否   | 备注和解释         |

> 说明：  
> 参数分组固定枚举值为：基础参数、激光打标参数、产品测试参数、产品组装参数、产品包装参数。

> 补充校验建议：  
> 参数名称长度建议为 1~100 个字符；文本类型参数值长度建议不超过 1000 个字符；`Link` 类型参数值长度建议不超过 500 个字符；参数说明长度建议不超过 500 个字符。

## 9.4 参数规则

- 参数必须归属于具体产品版本，不允许脱离版本单独存在。
- 同一产品版本下，参数名称必须唯一，不允许同版本内出现重复参数。
- 参数分组仅允许选择固定枚举值。
- 参数类型仅允许选择：文本、Link；其中 Link 用于打开软件或文档。
- `Link` 类型支持 `http://`、`https://`、UNC 共享路径 `\\` 和本地绝对路径（如 `C:\`）格式；格式不合法时不允许保存。
- 已生效版本的参数默认只读；如需修改，需复制出新版本进行变更。
- 参数新增、编辑、删除时均需记录修改人、修改时间、修改前值、修改后值和修改原因；修改原因在保存时必填。

## 11. 通用业务规则

## 11.1 产品名称规则

- 产品名称全局唯一。
- 产品名称应具备业务可识别性。
- 产品名称允许修改，但修改后仍必须保持全局唯一。

## 11.2 产品状态规则

- 产品状态包含启用、停用。
- 启用产品可被其他业务模块选择使用。
- 停用产品不可继续用于新增业务，但不影响历史数据追溯。

## 11.3 版本规则

- 每个产品至少保留一个版本记录。
- 每个产品同一时刻只能有一个生效版本。
- 每个产品同一时刻仅允许一个草稿版本。
- 当前版本指当前可继续维护的版本；存在草稿时取草稿版本，不存在草稿时取当前生效版本。
- 生效版本指当前业务实际使用版本。
- 当前版本与生效版本允许不一致。

## 11.4 删除规则

- 产品删除统一采用逻辑删除方式，不做无条件物理删除；停用作为状态控制手段，不等同于删除。
- 停用后的产品仍保留在常规列表中，可重新启用；逻辑删除后的产品默认不在常规列表中展示。
- 已产生业务数据的产品、版本、参数记录不允许物理删除。
- 已被业务单据、工单、生产记录、质检记录引用的产品，不允许逻辑删除，仅允许停用。
- 删除操作必须二次确认，并记录日志。

## 11.5 查询规则

- 产品管理、参数管理、参数查询页面均应支持基础搜索和常用筛选。
- 默认优先展示最近更新的数据。
- 无权限用户不可查看编辑入口。

## 11.6 权限与审计规则

- 权限粒度建议至少包含：查看、新建、编辑、删除、版本管理、参数维护、发布生效、导出。
- 启停用、删除、参数修改、版本切换、生效等关键动作需写入审计日志。
- 参数查询页面建议对更多只读角色开放，参数编辑和生效权限应单独控制。
- 本期默认仅控制功能权限，不引入按产品分类、组织、工厂维度的数据范围隔离；如后续需要，列入专项需求。

## 11.7 表单校验与异常提示规则

- 产品名称、参数名称、参数值、变更摘要、变更原因均需去除首尾空格后再校验是否为空。
- 产品重名、参数重名、非法 `Link`、无参数版本尝试生效、存在草稿仍重复新建版本等场景，系统需给出明确提示，不可仅返回通用失败信息。
- 前端需提供即时校验提示，后端需保留同等约束作为最终兜底，避免绕过前端校验写入脏数据。
- 对于已被引用而不可删除、版本状态不允许编辑或生效失败的场景，提示语需明确说明原因和下一步建议操作。

## 14. 评审确认结论

以下内容已在本版中按确认结果固化：

1. 产品模块本期聚焦 4 个页面：产品管理、版本管理、产品参数管理、产品参数查询。
2. 产品详情查看作为产品管理页面内能力，不单独设页面。
3. 新增/编辑产品作为产品管理页面内能力，不单独设页面。
4. 参数变更历史并入产品参数管理页面，发布生效并入版本管理页面。
5. 产品主数据不设置产品编码，产品名称全局唯一。
6. 新建/编辑产品表单仅保留产品名称、产品分类、默认状态、备注。
7. 产品分类采用固定枚举：贴片、DTU、套件。
8. 产品参数必须绑定到具体产品版本，不允许直接覆盖历史生效参数。
9. 产品参数不设置参数编码，参数类型仅保留文本、Link 两类。
10. 参数分组采用固定枚举：基础参数、激光打标参数、产品测试参数、产品组装参数、产品包装参数。
11. 同一产品同一时刻仅允许一个生效版本。
12. 本期支持新建版本、复制版本、立即生效；版本对比和计划生效列入 P1。
13. 产品删除默认采用逻辑删除，不做无条件物理删除；被业务引用时仅允许停用。
14. 参数变更需记录修改前后内容、修改人、修改时间和修改原因。
15. 产品参数查询页面作为只读查询入口保留。
16. 本期支持产品列表和版本参数导出。
17. 本期默认不引入复杂审批流，仅保留发布生效确认能力。
18. 当前版本定义为当前可继续维护版本；存在草稿时优先取草稿，不存在草稿时取当前生效版本。
19. 同一产品同一时刻仅允许一个草稿版本。
20. 版本号采用系统自动生成规则，初始版本默认为 `V1.0`，复制版本按次版本号递增生成。
21. 同一产品版本下参数名称必须唯一，`Link` 类型需通过格式校验后方可保存。
22. 逻辑删除与停用明确区分：停用可恢复，逻辑删除默认不在常规列表展示。
23. 本期不引入按产品分类、组织、工厂维度的数据范围权限隔离。

## 16. 验收要点（建议）

- 产品新建成功后，系统自动生成 `V1.0` 草稿版本，并可直接进入版本维护和参数维护流程。
- 同一产品存在草稿版本时，重复点击“新建版本”会被拦截，并给出继续编辑现有草稿的明确提示。
- 生效操作仅允许对草稿版本执行；生效成功后，新版本变为已生效，原生效版本自动变为已失效。
- 已生效版本参数不可直接修改；用户需通过复制版本生成新草稿后再维护参数。
- 同一版本下新增或编辑参数时，若参数名称重复，系统必须阻止保存并给出明确提示。
- `Link` 类型参数保存时会校验格式；非法链接或非法路径不得保存成功。
- 产品停用后不可用于新增业务选择，但历史记录、历史单据和历史版本仍可正常查询。
- 逻辑删除后的产品默认不在常规列表显示；已被业务引用的产品执行删除时，系统必须阻止并提示仅允许停用。
