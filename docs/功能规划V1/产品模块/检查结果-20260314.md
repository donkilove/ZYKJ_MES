## 各页面详细检查

### 1. 产品管理（P0）— ✅ 基本完成

**后端实现：**
- `products.py`：产品列表（分页/搜索/分类/状态/生效版本/更新时间范围筛选）、新增（名称唯一校验、自动生成 V1.0 草稿）、编辑、逻辑删除（密码确认）、生命周期状态管理、影响分析、回滚、CSV 导出
- 审计日志已接入

**前端实现：**
- `product_management_page.dart`：产品列表、搜索/筛选、新增/编辑弹窗、启用/停用、删除、版本管理入口、参数入口、导出、权限控制

**缺失功能点：**
- 产品详情弹窗缺少"当前生效参数快照区"（需求 6.2.3）
- 产品详情弹窗缺少"变更记录区"（需求 6.2.5）
- 后端未对产品分类做枚举校验（仅前端限制，后端未兜底）
- 删除时未校验业务引用（`delete_product` 直接设 `is_deleted=True`，未检查关联工单；影响分析接口已有但删除流程未调用）
- 前端未提供更新时间范围筛选（后端已支持 `updated_after`/`updated_before`）

### 2. 版本管理（P0）— ✅ 基本完成

**后端实现：**
- `products.py`：版本列表、新增、复制、激活/停用、删除、备注更新、版本对比（P1 已提前实现）、版本参数导出、版本回滚（P1 已提前实现）
- 业务规则：同一时刻仅一个生效版本、仅一个草稿版本、版本号自动生成

**前端实现：**
- `product_version_management_page.dart`：左侧产品列表 + 右侧版本列表、新建/复制/激活/停用/删除（含二次确认）

**缺失功能点：**
- 版本列表缺少"变更摘要"列（后端有 note 字段，前端未展示）
- 版本列表缺少"生效时间"列
- 缺少"查看版本详情"操作入口
- 缺少"编辑版本备注"操作入口（后端 PATCH note 接口已有）
- 缺少"维护参数"操作入口（从版本管理页跳转参数编辑）
- 缺少"导出版本参数"按钮（后端接口已有，前端服务层也有方法，页面 UI 未接入）

### 3. 参数管理（P1）— ✅ 基本完成

**后端实现：**
- `products.py`：参数列表查询、参数批量更新（含变更原因、confirmed 确认）、参数变更历史、参数导出
- 业务规则：参数名称同版本内唯一、类型仅 Text/Link、Link 格式校验、变更记录留痕

**前端实现：**
- `product_parameter_management_page.dart`：参数列表、参数编辑（名称/分组/类型/值/说明）、变更历史弹窗、导出、变更原因必填

**缺失功能点：**
- 参数未严格按版本维度管理（`ProductParameter` 通过 `product_id` 关联产品而非版本，版本快照通过 `snapshot_json` 实现）
- 参数列表缺少"产品分类"、"版本号"列
- 参数列表缺少"最后修改时间"列

### 4. 参数查询（P1）— ✅ 基本完成

**后端实现：**
- 复用产品列表和参数查询接口

**缺失功能点：**
