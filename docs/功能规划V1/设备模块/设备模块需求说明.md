## 1. 文档信息

- 模块名称：设备模块
- 文档版本：V1.0
- 文档状态：可评审
- 目标：完成设备台账、保养项目、保养计划、保养执行、保养记录与设备规则等核心能力建设，满足 MES 场景下设备基础档案管理、保养闭环执行和设备追溯分析需求。

---

## 2. 本期范围

本模块本期包含以下主页面：

1. 设备台账页面
2. 保养项目页面
3. 保养计划页面
4. 保养执行页面
5. 保养记录页面

本模块本期包含以下子页面或增强页面：

1. 设备详情页面（建议新增）
2. 保养执行详情页面（建议新增）
3. 设备规则 / 运行参数页面（建议新增）

### 2.1 模块边界说明

- **以上页面均纳入本期规划范围。**
- 设备模块负责设备档案、保养项目、保养计划、保养工单、保养记录、设备规则和设备运行参数等设备域数据。
- 设备模块中涉及工段、执行人、生产订单、品质异常等信息时，默认作为**引用信息或联动信息**使用，不在本模块内做主数据归口维护。
- 工艺模块提供工段数据，设备模块仅引用工段作为保养执行归属工段。
- 用户模块提供执行人、负责人、角色和权限数据。
- 生产模块、品质模块仅引用设备信息，不在各自模块内维护设备主数据。
- 本模块本期重点是“设备保养闭环”，不是完整的设备资产全生命周期管理系统。

### 2.2 本期建议新增页面与功能

在现有系统能力基础上，本需求说明**额外纳入以下新增建议**：

- 新增**设备详情页面**，支持按设备查看关联计划、未完成工单、近期保养记录和状态摘要。
- 新增**保养执行详情页面**，支持查看工单来源快照、执行人、执行过程、结果备注和附件。
- 新增**设备规则 / 运行参数页面**，支持维护设备规则、参数模板、阈值和启停控制口径。
- 在保养计划页面补充**默认执行人、预计时长**完整录入能力。
- 在保养执行页面补充**状态筛选、仅看我的任务、附件留证**能力。
- 在保养记录页面补充**执行人筛选、结果筛选、导出和详情查看**能力。
- 在设备台账页面取消位置硬编码，改为可配置位置或自由录入机制。

## 4. 核心对象定义

### 4.1 设备台账

设备台账是设备模块的一级主数据对象，用于记录设备编号、名称、型号、位置、负责人、启停状态等基础档案信息。

### 4.2 保养项目

保养项目用于表达设备保养标准项，例如点检、润滑、校准、清洁等，是保养计划配置的基础模板。

### 4.3 保养计划

保养计划用于将“设备 + 保养项目 + 执行工段 + 周期”组合成可持续生效的计划对象，并产出保养执行工单。

### 4.4 保养工单

保养工单用于承载某次具体保养执行任务，记录应执行日期、执行人、状态、开始时间、完成时间、结果摘要和附件等信息。

### 4.5 保养记录

保养记录用于保存已完成工单的结果快照，是设备保养追溯、稽核和统计的核心历史对象。

### 4.6 设备规则

设备规则用于表达设备运行、保养、阈值、检查项等可配置规则，是设备模块向更完整设备管理能力演进的基础对象。

### 4.7 运行参数

运行参数用于表达设备关键参数模板、单位、上下限和启用状态，用于辅助设备状态管理和后续联动分析。

## 5. 核心流程概述

本模块本期需支持以下核心流程：

1. 建立设备台账
2. 配置保养项目
3. 配置保养计划
4. 手工或自动生成保养工单
5. 执行保养工单
6. 完成保养并沉淀记录
7. 追溯记录与稽核

### 5.1 主链路说明

- 系统管理员或生产管理员维护设备台账。
- 维护保养项目和默认周期。
- 为具体设备配置保养计划，绑定执行工段、周期、开始日期、默认执行人等信息。
- 系统按计划到期自动生成工单，也支持手工立即生成工单。
- 执行人开始工单并完成保养，填写结果摘要、异常说明和附件。
- 完成后生成保养记录，供后续查询、追溯和审计使用。

| 周期天数 | 数字输入 | 否 | 不填时默认取项目周期 |

| 预计时长 | 数字输入 | 否 | 单位分钟 |
| 开始日期 | 日期选择 | 是 | 计划起算日期 |
| 下次到期日 | 日期选择 | 否 | 可自动计算 |

| 状态 | 单选 | 否 | 默认启用 |

### 8.2.5 页面业务规则

- 同一设备 + 同一保养项目只允许存在一条有效计划。
- 执行工段必须是启用状态的有效工段。
- 周期天数必须大于 0。
- 下次到期日可由系统根据开始日期和周期自动计算。
- 停用计划后，不应继续自动生成新工单。
- 删除计划前需校验是否存在未完成工单。
- 支持手工立即生成工单。
- 支持系统按计划自动生成工单。

### 8.2.6 自动生成规则

- 系统每日按配置时间扫描到期计划并生成工单。
- 若同一计划在同一到期日已存在工单，不重复生成。
- 自动生成后应更新下次到期日。
- 自动生成过程需写日志，便于排查重复生成或漏生单问题。

| 参数名称 | 输入框 | 是 | 运行参数名称 |
| 单位 | 输入框 | 否 | 参数单位 |
| 上限 / 下限 | 数字输入 | 否 | 阈值范围 |
| 默认值 | 输入框 | 否 | 默认值 |
| 状态 | 单选 | 否 | 默认启用 |
| 备注 | 输入框 | 否 | 规则说明 |

### 13.2.2 页面业务规则

- 规则编码必须唯一。
- 停用规则后不影响历史记录，但不再用于新配置。
- 若后续接入设备采集，可在本页继续扩展告警阈值能力。

## 14. 通用业务规则

## 14.1 状态定义

### 14.1.1 设备状态

| 状态 | 含义 |
| --- | --- |
| 启用 | 可被计划和执行流程使用 |
| 停用 | 不可继续新增使用，但保留历史追溯 |

### 14.1.2 保养工单状态

| 状态 | 含义 |
| --- | --- |
| 待执行 | 工单已生成，尚未开始 |
| 执行中 | 已开始，尚未完成 |
| 已完成 | 已完成执行 |
| 已逾期 | 超过到期日仍未执行 |
| 已取消 | 已作废，不再执行 |

## 14.2 删除保护规则

- 已被计划引用的设备不允许删除。
- 已存在未完成工单的设备不允许删除。
- 已被计划引用的保养项目不允许删除。
- 已存在未完成工单的计划不允许删除。

## 14.3 自动生成规则

- 系统按配置时间自动扫描到期计划。
- 同一计划同一到期日不允许重复生成工单。
- 自动生成结果必须可记录日志和统计结果。

## 14.4 工段与权限规则

- 普通执行人默认仅能执行本人所属工段范围内的保养工单。
- 系统管理员、生产管理员可查看或执行全部工单。
- 设备模块页面和动作按钮需受权限树控制。

## 14.5 快照与追溯规则

- 工单生成时需保留来源计划、设备、项目和执行工段快照。
- 保养记录生成后需保留设备、项目、执行人和结果快照。
- 详情页、记录页应优先展示快照字段，避免主数据变更后影响历史追溯。
