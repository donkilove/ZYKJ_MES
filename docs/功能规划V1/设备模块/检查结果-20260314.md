## 4. 本轮纠偏项

相较上一版设备模块检查结果，本轮确认以下结论需要纠正：

1. **“设备规则 / 运行参数未实现”结论已失效。**
   当前代码已具备：
   - 页面目录与权限：`backend/app/core/page_catalog.py`、`backend/app/core/authz_catalog.py`
   - 前端页面：`mes_client/lib/pages/equipment_rule_parameter_page.dart`
   - 前端服务：`mes_client/lib/services/equipment_service.dart`
   - 后端模型：`backend/app/models/equipment_rule.py`、`backend/app/models/equipment_runtime_parameter.py`
   - 后端接口：`backend/app/api/v1/endpoints/equipment.py`

2. **“保养执行页缺到期日期范围、执行工段筛选”结论已失效。**
   当前前端页面已具备到期开始、到期结束、工段和状态筛选，后端接口亦支持 `due_date_start`、`due_date_end`、`stage_code` 参数。

3. **“保养记录页缺执行人 / 设备筛选”结论需要收敛为部分缺口。**
   当前前端已具备执行人关键字输入和设备下拉筛选，后端也支持执行人与设备条件查询；真正仍缺的是导出能力，以及执行人筛选未做更精确的用户选项化。

## 5. 关键风险与剩余缺口

### 5.1 P0：自动生成调度返回值解包缺陷（已修复）

- 原问题：`generate_due_work_orders_for_today` 默认返回 3 个值，调度服务按 4 个值解包，触发运行时异常。
- 修复确认：`maintenance_scheduler_service.py:75` 调用时传入 `include_new_orders=True`，服务返回 4 元组，解包正确。
- 状态：已满足。

### 5.2 已完成项：审计日志补齐

- `equipment.py:create_equipment_ledger`：新增设备台账写入 `equipment.ledger.create` 审计日志。
- `equipment.py:delete_equipment_ledger`：删除设备台账写入 `equipment.ledger.delete` 审计日志。
- `equipment.py:complete_maintenance_execution`：完成保养工单写入 `equipment.work_order.complete` 审计日志。
- `equipment.py:cancel_maintenance_execution`：取消保养工单写入 `equipment.work_order.cancel` 审计日志。

### 5.3 P1：功能完备度缺口（待处理）

1. **导出能力未落地**
   - 设备台账、保养项目、保养计划、保养记录均未实现导出接口和前端按钮。

2. **筛选条件未完全对齐需求**
   - 设备台账缺独立“状态 / 位置 / 负责人”筛选。
   - 保养项目缺“状态 / 分类”筛选。
   - 保养计划缺“启用状态 / 执行工段 / 默认执行人”筛选。

3. **保养计划列表字段仍有缺口**
   - 当前列表未展示“预计时长”，与需求文档不完全一致。

4. **设备规则 / 运行参数页面语义仍不完整**
   - 规则侧缺“规则编码”字段和唯一性约束落地。
   - 前端未提供“绑定适用设备 / 设备类型”的可视化录入入口，虽然后端已预留 `equipment_id`。
   - 运行参数侧缺“默认值”字段。
   - 运行参数当前缺“启用状态”字段与启停操作，不完全符合需求中“启用状态”口径。

5. **记录筛选交互仍可增强**
   - 执行人筛选目前为关键字输入，不是独立用户选项，后续可继续增强精确筛选体验。
