# 品质模块 V1 使用说明

## 1. 模块范围

品质模块 V1 为独立侧边栏 `quality`，仅包含 2 个页面：

1. `first_article_management`（每日首件）
2. `quality_data_query`（品质数据）

本版本不包含消息中心、不包含缺陷/报废/返修模型。

## 2. 角色可见性

默认可见角色：

1. `system_admin`
2. `quality_admin`
3. `production_admin`

默认不可见角色：

1. `operator`

## 3. 每日首件页面

页面用途：

1. 查询指定日期的首件记录
2. 按订单号/产品/工序/操作员关键字过滤
3. 查看当日校验码（只读）

接口来源：

1. `GET /api/v1/quality/first-articles`

校验码显示规则：

1. 当天且数据库有记录：显示数据库校验码，来源 `stored`
2. 当天且数据库无记录：显示系统默认码，来源 `default`
3. 非当天且数据库无记录：校验码为空，来源 `none`

## 4. 品质数据页面

页面用途：

1. 展示品质总览指标
2. 展示工序维度统计
3. 展示人员维度统计

接口来源：

1. `GET /api/v1/quality/stats/overview`
2. `GET /api/v1/quality/stats/processes`
3. `GET /api/v1/quality/stats/operators`

时间范围：

1. 默认最近 30 天（含当天）
2. 支持手动指定开始/结束日期
3. 结束日期按“含当日”口径统计

## 5. 指标口径

数据主来源：

1. `mes_first_article_record`

总览指标：

1. 首件总数 `first_article_total`
2. 通过数 `passed_total`
3. 不通过数 `failed_total`
4. 通过率 `pass_rate_percent`
5. 覆盖订单数 `covered_order_count`
6. 覆盖工序数 `covered_process_count`
7. 覆盖人员数 `covered_operator_count`
8. 最近首件时间 `latest_first_article_at`

说明：

1. API 输出状态码值为英文（如 `passed`/`failed`），前端负责中文映射。

## 6. 回归影响

本次仅新增质量模块相关页面、接口与目录可见性配置，不修改现有生产/用户/产品/设备模块业务行为。
