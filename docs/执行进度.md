### 已完成功能模块

#### 1. 用户与权限管理

- 基于 JWT 的认证与 RBAC 权限控制
- 4 个固定角色：系统管理员、生产管理员、品质管理员、操作员
- 用户管理：查询、新增、更新、删除
- 操作员可分配工序（激光打标、产品测试、产品组装、产品包装）
- 非操作员角色不允许绑定工序
- 删除操作需验证密码

#### 2. 产品管理

- 产品基础资料维护（产品名称唯一）
- 产品参数模板管理（支持 Text/Link 类型）
- 产品参数值维护与历史追溯
- 参数变更审计（记录操作人、变更内容、备注）
- 产品删除需验证管理员密码

#### 3. 设备管理

- 设备台账管理（编码、名称、型号、位置、负责人）
- 设备启用/禁用状态管理
- 设备在线状态跟踪（基于 TTL）

#### 4. 保养管理

- 保养项目管理
- 保养计划管理（周期、执行工序、计划开始日期）
- 保养工单自动生成（后台调度任务）
- 保养工单执行与记录
- 支持按工序分配执行人员

#### 5. 页面可见性配置

- 支持按角色配置页面可见性
- 动态控制前端菜单展示

### 技术特性

- 数据库自动初始化（启动时自动创建数据库和执行迁移）
- 角色与管理员账号自动初始化（工艺工段/小工序由管理端配置）
- 保养工单自动生成（可配置时间和时区）
- 统一响应格式和错误处理
- 审计日志记录

  - `quality_data_query`
- Added backend quality APIs:
  - `GET /quality/first-articles`
  - `GET /quality/stats/overview`
  - `GET /quality/stats/processes`

### Craft Module V1 Delivered (Independent Sidebar)

- Added `craft` sidebar with 2 tabs:
  - `process_management`
  - `production_process_config`
- Completed process model upgrade:
  - introduced stage (大工序/工段) + process (小工序) hierarchy
  - enabled process assignment by multiple small processes for operator users
- Completed product multi-template flow:
  - product supports multiple process templates
  - order create/edit supports template selection + step adjustment
  - supports save-as-template from order dialog
- Implemented template sync to unfinished orders:
  - `pending` orders rebuild whole route
  - `in_progress` orders rebuild only steps after current process
  - conflict returns `409` with sync summary and reasons
- Updated maintenance domain execution process semantics to stage code.
- Frontend and backend integration completed:
  - craft pages/service/models
  - production/user/registration/maintenance linked with stage+process model
