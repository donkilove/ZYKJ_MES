# 测试与验收标准（生产域迁移）

## 1. 测试总体策略
- 单元测试：覆盖 service 规则、校验分支、状态流转。
- 接口测试：覆盖权限、参数、异常码、关键响应字段。
- 前端测试：覆盖页面入口、动作按钮态、关键交互流程。
- 集成测试：覆盖“建单 -> 首件 -> 报工 -> 流转 -> 完工”主链路及新增分支。

## 2. 差异矩阵验收（每批必做）
- 依据 `01_旧新功能差异矩阵.md` 逐项标记：
  - 已迁移
  - 本批迁移
  - 后续批次
  - 不纳入
- 验收标准：
  - 不允许“无状态项”。
  - 不允许“重复开发项”。
  - 所有“本批迁移”项必须有可验证用例。

## 3. 分批验收用例
### 3.1 A 批（子订单分配 + 代班审批）
- 子订单可见量计算正确，跨角色查询一致。
- 未授权代班首件/报工返回 403。
- 代班申请审批通过后可执行对应动作，拒绝后不可执行。
- 生产管理员代理视角切换后数据范围正确。

### 3.2 B 批（并行生产）
- 并行模式开启后，按钮态与后端校验一致。
- 并行模式关闭后恢复串行逻辑。
- 并行实例状态可追溯，不出现重复实例或越序执行。

### 3.3 C 批（生产数据增强）
- 今日实时、未完工进度、手动筛选三视图统计口径一致。
- 图表与表格核心数值一致。
- 手动导出文件内容与筛选条件一致。

### 3.4 D 批（报废统计 + 维修闭环）
- 报废统计数据与报工异常数据一致。
- 维修完成满足数量守恒：报废 + 回流 = 送修。
- 回流目标校验与权限校验生效。
- 维修导出功能可用且字段完整。

## 4. 回归测试清单
- 订单管理：创建、编辑、删除、手工完工。
- 订单查询：详情、首件、报工。
- 生产统计：现有 overview/process/operator 统计接口。
- 权限边界：系统管理员、生产管理员、操作员越权均返回 403。
- 页面可见性：新增页面/Tab 能按角色配置显示或隐藏。

## 5. 非功能验收
- 接口性能：常规查询与报工操作保持秒级响应。
- 数据一致性：关键写操作事务完整，失败可回滚。
- 可追溯性：关键动作均有事件日志。

## 6. 发布验收门槛
- 每批必须满足：
  - 需求项全部通过验收用例。
  - 回归清单无阻断缺陷。
  - 接口文档与页面说明更新完成。
  - 数据迁移在目标环境演练通过。

## 7. 缺陷分级与处理
- 阻断级：主流程不可用、数据错账、权限失控，禁止上线。
- 严重级：关键功能不可用或稳定性显著下降，需修复后上线。
- 一般级：不影响主流程的体验问题，可纳入后续迭代但需记录。

## 8. 交付物要求
- 每批提交：
  - 测试报告（通过/失败用例明细）
  - 回归结果
  - 已知风险清单
  - 上线与回滚说明
