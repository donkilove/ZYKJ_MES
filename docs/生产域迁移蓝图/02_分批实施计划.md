# 分批实施计划（A -> B -> C -> D）

## 1. 执行原则
- 严格按 A -> B -> C -> D 顺序推进。
- 每批完成后必须经过：开发自测 -> 联调 -> 回归 -> 验收 -> 文档更新。
- 每批数据库变更独立 Alembic 版本，不跨批混合迁移。
- 每批可单独上线，禁止“未验收跨批叠加”。

## 2. A 批：子订单分配 + 代班审批（第一优先）
### 2.1 目标
- 补齐旧版核心协同能力，确保多角色生产分配、授权代班与代理操作闭环。

### 2.2 后端任务
- 新增代班审批实体与审批状态流转。
- 扩展 `GET /production/my-orders` 支持生产管理员代理视角。
- 新增代班申请、审批、列表接口。
- 在“首件提交/报工”链路加入授权校验：
  - 未授权代班不可操作。
  - 已授权可按目标角色进行操作并留痕。

### 2.3 前端任务
- 订单查询页新增“角色切换（代理模式）”与“申请代班”入口。
- 订单管理页新增“代班审批”入口（仅审批角色可见）。
- 操作按钮状态与后端授权态一致。

### 2.4 数据与迁移
- 新增代班审批表（申请人、代班人、目标角色、订单、工序、状态、审批人、审批时间）。
- 建立必要索引（状态、代班人、订单+工序）。

### 2.5 验收门槛
- 未授权代班报工返回 403，授权后成功。
- 审批通过/拒绝链路完整可追溯。
- 生产管理员代理视角可以切换并得到正确数据。

## 3. B 批：并行生产（Pipeline）模式
### 3.1 目标
- 提供订单级并行生产配置，控制“可开始/可结束”动作边界。

### 3.2 后端任务
- 新增订单并行模式配置接口（查询/更新）。
- 新增并行实例跟踪表，记录子订单并行序列与激活状态。
- 在报工流程增加并行规则校验。
- 扩展 my-orders 返回并行动作可用标记。

### 3.3 前端任务
- 订单管理页新增并行模式开关与配置入口。
- 订单查询页根据并行返回字段动态控制按钮。

### 3.4 数据与迁移
- 订单表新增并行开关与配置快照字段。
- 新增并行实例表（子订单、序列号、状态、时间戳）。

### 3.5 验收门槛
- 开启并行后，按钮态和后端校验一致。
- 关闭并行后恢复串行规则。
- 并行实例可追溯且无重复主键冲突。

## 4. C 批：生产数据查询增强与导出
### 4.1 目标
- 将生产数据页扩展为“今日实时 + 未完工进度 + 手动筛选导出”三视图。

### 4.2 后端任务
- 新增三类统计查询接口。
- 新增手动筛选导出接口。
- 统一统计口径与时间维度，保证图表和表格一致。

### 4.3 前端任务
- 生产数据页重构为三子视图。
- 增加筛选控件、图表渲染、导出动作与状态反馈。

### 4.4 数据与迁移
- 原则上不新增核心业务表。
- 如需导出任务追踪，可新增轻量导出记录表（可选）。

### 4.5 验收门槛
- 三视图统计值一致、空态正确。
- 导出文件与筛选条件一致。
- 页面性能可接受（常规数据量下秒级返回）。

## 5. D 批：报废统计与维修闭环
### 5.1 目标
- 建立“报工异常 -> 报废统计 -> 维修订单 -> 回流分配”的生产后段闭环。

### 5.2 后端任务
- 新增报废统计接口与导出接口。
- 新增维修订单查询、现象汇总、完成维修接口。
- 新增维修完成时的回流分配校验（数量守恒）。

### 5.3 前端任务
- 新增报废统计页面（筛选、列表、勾选导出）。
- 新增维修订单页面与“完成维修”弹窗（原因项、回流分配）。

### 5.4 数据与迁移
- 新增报废记录表。
- 新增维修订单主表、维修原因明细表、维修回流路由表。

### 5.5 验收门槛
- 报废统计口径与报工异常一致。
- 维修完成后满足“报废 + 回流 = 送修数量”。
- 维修导出正确可用。

## 6. 跨批公共任务
- 统一事件日志：所有关键动作写入订单事件日志。
- 统一权限：系统管理员、生产管理员、操作员权限边界明确。
- 统一文档：每批交付后同步更新专题文档与接口文档。

## 7. 退出与回滚策略
- 单批上线失败时，仅回滚当批接口和前端入口，不回滚已验收批次。
- 数据迁移失败时直接回滚事务，保证库结构一致。
- 每批上线前都保留“特性开关”预案，支持快速禁用新增能力。
