## 3. 差异结论
- 当前版本已具备生产域 V1 主流程，但缺少旧版的“协同控制能力”（代班、并行）与“后段闭环能力”（报废、维修）。
- 迁移重点不是重写订单核心流程，而是增量补齐四类能力：
  - 协同能力：子订单分配增强、代班审批、代理视角。
  - 控制能力：并行生产模式。
  - 数据能力：多视图统计与导出。
  - 闭环能力：报废统计与维修回流。

## 4. 批次依赖关系
- A 是 B/C/D 的前置依赖（子订单与授权模型会被后续复用）。
- B 与 C 可并行开发但按上线顺序仍为 A -> B -> C。
- D 依赖 A/B 的过程数据完整性，必须最后落地。

## 5. 不纳入项说明
- 消息中心明确不纳入：避免引入旧版消息表与读状态复杂性。
- 若后续需要消息通知，另行立项，基于事件日志派生通知，不复刻旧版方案。
