# 软件设置页设计

## 1. 背景

当前前端应用存在三个和“软件设置”直接相关的现状问题：

1. [main.dart](C:\Users\Donki\Desktop\ZYKJ_MES\frontend\lib\main.dart) 仅配置了单一亮色 `theme`，没有 `darkTheme`、`themeMode` 或应用级设置状态。
2. 项目虽然已有 [account_settings_page.dart](C:\Users\Donki\Desktop\ZYKJ_MES\frontend\lib\features\user\presentation\account_settings_page.dart)，但其职责是账号资料、密码和会话安全，不适合作为软件偏好的承载页。
3. [main_shell_scaffold.dart](C:\Users\Donki\Desktop\ZYKJ_MES\frontend\lib\features\shell\presentation\widgets\main_shell_scaffold.dart) 左侧栏当前只有业务菜单和退出登录，没有全局工具入口来承载软件设置。

与此同时，产品希望前端支持“软件级偏好”，首要诉求是：

- 明暗主题切换
- 跟随系统
- 布局偏好

并且这些偏好应当：

- 独立于登录态存在
- 在每次启动强制重新登录的产品要求下仍能保留
- 覆盖登录页、强制改密页和登录后的主壳层

## 2. 目标

1. 新增一个“全局独立入口”的软件设置页，而不是挂在个人中心中。
2. 首版支持两类偏好：`外观`、`布局偏好`。
3. 让主题模式切换在整个应用范围内即时生效。
4. 让软件偏好使用本地持久化保存，但不触碰登录态和会话恢复。
5. 保持现有前端结构风格，不引入新的大型状态管理框架。

## 3. 非目标

1. 本轮不做后端用户偏好接口，不做跨设备同步。
2. 本轮不把“通知设置”“关于软件”“实验功能”作为可用首版能力落地。
3. 本轮不改造个人中心的业务职责，不把软件设置混入账号资料或密码管理页。
4. 本轮不保存 access token、登录态、会话信息或其他任何鉴权相关数据。

## 4. 候选方案

### 方案 A：应用级设置中心

- 做法：新增应用级 `SoftwareSettingsController + SoftwareSettingsService + SoftwareSettingsPage`，把主题和软件偏好提升到 `MaterialApp` 之上统一管理。
- 优点：
  - 主题可以同时覆盖登录页、改密页和主界面
  - 软件设置与个人中心边界清晰
  - 结构最适合后续继续扩展语言、通知、实验功能
- 缺点：
  - 需要补一层应用级设置状态和本地持久化服务

### 方案 B：挂在 MainShell 内部

- 做法：在 `MainShellPage` 或 `MainShellController` 中维护软件设置状态，软件设置页仅登录后可见和生效。
- 优点：
  - 改动量较小
  - 接入现有壳层更直接
- 缺点：
  - 登录页无法享受主题切换
  - 软件设置会和主壳层业务状态耦合
  - 后续扩展时边界容易变差

### 方案 C：服务端偏好同步

- 做法：把软件偏好落到后端 API，由用户维度做跨设备同步。
- 优点：
  - 企业产品感最强
  - 多终端体验一致
- 缺点：
  - 会引入后端/API/契约/权限联动
  - 对当前首版明显过重

## 5. 推荐方案

推荐 **方案 A：应用级设置中心**。

原因：

1. 软件设置本质上是“应用级能力”，不应被挂死在 `MainShellPage` 的登录后状态里。
2. 当前产品又明确要求“每次启动都强制重新登录”，因此软件偏好和登录态必须彻底分离。
3. 这是唯一能自然覆盖登录页、改密页和主壳层的方案。
4. 它和现有“个人中心 = 账号安全”“软件设置 = 应用偏好”的心智边界最清晰。

## 6. 推荐架构

### 6.1 新增模块边界

建议新增以下单元：

- `frontend/lib/features/settings/models/software_settings_models.dart`
- `frontend/lib/features/settings/services/software_settings_service.dart`
- `frontend/lib/features/settings/presentation/software_settings_controller.dart`
- `frontend/lib/features/settings/presentation/software_settings_page.dart`

职责划分：

- `software_settings_models.dart`
  - 定义设置模型、枚举和值对象
- `software_settings_service.dart`
  - 负责本地读取、保存、恢复默认值
  - 首版使用 `shared_preferences`
- `software_settings_controller.dart`
  - 持有当前设置状态
  - 响应 UI 修改
  - 驱动自动保存和错误提示
- `software_settings_page.dart`
  - 只负责展示、交互和即时反馈

### 6.2 应用层挂载方式

设置状态提升到 `main.dart` 的 `MaterialApp` 外层：

- 应用启动时先读取本地软件设置
- 使用读取结果初始化应用级 `SoftwareSettingsController`
- `MaterialApp` 直接监听该 controller
- 用它驱动：
  - `theme`
  - `darkTheme`
  - `themeMode`
  - 全局密度相关样式

这样登录页、强制改密页、主壳层都共享同一份软件偏好。

### 6.3 与现有页面的职责分工

- 个人中心：
  - 个人资料
  - 修改密码
  - 当前会话
- 软件设置：
  - 主题模式
  - 界面密度
  - 默认进入页
  - 侧边栏默认状态

二者不互相侵入。

## 7. 入口与导航设计

软件设置应作为 **主壳层左侧底部的全局工具入口**，位于“退出登录”上方。

入口特征：

- 图标：齿轮
- 名称：`软件设置`
- 不纳入业务菜单排序和页面可见权限体系
- 进入后仍使用现有主壳层右侧内容区展示

原因：

1. 它不是业务模块，不应和用户、生产、质量等菜单并列。
2. 它也不是账号安全页，不应塞进用户模块页签。
3. 放在全局工具区最符合桌面软件用户心智。

## 8. 页面信息架构

页面采用 **左侧分区导航 + 右侧内容区** 的双栏结构。

### 8.1 左侧分区导航

首版仅开放两个分区：

- `外观`
- `布局偏好`

后续可扩展但本轮不开放：

- 通知
- 实验功能
- 关于软件

### 8.2 右侧页面框架

右侧内容区统一包含：

- 页面标题：`软件设置`
- 简短说明：说明这是“影响本机软件体验的偏好项”
- `恢复默认` 按钮
- 自动保存状态提示：
  - 已自动保存
  - 保存失败，请重试

### 8.3 外观分区字段

首版开放：

- `主题模式`
  - `浅色`
  - `深色`
  - `跟随系统`
- `界面密度`
  - `舒适`
  - `紧凑`
- `预览卡片`
  - 实时展示当前主题与密度效果

首版不开放：

- 自定义主题色
- 自定义字号
- 圆角风格

### 8.4 布局偏好分区字段

首版开放：

- `启动后默认进入`
  - `首页`
  - `上次停留模块`
- `侧边栏默认状态`
  - `展开`
  - `折叠`

首版不开放：

- 自定义菜单顺序
- 模块收藏
- 拖拽排序
- 复杂页面记忆规则

## 9. 状态流与持久化策略

### 9.1 读取时机

- 应用启动时读取本地软件偏好
- 若读取成功，用读取值初始化应用级设置状态
- 若读取失败，回退到默认值启动应用

### 9.2 写入时机

- 用户每次修改设置后，立即更新内存态
- 同时立即尝试写入本地持久化
- 不设置显式“保存”主按钮

### 9.3 即时生效边界

- `主题模式`：立即刷新整个应用主题
- `界面密度`：立即刷新全局列表、卡片、间距相关样式
- `默认进入页`：影响下次登录后进入主壳层的落点，不强制打断当前页
- `侧边栏默认状态`：当前会话立即同步，后续启动按该默认值生效

### 9.4 默认值

默认值如下：

- 主题模式：`跟随系统`
- 界面密度：`舒适`
- 启动后默认进入：`首页`
- 侧边栏默认状态：`展开`

### 9.5 登录态隔离原则

软件设置只保存“软件偏好”，不保存：

- 登录态
- access token
- refresh token
- 当前用户会话
- 自动恢复登录信息

这条边界必须写死，避免与“每次启动强制重新登录”的产品要求冲突。

## 10. 异常处理

### 10.1 读取失败

- 应用继续启动
- 自动使用默认值
- 不阻塞登录页和主壳层展示

### 10.2 保存失败

- 当前 UI 先保留用户刚刚选择的结果
- 设置页顶部展示轻量错误提示：
  - `设置保存失败，本次重启后可能不会保留`

### 10.3 非法配置或旧版本配置

- 发现非法值时自动回退到默认值
- 不允许因旧配置导致应用崩溃

### 10.4 恢复默认

- 仅重置软件偏好
- 不影响账号、密码、登录态和业务数据

## 13. 验收标准

1. 应用提供独立的软件设置入口。
2. 软件设置首版支持：
   - 主题模式
   - 界面密度
   - 默认进入页
   - 侧边栏默认状态
3. 主题切换覆盖登录页、改密页和主壳层。
4. 软件偏好重启后仍可恢复。
5. 软件偏好与登录态彻底分离。
6. 首版不引入后端接口依赖。

## 14. 风险与控制

### 风险 1：主题切换会放大回归面

- 控制：先把主题状态接到应用层，再逐步修正页面在深色模式下的显示问题

### 风险 2：全局密度切换会影响表格与复杂页面布局

- 控制：首版仅做“舒适 / 紧凑”两档，不引入自定义字号和更细粒度间距

### 风险 3：设置页与个人中心边界混淆

- 控制：入口位置和页面命名明确区分，软件设置不进入用户模块页签

### 风险 4：本地持久化异常导致体验不稳定

- 控制：所有读取失败都回退默认值，所有写入失败都走轻量提示，不阻塞主流程
