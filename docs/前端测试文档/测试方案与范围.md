# 前端测试方案与范围

## 1. 目标
- 对 `mes_client/lib` 前端代码建立可持续执行的自动化测试。
- 测试文件统一存放在 `mes_client/test`。
- 覆盖核心业务模型、服务调用、通用组件，确保代码覆盖率不低于 80%。

## 2. 测试范围

### 2.1 模型层（`mes_client/lib/models`）
- 覆盖全部模型文件：
  - `app_session.dart`
  - `craft_models.dart`
  - `current_user.dart`
  - `equipment_models.dart`
  - `page_catalog_models.dart`
  - `production_models.dart`
  - `product_models.dart`
  - `quality_models.dart`
  - `user_models.dart`
- 覆盖点：
  - `fromJson` 字段映射与默认值回退。
  - `toJson` 序列化结构。
  - 状态/来源文案函数分支。
  - 关键 getter（如展示名拼接）。

### 2.2 服务层（`mes_client/lib/services`）
- 覆盖全部服务文件：
  - `api_exception.dart`
  - `auth_service.dart`
  - `craft_service.dart`
  - `equipment_service.dart`
  - `page_catalog_service.dart`
  - `production_service.dart`
  - `product_service.dart`
  - `quality_service.dart`
  - `session_store.dart`
  - `user_service.dart`
- 覆盖点：
  - GET/POST/PUT/DELETE 请求路径、查询参数、请求体拼装。
  - 成功分支数据解析。
  - 失败分支异常抛出（`ApiException`）及错误消息提取。
  - 特殊分支（如空响应、无数据、非 JSON 错误响应）。

### 2.3 通用组件层（`mes_client/lib/widgets`）
- 覆盖：
  - `adaptive_table_container.dart`
  - `locked_form_dialog.dart`
- 覆盖点：
  - 组件渲染结构。
  - 键盘 `Esc` 行为锁定与显式按钮关闭流程。

## 3. 不在本轮范围
- 大型业务页面（`mes_client/lib/pages/*`）的端到端交互场景未在本轮做全量 Widget/UI 自动化。
- 本轮重点是高稳定性单元测试与服务契约测试，优先支撑日常回归与接口改动检测。

## 4. 目录规范
- 测试根目录：`mes_client/test`
- 子目录：
  - `mes_client/test/models`
  - `mes_client/test/services`
  - `mes_client/test/widgets`
  - `mes_client/test/support`

## 5. 维护要求
- 新增前端模型或服务时，必须同步新增对应测试。
- 服务层新增接口时，至少补充：
  - 1 条成功路径。
  - 1 条失败路径。
  - 关键参数断言（query/body/header）。
