## 1. 概述

### 1.1 业务背景

MES 系统中的生产订单流转是核心业务流程，涉及从订单创建到完工的完整生命周期管理。系统需要支持：

- 多工序顺序生产
- 并行生产模式（上一工序未完成时，下一工序可提前开始）
- 精确的数量控制和传递
- 角色权限隔离
- 实时消息通知

### 1.2 核心概念

| 术语 | 说明 |
|------|------|
| 订单 (Order) | 生产任务的基本单位，包含产品、数量、工序等信息 |
| 工序 (Process) | 订单生产的一个步骤，如激光打标、程序烧录等 |
| 子订单 (Sub-order) | 订单在特定工序的分解，用于并行生产管理 |
| 可见数量 (Visible Quantity) | 某工序角色可以看到的订单数量 |
| 并行生产 | 多个工序同时进行的生产模式 |

## 3. 并行生产模式详解

### 3.1 什么是并行生产

并行生产是指订单的多个工序可以同时进行的生产模式。当上一工序完成一定数量（但未完成全部）并结束生产时，下一工序可以提前开始生产，无需等待上一工序全部完成。

- 工序：激光打标 → 程序烧录 → 程序版本读取

### 3.3 数量传递规则

| 场景 | 上一工序状态 | 下一工序可见数量 | 说明 |
|------|-------------|-----------------|------|
| 初始状态 | 待生产 | 0 | 订单刚创建 |
| 生产中 | 生产中（未完成） | 上一工序已完成数量 | 并行生产模式 |
| 生产完成 | 生产完成 | 订单总数量 | 上一工序全部完成 |

**数量计算逻辑（伪代码）：**

```python
def get_visible_quantity(order, process):
    """
    获取工序可见数量
    """
    if process.order == 1:
        # 第一道工序：始终看到订单总量
        return order.total_quantity
    else:
        # 其他工序：获取上一工序的完成数量
        prev_process = get_previous_process(order.id, process.order)
        
        if prev_process.status == '生产完成':
            # 上一工序已完成：看到订单总量
            return order.total_quantity
        else:
            # 上一工序未完成：看到上一工序的实际完成数量
            return prev_process.visible_quantity
```

## 4. 角色可见性规则

### 4.1 可见性判定逻辑

#### 4.1.1 订单状态 = "待生产"

**可见角色：**
- 第一个工序对应的角色（订单刚创建）
- 上一工序已生产一定数量时，下一工序对应的角色（并行生产）

**按钮显示：**
- 无首件记录 → "首件检验"
- 有首件记录 → "结束生产"

#### 4.1.2 订单状态 = "生产中"

**可见角色：**
- 所有状态为"进行中"或"生产中"的工序对应的角色

**按钮显示：**
- status = "进行中" → "结束生产"
- status = "生产中" → "再次首件"

#### 4.1.3 订单状态 = "生产完成"

**可见角色：**
- 无（对所有生产角色隐藏）

**说明：**
- 订单全部工序已完成
- 只有管理员可在历史记录中查看

### 4.2 可见性判定算法（伪代码）

```python
def is_order_visible_to_role(order, user_role):
    """
    判断订单是否对某角色可见
    """
    processes = get_order_processes(order.id)
    
    if order.status == '待生产':
        # 待生产状态：检查所有待生产的工序
        for process in processes:
            if process.operator == user_role:
                if process.status == '待生产':
                    # 计算可见数量
                    if process.order == 1:
                        visible_quantity = order.total_quantity
                    else:
                        visible_quantity = process.visible_quantity
                    
                    # 只有可见数量 > 0 时才可见
                    if visible_quantity > 0:
                        return True
                
                elif process.status == '生产中':
                    # 工序处于生产中状态（可再次首件），可见
                    return True
    
    elif order.status == '生产中':
        # 生产中状态：检查所有进行中或生产中的工序
        for process in processes:
            if process.operator == user_role:
                if process.status in ['进行中', '生产中']:
                    return True
                elif process.status in ['生产完成', '已完成']:
                    # 当前角色的工序已完成，隐藏订单
                    return False
    
    # 生产完成状态或无匹配工序
    return False
```

## 6. 关键业务逻辑实现

### 6.1 订单创建

```python
def add_order_with_processes(order_data, processes):
    """
    创建订单并初始化所有工序
    
    Args:
        order_data: 订单基本信息
        processes: 工序列表，包含工序名称、角色、顺序等
    
    Returns:
        订单 ID
    """
    with db.transaction():
        # 1. 创建订单主记录
        order_id = db.execute(
            """INSERT INTO orders (order_code, product_id, quantity, status, 
                                   start_date, end_date, operator, created_at)
               VALUES (%s, %s, %s, '待生产', %s, %s, %s, %s)
               RETURNING order_id""",
            (order_data.order_code, order_data.product_id, order_data.quantity,
             order_data.start_date, order_data.end_date, 
             order_data.operator, datetime.now())
        )
        
        # 2. 创建工序记录
        for i, process in enumerate(processes):
            # 第一道工序的可见数量设置为订单总量
            visible_qty = order_data.quantity if i == 0 else 0
            
            db.execute(
                """INSERT INTO order_processes 
                   (order_id, process_name, status, operator, process_order, 
                    visible_quantity, created_at)
                   VALUES (%s, %s, '待生产', %s, %s, %s, %s)""",
                (order_id, process.name, process.operator, i + 1, 
                 visible_qty, datetime.now())
            )
            
            # 3. 创建工序状态记录
            db.execute(
                """INSERT INTO process_status 
                   (order_id, process_name, status, product_quantity, created_at)
                   VALUES (%s, %s, '待生产', 0, %s)""",
                (order_id, process.name, datetime.now())
            )
        
        # 4. 清除缓存
        clear_orders_cache()
        
        return order_id
```

### 6.2 首件检验

```python
def first_article_inspection(order_id, process_name, operator, validation_code):
    """
    首件检验
    
    Args:
        order_id: 订单 ID
        process_name: 工序名称
        operator: 操作员
        validation_code: 校验码
    
    Returns:
        是否成功
    """
    with db.transaction():
        # 1. 验证校验码
        if not validate_code(validation_code):
            raise ValueError("校验码错误")
        
        # 2. 更新工序状态
        db.execute(
            """UPDATE order_processes 
               SET status = '进行中', updated_at = %s
               WHERE order_id = %s AND process_name = %s""",
            (datetime.now(), order_id, process_name)
        )
        
        # 3. 更新工序状态表
        db.execute(
            """UPDATE process_status 
               SET status = '进行中', ongoing_users = %s, updated_at = %s
               WHERE order_id = %s AND process_name = %s""",
            (json.dumps([operator]), datetime.now(), order_id, process_name)
        )
        
        # 4. 创建首件生产记录
        db.execute(
            """INSERT INTO production_records 
               (order_id, process_name, operator, production_quantity, 
                record_type, created_at)
               VALUES (%s, %s, %s, 1, '首件', %s)""",
            (order_id, process_name, operator, datetime.now())
        )
        
        # 5. 清除缓存
        clear_orders_cache()
        
        return True
```

### 6.3 结束生产

```python
def end_production(order_id, process_name, operator, production_quantity):
    """
    结束生产
    
    Args:
        order_id: 订单 ID
        process_name: 工序名称
        operator: 操作员
        production_quantity: 本次生产数量
    
    Returns:
        是否成功
    """
    with db.transaction():
        # 1. 获取当前工序信息
        process = db.fetch_one(
            """SELECT * FROM order_processes 
               WHERE order_id = %s AND process_name = %s""",
            (order_id, process_name)
        )
        
        # 2. 获取订单总量
        order = db.fetch_one(
            "SELECT quantity FROM orders WHERE order_id = %s",
            (order_id,)
        )
        total_quantity = order.quantity
        
        # 3. 更新工序状态表（累加生产数量）
        process_status = db.fetch_one(
            """SELECT product_quantity FROM process_status 
               WHERE order_id = %s AND process_name = %s""",
            (order_id, process_name)
        )
        current_quantity = process_status.product_quantity
        new_quantity = current_quantity + production_quantity
        
        db.execute(
            """UPDATE process_status 
               SET product_quantity = %s, 
                   status = '生产中',
                   updated_at = %s
               WHERE order_id = %s AND process_name = %s""",
            (new_quantity, datetime.now(), order_id, process_name)
        )
        
        # 4. 更新 order_processes 表
        if new_quantity >= total_quantity:
            # 工序完成
            db.execute(
                """UPDATE order_processes 
                   SET status = '生产完成', updated_at = %s
                   WHERE order_id = %s AND process_name = %s""",
                (datetime.now(), order_id, process_name)
            )
        else:
            # 工序未完成，可再次首件
            db.execute(
                """UPDATE order_processes 
                   SET status = '生产中', updated_at = %s
                   WHERE order_id = %s AND process_name = %s""",
                (datetime.now(), order_id, process_name)
            )
        
        # 5. 更新下一工序的可见数量（实现并行生产）
        next_process = db.fetch_one(
            """SELECT * FROM order_processes 
               WHERE order_id = %s AND process_order = %s""",
            (order_id, process.process_order + 1)
        )
        
        if next_process:
            # 更新下一工序的可见数量为当前工序的完成数量
            db.execute(
                """UPDATE order_processes 
                   SET visible_quantity = %s, updated_at = %s
                   WHERE order_id = %s AND process_name = %s""",
                (new_quantity, datetime.now(), order_id, next_process.process_name)
            )
            
            # 发送消息通知下一工序的操作员
            _notify_next_process_operator(
                order_id, next_process.process_name, 
                next_process.operator, new_quantity
            )
        
        # 6. 创建生产记录
        db.execute(
            """INSERT INTO production_records 
               (order_id, process_name, operator, production_quantity, 
                record_type, created_at)
               VALUES (%s, %s, %s, %s, '正常生产', %s)""",
            (order_id, process_name, operator, production_quantity, datetime.now())
        )
        
        # 7. 检查订单是否全部完成
        _check_order_completion(order_id)
        
        # 8. 清除缓存
        clear_orders_cache()
        
        return True
```

```python
def _notify_next_process_operator(order_id, process_name, operator_role, quantity):
    """
    通知下一工序操作员
    
    Args:
        order_id: 订单 ID
        process_name: 工序名称
        operator_role: 操作员角色
        quantity: 可生产数量
    """
    # 1. 获取订单信息
    order = db.fetch_one(
        """SELECT order_code, product_id FROM orders WHERE order_id = %s""",
        (order_id,)
    )
    
    # 2. 获取产品信息
    product = db.fetch_one(
        "SELECT product_name FROM products WHERE id = %s",
        (order.product_id,)
    )
    
    # 3. 获取该角色的所有用户
    users = db.fetch_all(
        """SELECT username FROM users 
           WHERE role_name = %s AND status = 'active'""",
        (operator_role,)
    )
    
    # 4. 为每个用户创建消息
    for user in users:
        message_title = f"新订单流转 - {order.order_code}"
        message_content = (
            f"订单 {order.order_code} ({product.product_name}) "
            f"已流转到工序：{process_name}\n"
            f"当前可生产数量：{quantity} 个\n"
            f"请尽快进行首件检验并开始生产。"
        )
        
        db.execute(
            """INSERT INTO messages 
                (sender, recipient, title, content, message_type, 
                 related_order_id, is_read, created_at) 
                VALUES (%s, %s, %s, %s, %s, %s, %s, %s)""",
            ("system", user.username, message_title, message_content, 
             "order_flow", order_id, False, datetime.now())
        )
```

### 6.5 订单完成检查

```python
def _check_order_completion(order_id):
    """
    检查订单是否全部完成
    
    Args:
        order_id: 订单 ID
    """
    # 1. 获取所有工序
    processes = db.fetch_all(
        """SELECT * FROM order_processes 
           WHERE order_id = %s ORDER BY process_order""",
        (order_id,)
    )
    
    # 2. 检查是否所有工序都已完成
    all_completed = all(
        p.status in ['生产完成', '已完成'] 
        for p in processes
    )
    
    if all_completed:
        # 3. 更新订单状态
        db.execute(
            """UPDATE orders 
               SET status = '生产完成', updated_at = %s
               WHERE order_id = %s""",
            (datetime.now(), order_id)
        )
        
        # 4. 通知生产管理员
        order = db.fetch_one(
            "SELECT order_code FROM orders WHERE order_id = %s",
            (order_id,)
        )
        
        admins = db.fetch_all(
            """SELECT username FROM users 
               WHERE role_name = '生产管理员' AND status = 'active'"""
        )
        
        for admin in admins:
            db.execute(
                """INSERT INTO messages 
                    (sender, recipient, title, content, message_type, 
                     related_order_id, is_read, created_at) 
                    VALUES (%s, %s, %s, %s, %s, %s, %s, %s)""",
                ("system", admin.username, 
                 f"订单完成通知 - {order.order_code}",
                 f"订单 {order.order_code} 已全部工序完成",
                 "order_complete", order_id, False, datetime.now())
            )
```

```python
def get_action_button(order, process_status, product_quantity, total_quantity):
    """
    根据状态获取操作按钮
    
    Returns:
        按钮类型："首件检验"、"结束生产"、"再次首件" 或 None（隐藏）
    """
    if product_quantity >= total_quantity:
        return None  # 数量已完成，隐藏按钮
    
    if process_status == "进行中":
        return "结束生产"  # 首件已完成，可结束生产
    
    if process_status == "生产中":
        return "再次首件"  # 可再次首件继续生产
    
    if process_status == "待生产":
        if 有首件记录:
            return "结束生产"  # 已首件但未开始生产
        else:
            return "首件检验"  # 首次首件
    
    return None
```

## 8. 异常处理

### 8.1 可能的异常情况

| 异常场景 | 处理方式 |
|---------|---------|
| 上一工序未完成，下一工序提前开始 | 允许，但数量按实际完成数显示 |
| 多个工序同时"进行中" | 允许并行生产 |
| 生产数量超过订单总数量 | 系统限制，不允许超过 |
| 校验码错误 | 提示重新输入 |
| 网络中断导致操作失败 | 本地缓存操作，网络恢复后重试 |
| 并发操作冲突 | 使用数据库事务和乐观锁 |

### 8.2 数据一致性保障

1. **数据库事务**：所有关键操作使用事务确保原子性
2. **缓存失效**：数据变更后立即清除相关缓存
3. **操作日志**：记录所有操作的详细日志
4. **定期校验**：定期检查订单数量一致性
5. **并发控制**：使用乐观锁防止并发冲突
