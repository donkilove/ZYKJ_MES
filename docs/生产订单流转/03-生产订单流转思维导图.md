## 2. 主流程图

```mermaid
flowchart TD
    A[创建生产订单] --> B[订单 pending]
    B --> C[首工序可见<br/>后续工序不可见]
    C --> D{操作端查询}
    D -->|我的工单/代理工单| E[按 visible_quantity + 子工单可见性展示]
    D -->|我的代班工单| F[按 approved 且未 consumed 的代班授权展示]

    E --> G{开始首件}
    F --> G
    G -->|不通过| H[保留首件记录<br/>状态不推进]
    G -->|通过| I[工序 pending 时推进 in_progress<br/>当前操作员子工单 in_progress<br/>订单 in_progress<br/>其他操作员子工单保持 pending]

    I --> J{结束生产}
    J -->|数量合法| K[累计 completed_quantity]
    K --> L{当前工序是否完成整单}
    L -->|否| M[工序 partial]
    L -->|是| N[工序 completed]

    M --> O[子工单 done 或回到 pending]
    N --> O
    O --> P[按累计完成量放行下工序 visible_quantity]
    P --> Q[同步下工序子工单 assigned_quantity / is_visible]
    Q --> R{有缺陷项?}
    R -->|是| S[自动创建维修单]
    R -->|否| T[继续下一轮执行]
    S --> T

    T --> U{所有工序完成?}
    U -->|否| D
    U -->|是| V[订单 completed]

    B --> W{手工完工}
    I --> W
    M --> W
    N --> W
    W --> X[校验当前密码]
    X --> X1{有 in_repair 维修单?}
    X1 -->|是| X2[拒绝完工 409<br/>提示先收维修单]
    X1 -->|否| Y[强制所有工序/子工单 completed]
    Y --> Y1[级联收口<br/>首件会话 cancelled<br/>代班授权 consumed<br/>并行实例失活]
    Y1 --> V
```

## 3. 维修单完成回流流程

```mermaid
flowchart TD
    M1[维修单完成] --> M2{有可修件?}
    M2 -->|是| M3[目标工序 visible_quantity 增加]
    M3 --> M4{来源 != 目标?}
    M4 -->|是| M5[来源工序 visible_quantity 压低<br/>不低于 completed_quantity]
    M4 -->|否| M9[订单状态按 completed vs visible 重算]
    M5 --> M9
    M2 -->|否| M6[全部报废]
    M1 --> M7{报废且 scrap_replenished?}
    M7 -->|是| M8[首工序 visible_quantity 增加报废量]
    M8 --> M9
    M7 -->|否| M9
    M6 --> M9
```
