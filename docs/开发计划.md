# MES 开发计划（初版）

## 1. 计划前提

- 技术栈：Flutter + Python FastAPI + PostgreSQL。
- 协作方式：前后端并行开发，接口先行。
- 版本节奏：双周迭代（2 周一个 Sprint）。

## 2. 里程碑

| 阶段 | 时间建议 | 目标 | 交付物 |
| --- | --- | --- | --- |
| M0 启动阶段 | 第 1 周 | 完成架构与规范定稿 | 技术方案、数据库草案、API草案 |
| M1 核心基础 | 第 2-3 周 | 完成认证、用户、权限、日志基础能力 | 登录鉴权、用户管理 API 与前端页面 |
| M2 产品域 | 第 4-5 周 | 完成产品与产品参数管理 | 产品与参数管理全流程 |
| M3 订单域 | 第 6-7 周 | 完成生产订单管理与查询 | 订单管理、订单查询、状态流转 |
| M4 首件域 | 第 8-9 周 | 完成首件提报审核闭环 | 首件管理流程、追溯查询 |
| M5 稳定发布 | 第 10 周 | 联调、测试、上线准备 | 测试报告、上线清单、操作手册 |

## 3. Sprint 切分建议

### Sprint 1（第 1-2 周）

- 项目脚手架初始化（前后端仓库结构）。
- 登录鉴权（JWT）与权限中间件。
- 用户管理基础 CRUD。
- Flutter 登录页与用户管理页初版。

### Sprint 2（第 3-4 周）

- 产品管理 API 与页面。
- 产品参数模板、参数项 API 与页面。
- 数据校验、唯一约束、基础审计日志。

### Sprint 3（第 5-6 周）

- 生产订单创建、下发、状态流转。
- 订单查询条件组合与分页。
- 关键接口性能优化（索引、分页策略）。

### Sprint 4（第 7-8 周）

- 首件提报、审核、驳回重提流程。
- 首件记录与订单关联追溯。
- 前后端联调与回归测试。

### Sprint 5（第 9-10 周）

- 缺陷修复与体验优化。
- 上线脚本与部署文档完善。
- UAT 验收与发布。

## 4. 验收标准

- 功能验收：MVP 范围内功能可完整演示。
- 质量验收：关键模块具备单元测试，主流程具备集成测试。
- 运维验收：可按文档独立部署，日志可追踪错误。

## 5. 风险与应对

- 需求变动频繁：冻结每个 Sprint 范围，变更走评审。
- 接口反复调整：先冻结 API 契约，再并行开发。
- 进度偏差：每周例会跟踪燃尽，必要时拆分低优先级需求。
