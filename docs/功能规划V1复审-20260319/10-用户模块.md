# 用户模块复审

## 1. 当前总体判断

- 当前用户模块已具备较完整的前后端闭环。
- 上一轮两项阻断问题“自定义角色无法真正分配”“系统管理员权限配置保底规则缺失”均已修复。
- 但整体仍未达到需求验收级别，主要卡在工段实时查询、完整在线/离线会话视图、会话失效处理闭环。

## 2. 已满足项

- 7 个页面均已落地并纳入统一用户模块页签容器。
- 用户管理已支持搜索、筛选、新建、编辑、启停、逻辑删除、重置密码、导出。
- 用户规则已落实：用户名唯一、仅系统管理员可改用户名、操作员必须绑定工段、非操作员不可绑定工段。
- 注册审批已支持 `pending/approved/rejected` 查询、通过、驳回、驳回原因、审批通过设置初始密码。
- 首次登录强制改密已打通。
- 角色管理、审计日志、个人中心、登录日志、在线会话均已有前后端实现。
- 系统管理员权限配置保底规则已落地。

## 3. 部分满足项

- 工段“实时查询”仅做到页面初始化预加载，不是弹窗级实时拉取。
- 在线会话页面虽已显示“在线”文案，但接口仍只返回活跃会话，无法形成完整在线/离线视图。
- 个人中心会话失效处理仍不够闭环，定时刷新异常被静默处理。

## 4. 未满足项

- 未真正满足“工段数据实时查询方式获取”。
- 未满足“在线会话状态应与系统登录状态保持一致”的完整展示要求。
- 未完全满足“会话已过期时提示并跳转登录页”。

## 5. 相比上一轮已修复项

- 自定义角色已可在用户新建、编辑、注册审批通过中真正分配。
- 系统管理员固定保留权限配置页面访问与保存能力。
- 非系统管理员已被禁止配置 system 模块能力。
- 注册审批页已支持历史状态筛选。
- 用户删除提示语义已与后端逻辑删除一致。

## 6. 仍遗留的关键问题

- 工段引用数据不是实时查询。
- 在线会话仍缺离线/过期统一视图。
- 个人中心对会话失效的处理不够闭环。
