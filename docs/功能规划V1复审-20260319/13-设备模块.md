# 设备模块复审

## 1. 当前总体判断

- 设备模块“台账-项目-计划-执行-记录”的主闭环后端基本存在，前端页面也覆盖主要入口。
- 但上一轮指出的多个确定性缺陷仍基本存在，当前仍不能按需求完整验收。

## 2. 已满足项

- 设备台账已支持列表、搜索、状态筛选、新增、编辑、启停、删除、详情查看、导出接口。
- 已实现设备编号唯一校验、删除前计划引用校验、未完成工单校验。
- 保养项目、保养计划、保养执行、保养记录的基本页面与后端链路均已存在。
- 自动生成工单链路仍然存在。
- 保养记录模型仍保留来源计划/设备/项目快照字段。

## 3. 部分满足项

- 设备详情、保养执行详情仍是轻量弹窗，不是完整子页面。
- 保养记录页“执行人筛选”只是并入关键字，不是独立筛选参数。
- 保养计划页缺执行工段/默认执行人筛选，列表列也不完整。
- 设备规则/运行参数页仍只是基础雏形。
- 导出更多是弹窗展示 CSV 文本，不是正式导出体验。

## 4. 未满足项

- 设备台账缺位置筛选、负责人筛选。
- 保养项目缺项目分类筛选。
- 保养记录缺附件列与独立执行人筛选。
- 设备规则/运行参数页面未达到需求要求。
- 设备台账必填规则仍与需求相反。
- 前端 `equipment_service.dart` 中保养计划请求体仍有确定性语法问题。
- 设备规则启停接口前后端字段仍不一致且后端读错字段。
- 工单列表/详情仍未优先使用快照字段。

## 5. 相比上一轮已修复项

- 未发现上一轮关键确定性缺陷已明确修复的证据。

## 6. 仍遗留的关键问题

- 前端保养计划请求体语法错误。
- 设备规则启停接口不通。
- 追溯展示未优先使用快照字段。
- 设备规则/运行参数、设备详情、执行详情仍未达验收范围。
