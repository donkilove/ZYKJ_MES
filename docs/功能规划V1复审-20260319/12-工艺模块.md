# 工艺模块复审

## 1. 当前总体判断

- 工艺模块较上一轮已有明显收敛，重点整改项中轻量查询、步骤扩展字段、来源留痕、已发布模板编辑限制、历史引用删除保护、模板/版本导出均已落地。
- 当前可判定为“大部分满足”，但仍未完全到验收完态。

## 2. 已满足项

- 已提供工段/工序轻量查询接口。
- 模板步骤、系统母版步骤、版本步骤均支持 `标准工时/关键工序/步骤说明`。
- 模板复制、跨产品复制、系统母版套版均已记录来源元数据。
- 已发布/已归档模板不可直接编辑，需创建草稿后再改。
- 工段/工序删除保护与引用分析已纳入历史版本引用。
- 已补单模板导出、模板版本导出、看板导出等能力。
- 后端已有 craft 集成测试，前端已有 service/model 测试。

## 3. 部分满足项

- 引用跳转已能切模块/切工艺 Tab，但还不能精确落到具体记录。
- 系统母版主区域未直接完整展示步骤列表。
- 回滚前仅有确认，不像发布前那样先展示完整影响分析结果。
- 前端关键页面交互仍缺 widget/页面级测试。

## 4. 未满足项

- 未看到按 `jump_target` 精确落到来源对象详情的完整实现。
- 回滚前影响分析未独立展示。
- 关键页面级前端测试仍缺失。

## 5. 相比上一轮已修复项

- 已修复“无轻量查询接口”。
- 已修复“历史版本引用未纳入删除保护与引用分析”。
- 已修复“模板复制/套版未记录来源”。
- 已修复“步骤扩展字段未落地”。
- 已修复“单模板导出与模板版本导出缺失”。
- 已修复“已发布模板仍可直接编辑当前实体”。

## 6. 仍遗留的关键问题

- 跳转未打通到记录级。
- 回滚链路缺少发布级别的影响分析展示。
