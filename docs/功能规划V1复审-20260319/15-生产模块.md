# 生产模块复审

## 1. 当前总体判断

- 生产模块已从“主链路可用但关键缺口较多”提升到“主链路和大部分重点整改已落地，仍有少量关键收尾问题”。
- 上一轮点名的三项核心问题：数量口径、删除留痕、多回流分配，本轮代码均已明显整改到位。

## 2. 已满足项

- 报工数量口径已闭环：`quantity + defect_quantity` 统一参与可生产量校验。
- 删除订单后事件日志可在数据库层保留，日志表已补订单/产品/工序快照字段。
- 维修完成已支持多回流分配，前后端均有数量合计校验。
- 维修订单导出已带时间范围并走本地保存文件流程。
- 订单详情、执行详情、报废详情、维修详情、并行实例、事件日志基础能力均已存在。
- 代班状态与报废进度中文映射已修正。

## 3. 部分满足项

- 订单导出仍主要是弹窗预览，不是正式文件导出闭环。
- 维修详情、报废详情仍为弹窗而非独立子页面。
- 事件日志虽保存了快照，但前端详情页尚未完整展示这些结构化快照。
- 并行实例追踪仍偏技术字段导向。

## 4. 未满足项

- 删除后的“可追溯”目前只完成到数据库保留层，缺少业务侧可直接查询的入口。

## 5. 相比上一轮已修复项

- 已修复报工数量与异常数量口径不闭环。
- 已修复删除订单后事件日志被级联删除。
- 已修复前端不支持多回流分配。
- 已修复维修导出未带时间范围。
- 已修复代班/报废核心文案映射错误。

## 6. 仍遗留的关键问题

- 删除追溯缺少用户可访问的查询入口。
- 订单导出仍未形成真正文件导出闭环。
- 事件快照虽已入库，但前端未完整展示。
