# 产品模块复审

## 1. 当前总体判断

- 产品模块较上一轮已明显改善，版本参数主模型、复制/生效闭环、删除保护与测试均已有实质落地。
- 但仍不能按需求完整验收，主要问题集中在版本文案不一致、查询/导出与当前筛选不完全一致、详情变更记录不完整，以及“重新启用产品可能绕过正式生效校验”。

## 2. 已满足项

- 版本参数主模型已落地，后端已引入 `ProductRevisionParameter`，参数查询、编辑、导出都可按版本读取。
- 新建产品自动生成 `V1.0` 草稿版本。
- 复制/生效主链路已基本闭环：复制基于版本参数生成新草稿，生效会同步当前参数并失效旧版本。
- 已加入 `expected_effective_version` 并发保护。
- 当前生效参数查询、版本参数导出已基于版本/生效版本快照。
- 参数历史已支持 `revision_id`、`version`、`change_type`、前后快照、修改人、时间等信息。
- 删除保护已扩展到生产工单、报废统计、维修单、不良现象、维修原因等引用。
- 后端集成测试与前端回归测试均已补入。

## 3. 部分满足项

- 历史追溯更多偏参数历史，产品详情中的“变更记录”仍不是完整统一变更视图。
- 查询/导出已明显改善，但前端导出仍未完整带上当前所有筛选条件。
- 删除保护已增强，但未见对质检记录等更广泛引用源的显式覆盖。

## 4. 未满足项

- 前端多个页面的版本号展示与筛选仍未完全统一到 `version_label` 语义。
- 查询页、参数管理页、产品列表导出都未完全做到“所见即所导”。
- 产品详情“变更记录区”仍未覆盖产品编辑、启停、生效等全量动作。
- 产品重新启用时仍可能绕过正式版本生效流程。

## 5. 相比上一轮已修复项

- “参数未绑定具体版本”的核心问题已修复。
- “复制版本污染当前参数表”的问题已修复。
- “生效缺少并发冲突校验”的问题已修复。
- “删除保护只覆盖未完成工单”的问题已明显改善。
- 测试薄弱的问题已有明显改善。

## 6. 仍遗留的关键问题

- 前端版本号文案/筛选逻辑不统一。
- 查询导出与当前筛选条件仍未完全收口。
- 重新启用产品可能绕过正式生效规则，这是当前最重要的后端遗留问题。
