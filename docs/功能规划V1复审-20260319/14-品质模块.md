# 品质模块复审

## 1. 当前总体判断

- 品质模块不是“只有计划”，而是已有首件查询/导出、品质统计、质量趋势页、不良分析页、首件处置提交等实质实现。
- 但当前状态仍应判定为“部分落地、未达需求验收状态”，主要问题集中在首件处置闭环、统计口径一致性、趋势/不良分析能力缺项、报废导出副作用。

## 2. 已满足项

- 每日首件列表已支持日期、关键字、结果、产品、工序、操作员筛选，分页、刷新、导出和校验码展示均已实现。
- 品质数据页面已实现总览、按工序统计、按人员统计、按产品统计。
- 已存在独立的质量趋势页和不良分析页。
- 首件详情接口、首件导出接口、首件处置提交接口均已存在。
- 品质入口下已挂接报废统计与维修订单页面。

## 3. 部分满足项

- 首件处置已有真实提交能力，且后端有历史表，但前端仍只有弹窗，没有历史日志查看页面。
- 品质数据页没有内嵌趋势分析区，而是拆到独立趋势页。
- 不良分析页已有基础统计，但维度仍不够。
- 品质模块中的报废/维修能力大多复用生产模块页面和服务。

## 4. 未满足项

- 首件处置日志查看仍未实现。
- 首件处置未限制为“不通过首件记录”。
- 首件详情入口前端仍绑定到处置权限。
- 品质数据导出未覆盖产品统计与趋势分析。
- 产品统计、质量趋势中的报废/维修聚合口径仍未完全一致。
- 质量趋势页无导出；不良分析缺操作员/不良类型筛选与导出接入。
- 报废统计导出仍会修改进度状态。

## 5. 相比上一轮已修复项

- 后端首件详情权限已从处置权限中独立拆分。
- 后端首件处置已新增历史表，不再只有单条覆盖快照。
- 当前仓库确实已有独立质量趋势页、不良分析页、产品统计能力，不是纯计划态。

## 6. 仍遗留的关键问题

- 统计口径不一致仍是品质模块最关键的问题。
- 首件处置闭环仍未完成。
- 前后端对首件详情权限链仍不一致。
- 报废导出副作用问题仍在。
