# 生产模块 V1 使用说明（无消息中心）

## 1. 模块范围

本版本上线三个页面：

- 生产订单管理（`production_order_management`）
- 生产订单查询（`production_order_query`）
- 生产数据查询（`production_data_query`）

消息中心、品质数据、维修单相关页面不在 V1 范围内。

## 2. 角色可见性

- `system_admin`：可见生产模块 3 个 Tab
- `production_admin`：可见生产模块 3 个 Tab
- `quality_admin`：可见 `production_order_query`、`production_data_query`
- `operator`：可见 `production_order_query`

## 3. 主流程

1. 订单管理页建单，配置产品、数量、工序路线。
2. 操作员在订单查询页执行首件。
3. 操作员执行报工（结束生产）。
4. 系统自动推进下道工序可见量。
5. 最后一工序达标后订单自动完工。

## 4. 页面说明

### 4.1 订单管理

- 支持订单查询、创建、编辑（仅待生产）、删除（仅待生产）、手工完工。
- 提供订单详情弹窗，查看工序、生产记录、事件日志。

### 4.2 订单查询

- 支持手动刷新与 12 秒轮询刷新。
- 操作员可按按钮态执行“首件”与“结束生产”。
- 并发冲突时后端返回 409，前端会提示并建议刷新后重试。

### 4.3 生产数据

- 展示总览统计（订单状态、总数、完成量）。
- 展示工序维度统计。
- 展示操作员维度效率统计。

## 5. 关键数据一致性

- 报工数量不得超过可见量与可生产量。
- 工序完成量与子订单完成量保持一致性。
- 已完成工序不可继续报工。

## 6. 回归说明

生产模块为新增模块，不改变既有用户、产品、设备模块接口结构。
