# AGENTS 规则入口

## 1. 文档定位

- 根 `AGENTS.md` 是项目默认读取入口与规则装配点。
- 根 `AGENTS.md` 与 `docs/AGENTS/*.md` 共同组成同一套规则，不是并列规则源。
- 根文件只保留必须先读到的硬约束、优先级与阅读顺序；细则正文拆分到 `docs/AGENTS/`。
- 若根文件与分册正文出现冲突，以根文件为准；若分册之间出现冲突，按 `docs/AGENTS/00-导航与装配说明.md` 的顺序判定。

## 2. 适用范围与优先级

- 本规则集适用于仓库全部目录，除非子目录存在更严格的局部约束。
- 优先级顺序：
  1. 系统级与开发者级强制指令。
  2. 用户显式指令。
  3. 根 `AGENTS.md`。
  4. `docs/AGENTS/*.md` 分册正文。
  5. 其他支撑文档、历史日志与说明材料。
- 只要任务命中以下任一事项，必须继续阅读对应分册后再实施：
  - 通用执行、质量基线、安全退避：`docs/AGENTS/10-执行总则.md`
  - 指挥官模式、任务拆分、执行闭环：`docs/AGENTS/20-指挥官模式与工作流.md`
  - 工具优先级、验证门禁、降级代偿：`docs/AGENTS/30-工具治理与验证门禁.md`
  - 质量交付、evidence 留痕：`docs/AGENTS/40-质量交付与留痕.md`
  - 模板与索引：`docs/AGENTS/50-模板与索引.md`

## 3. 阅读顺序

1. `docs/AGENTS/00-导航与装配说明.md`
2. `docs/AGENTS/10-执行总则.md`
3. `docs/AGENTS/20-指挥官模式与工作流.md`
4. `docs/AGENTS/30-工具治理与验证门禁.md`
5. `docs/AGENTS/40-质量交付与留痕.md`
6. `docs/AGENTS/50-模板与索引.md`

## 4. 必须立即生效的硬约束

- 所有沟通、代码注释、文档与新增文件默认使用中文，编码为 UTF-8（无 BOM）。
- 每次任务开始与结束都要更新 `evidence/` 或等效项目日志。
- 每次对外回复开头提供“前置说明”；若默认强制优先使用的 Docker 宿主 MCP 工具（统一口径为 `MCP_DOCKER`）缺失、未配置、不可达、权限不足，或受更高优先级指令约束无法使用，必须说明缺失工具、原因、替代工具与影响范围；若存在外部调用，在末尾补“工具调用简报”。
- 编码前必须完成 `MCP_DOCKER Sequential Thinking` 分析；若所需 `MCP_DOCKER` 工具不可用，必须记录降级原因、触发时间、替代动作、影响范围与残余风险，并在 `evidence/` 中同步留痕。
- 工作流状态统一使用 `update_plan` 或 `TodoWrite` 维护；若工具不可用，允许在任务日志中维护步骤、状态、验收标准与更新时间。
- 仅运行安全命令，禁止破坏性清理、泄露密钥、令牌或内部敏感链接。
- 默认采取破坏性改动并拒绝向后兼容，但必须在交付中明确迁移口径；若无迁移需求，统一写“无迁移，直接替换”。
- 只要命中用户模块行为、入口、筛选、状态流转、文案边角分支等变更，必须在同一轮任务内同步收敛后端/API、Flutter、`integration_test`、`evidence/` 与执行命令口径。

## 5. 维护要求

- 修改分册正文时，不得重新引入“单文件唯一正文”或“并列规则源”口径。
- 若新增规则属于“必须先读到的硬约束”，需同时更新根 `AGENTS.md` 与对应分册。
- 若仅调整细则、模板、表格或索引，优先在对应分册修改，不将长表格和完整模板重新塞回根入口。

## 6. 迁移说明

- 本轮结构迁移口径：无数据迁移，直接替换。
