# AGENTS 规则入口

## 1. 文档定位

- 根 `AGENTS.md` 是项目默认读取入口与规则装配点。
- 根 `AGENTS.md` 与 `docs/AGENTS/*.md` 共同组成同一套规则，不是并列规则源。
- 根文件只保留必须先读到的硬约束、优先级与阅读顺序；细则正文拆分到 `docs/AGENTS/`。
- 若根文件与分册正文出现冲突，以根文件为准；若分册之间出现冲突，按 `docs/AGENTS/00-导航与装配说明.md` 的顺序判定。

## 2. 适用范围与优先级

- 本规则集适用于仓库全部目录，除非子目录存在更严格的局部约束。
- 优先级顺序：
  1. 系统级与开发者级强制指令。
  2. 用户显式指令。
  3. 根 `AGENTS.md`。
  4. `docs/AGENTS/*.md` 分册正文。
  5. 其他支撑文档、历史日志与说明材料。
- 只要任务命中以下任一事项，按需阅读对应分册后再实施：
  - 通用执行、质量基线、安全退避：`docs/AGENTS/10-执行总则.md`
  - 轻量工作流、任务拆分、收口口径：`docs/AGENTS/20-指挥官模式与工作流.md`
  - 工具参考、验证建议、降级代偿：`docs/AGENTS/30-工具治理与验证门禁.md`
  - 质量交付、按需留痕：`docs/AGENTS/40-质量交付与留痕.md`
  - 模板与索引：`docs/AGENTS/50-模板与索引.md`

## 3. 阅读顺序

1. `docs/AGENTS/00-导航与装配说明.md`
2. `docs/AGENTS/10-执行总则.md`
3. `docs/AGENTS/20-指挥官模式与工作流.md`（内容已改为轻量工作流，指挥官模式废止）
4. `docs/AGENTS/30-工具治理与验证门禁.md`
5. `docs/AGENTS/40-质量交付与留痕.md`
6. `docs/AGENTS/50-模板与索引.md`

## 4. 必须立即生效的硬约束

- 所有沟通、代码注释、文档与新增文件默认使用中文，编码为 UTF-8（无 BOM）。
- 仅运行安全命令，禁止破坏性清理、泄露密钥、令牌或内部敏感链接；需要停止进程、删除文件、重写历史或影响外部环境时，先确认目标与影响范围。
- 改代码前先读相关上下文，优先沿用现有架构、命名、测试与工具链；不为当前目标引入无关重构。
- 小任务不强制 `evidence/`、计划工具或 `Sequential Thinking`；复杂、跨模块、高风险、规则变更、排障超过 15 分钟或提交前，需使用当前可用的计划/拆解方式，并按需更新 `evidence/`、`progress.md` 或等效项目日志。
- 对外回复不强制固定“前置说明”格式；当工具缺失、降级、外部调用失败、验证无法执行或影响结论可靠性时，需在回复中说明原因、替代动作与影响范围。
- 完成前运行与改动匹配的真实验证；无法验证时必须说明原因、影响范围与已做的替代检查。
- 若仓库存在 Graphify 项目图谱（`graphify-out/GRAPH_REPORT.md` 或 `graphify-out/graph.json`），架构理解、跨模块排查、入口定位与关系追踪时优先作为结构导航；只有形成长期复用价值的稳定项目事实时，才写入 `docs/project-memory/` 并记录 Graphify 刷新或待语义更新状态。Graphify 只作辅助索引，不能替代源码、测试、接口契约、`AGENTS.md` 与真实验证；刷新图谱前必须核对 `.graphifyignore`，禁止纳入密钥、令牌、环境文件、构建产物、缓存、日志与临时目录。
- Memory MCP 只记录长期偏好、稳定约定和可复用项目事实；不记录密钥、隐私、临时流水或未验证猜测。
- 盘点或汇报当前会话可用工具、MCP、技能时，需同时核对工具命名空间、MCP tools、MCP resources/templates 与至少一次可行的只读真实调用；不得因 resources/templates 为空就判定已暴露 MCP tools 不可用。
- 所有 git 提交信息必须使用中文，不得使用英文或中英混杂标题作为默认提交口径。
- 提交前至少执行 `git status --short --untracked-files=all` 与 `git diff --check`，并确认提交范围不包含无关或敏感文件。
- 涉及接口契约、真实用户流程、用户模块入口、筛选、状态流转或文案边角分支时，同步检查后端/API、Flutter、`integration_test` 与执行命令口径；不涉及这些边界的小改动不强制全链路收敛。
- 规则与用户当前明确指令冲突时，优先执行用户当前指令；若涉及安全、数据丢失或不可逆操作，先说明风险再执行。

## 5. 维护要求

- 修改分册正文时，不得重新引入“单文件唯一正文”或“并列规则源”口径。
- 若新增规则属于“必须先读到的硬约束”，需同时更新根 `AGENTS.md` 与对应分册。
- 若仅调整细则、模板、表格或索引，优先在对应分册修改，不将长表格和完整模板重新塞回根入口。

## 6. 迁移说明

- 本轮结构迁移口径：无数据迁移，直接替换。
