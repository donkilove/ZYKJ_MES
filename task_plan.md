## P0 修复口径

- 日期解析：模型内部新增安全解析，必填 DateTime 字段遇 null/空/非法值时使用稳定 fallback，nullable 字段保持 null。
- 生产数量：按报告语义，`userCompletedQuantity == null` 或 `<= 0` 时回退到 `processCompletedQuantity`。
- 工艺表单：构建阶段只计算有效展示值，不修改 draft；状态更新集中在事件和提交前归一化。
- 设备状态码：POST 创建规则/运行参数接受 200 与 201。

## P1 修复口径

- 静默 `catch (_)`：本轮 DeepSeek 六大模块范围内的静默吞错已改为可见错误、登录态退出或调试日志；其他历史模块遗留 `catch (_)` 不在本轮 P1 范围。
- 网络异常：用户、产品、工艺、品质、生产、设备六个 service 改用统一 HTTP 包装，将超时和底层连接失败归一为 `ApiException('网络请求失败...', 0)`。
- 危险操作确认：强制下线、角色启停、工段/工序启停、设备规则/运行参数/保养计划启停、保养执行开始执行均补确认对话框。

## P2 修复口径

- P2-A：优先处理低风险一致性项，包括自动查询、可并行 API、共享常量/工具、已确认的死代码清理。
- P2-B：再处理分页、搜索和历史记录翻页等用户体验增强项。
- P2-C：最后处理导出格式统一、`void` 返回值收敛等契约面较广的调整。
