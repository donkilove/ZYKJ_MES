# FlaUI 基础设施

本目录提供 `mes_client.exe` 的 Windows 桌面自动化测试基础设施，基于 `FlaUI.UIA3`。

当前目录不再作为项目默认测试基线，而是历史保留的桌面系统级自动化工程与按需 fallback 入口。项目默认测试基线已切换为：后端使用 `pytest`；前端使用 `flutter test` 与 `integration_test`（主线）。除非用户显式要求桌面系统级场景或历史回放，否则不得将本目录作为默认验证入口。

## 目录说明

- `MesDesktop.FlaUI.Tests/`：独立的 FlaUI 测试工程，包含 smoke、登录页、主壳层与消息中心首批真实用例。

## 默认目标路径

默认定位仓库内以下可执行文件：

```text
frontend/build/windows/x64/runner/Debug/mes_client.exe
```

如需覆盖，请设置环境变量：

```powershell
$env:MES_CLIENT_EXE_PATH = "C:\path\to\mes_client.exe"
```

## 运行前提

1. 已安装 `.NET 8 SDK`。
2. 已构建 Flutter Windows 客户端，或已通过环境变量提供可执行文件路径。
3. `FlaUInspect` 已弃用并仅历史保留，不作为当前测试前置条件或默认工具。
4. 如需执行登录后用例，测试会在仓库根目录尝试启动 `start_backend.py --no-reload`，并默认使用测试环境账号 `admin / Admin@123456`。

## 执行命令

仅在用户显式要求桌面系统级场景、历史回放或默认前端测试基线无法覆盖时，才使用本目录 FlaUI 用例；执行时仍需串行，不允许并发多进程、多个 `dotnet test` 会话或多个桌面会话同时运行本目录用例。

```powershell
dotnet restore .\desktop_tests\flaui\MesDesktop.FlaUI.Tests\MesDesktop.FlaUI.Tests.csproj
dotnet test .\desktop_tests\flaui\MesDesktop.FlaUI.Tests\MesDesktop.FlaUI.Tests.csproj
```

- 工程侧已通过 MSTest assembly 级配置禁止并行；需要运行本目录时，执行与验证应仅在显式要求桌面系统级场景或历史回放时复用上面的 fallback 入口命令。
- 如需缩小范围，应只在同一条 `dotnet test` 命令上追加 `--filter`，不要新开并发会话分摊用例。

## 用户模块同步更新规则

- 用户模块后续新增页签、补齐边角分支或修正断言时，默认应在同一轮任务内同步核对后端/API、Flutter 页面、`integration_test`、`evidence/`、执行命令口径；仅在显式要求桌面系统级场景时，才补本目录 FlaUI 入口与断言。
- 推荐节奏固定为：先收敛后端/API 与 Flutter 可见行为，再补 `integration_test`，最后同步 `evidence/` 描述与执行命令；若需桌面系统级 fallback，再补本目录导航入口、页签定位与断言，避免规则、实现与验证口径漂移。
- 当前本目录保留的用户模块入口仍以主壳层 `用户` 导航进入后，再落到 `用户管理` 或 `注册审批` 等业务页签作为历史口径；仅在需要桌面自动化 fallback 时沿用，不作为默认主线入口说明。

## Smoke 验收范围

- 解析 `mes_client.exe` 路径，支持环境变量覆盖。
- 启动应用并等待主窗口出现。
- 输出明确通过/失败原因。
- 测试结束后清理进程，避免残留。

## 已覆盖真实用例范围

- 登录页关键元素存在：校验 `接口地址`、`账号`、`密码`、`登录` 等 UIA 元素可见。
- 管理员登录后主壳层可见：校验 `用户`、`产品`、`工艺`、`生产`、`质量/品质`、`设备`、`消息` 等导航元素可见。
- 消息中心入口可进入：点击包含 `消息` 的导航按钮，并校验 `消息中心`、`搜索标题/摘要`、列表操作按钮等元素可见。
- 用户模块可进入：点击主壳层 `用户` 导航，并校验 `用户管理` 或 `注册审批` 等业务页签已暴露到当前 UIA 树。
- 用户管理页可进入：点击 `用户管理` 页签，并校验 `角色管理`、`账号`、`角色`、`状态` 等关键按钮或表头可见。
- 注册审批页可进入：点击 `注册审批` 页签，并校验 `申请状态`、`用户名`、`待审批` 等关键筛选或表头可见。
- 个人中心页补齐：优先复用用户模块默认落点；若 `个人中心` 页签节点未稳定暴露，则回退到已进入用户模块后的 `个人中心`、`修改密码`、`当前会话`、`用户名` 等真实页面区域断言。
- 登录会话页补齐：点击 `登录会话` 页签，并校验 `在线会话`、`关键词`、`强制下线`、`用户名` 等关键表头与操作元素可见；额外覆盖单条真实强制下线链路。
- 角色管理页补齐：点击 `角色管理` 页签，并校验 `角色名称`、`新增角色`、`关键词`、`状态` 等关键区域可见；额外覆盖单条真实删除链路。
- 审计日志、功能权限配置补齐：分别验证页签可进入，并校验 `操作时间/操作人/查询` 与 `模块/保存/角色` 等关键区域可见。

## UIA 调研摘要

- 当前 Flutter Windows 客户端在 UIA 中可稳定暴露登录页 `Edit/Button/Text` 控件，适合直接通过文本和控件类型定位。
- 主壳层侧边导航在 UIA 中表现为具名 `Button`，其中消息入口名称当前带有未读数，例如 `38\n消息`，测试需按“名称包含 `消息`”而非全等匹配。
- 登录后主壳层就绪判定采用更宽松的组合特征等待：欢迎区候选文案、多个主导航命中、`质量/品质` 与 `消息` 等组合地标任意达标即可，通过降低对单一 `消息` 入口的硬依赖，减少壳层刚加载完成时的偶发误判。
- 消息中心页面标题、搜索框、列表操作按钮与消息卡片摘要均可见，可作为首批回归断言。
- 用户模块入口与业务页签定位改为“名称包含中文文案 + 控件类型优先级 + 等待”，可覆盖 `用户管理`、`注册审批` 的真实进入路径，避免依赖脆弱索引。
- 用户管理与注册审批的二次交互增加了跨顶层窗口查找：右侧菜单、确认弹窗、输入框与确认按钮优先在独立顶层窗口中定位。用户管理侧对未稳定暴露名称的行内菜单项改用“唯一账号定位行 + 打开 `操作. 显示菜单` + 键盘 fallback”；注册审批侧移除不存在的用户名筛选输入假设，改为直接翻页定位目标账号后完成驳回。
- `个人中心` 页签节点在当前客户端 UIA 上仍可能不稳定，自动化优先接受“打开用户模块后已直接进入个人中心内容区”的真实页面断言；若未来入口节点稳定，再收敛为固定页签点击口径。
- `登录会话` 页面标题实际为 `在线会话`，FlaUI 断言需区分导航页签文案与页面主标题，避免误把标题差异判为未进入页面。
- `登录会话` 的真实单个下线链路目前仍受 UIA 列表暴露影响：通过 API 预置的新会话未能在当前运行态稳定出现在页面筛选结果中，因此本轮保留入口可见断言，真实下线动作暂列阻塞项。
- `功能权限配置` 页签虽可在用户模块范围内参与候选定位，但当前页面内容区在 UIA 中未稳定暴露 `模块/保存/角色` 等关键节点，本轮仅保留为已知阻塞项。

## 常见失败层级

- SDK 层：本机没有可用的 `.NET SDK`。
- 应用层：`mes_client.exe` 不存在或无法启动。
- 窗口层：进程已启动，但主窗口未在超时内出现。
- UI 自动化层：FlaUI 或 UIA3 初始化异常。
