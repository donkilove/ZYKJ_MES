## 3. 任务目标、范围与非目标

### 3.1 任务目标

1. 按测试工程师视角，把用户模块剩余高优先与中高优先缺口一次性补齐。
2. 在后端/API、Flutter、FlaUI 三条线上同步完成实现、测试与复检。
3. 完成用户模块最终综合复测与独立终验，形成“当前阶段完整收口”结论。

### 3.2 任务范围

1. 后端/API：auth/authz 剩余契约、用户守卫、列表/导出过滤、会话筛选等。
2. Flutter：功能权限配置、审计日志、角色管理、登录会话及其余高优先边角行为。
3. FlaUI：个人中心、角色管理、登录会话、功能权限配置、审计日志等剩余页签与更深层动作，严格串行执行。
4. evidence：本轮实现、验证、失败重试与最终结论留痕。

### 3.3 非目标

1. 暂不推进非用户模块剩余缺口。
2. 暂不做性能压测、安全扫描与发布前审计。
3. 暂不为了测试而大改用户模块业务设计，除非真实缺陷阻断收口。
