1. 操作员页二维码内容应为微信可直接打开的后端局域网网页地址。

## 5. 子 agent 输出摘要

- 调研摘要：后端创建会话返回相对 `review_url`；状态轮询返回空 `review_url` 会覆盖页面持有的旧会话，导致 3 秒后二维码消失。用户进一步确认手机只能访问后端所在局域网，因此扫码页必须由后端直接承载。
- 执行摘要：后端新增 `app.web.first_article_review_page` 与静态 HTML 手机页，`/first-article-review` 由 FastAPI 直接返回；创建/刷新扫码复核会话时由 API 层按请求主机生成后端完整 URL。Flutter 操作员页改为优先使用后端完整 URL，并在状态轮询中保留已存在的 `reviewUrl`，避免二维码 3 秒后消失。后续跟进中又补充了后端 `public_base_url` 配置与 `localhost/127.0.0.1` 自动替换局域网 IP 的逻辑，避免二维码继续显示回环地址。
- 验证摘要：后端手机页单测通过；Flutter widget 与 Windows integration_test 通过；后端数据库集成测试因本机 PostgreSQL 不可达未完成。后续针对“127.0.0.1 误用”补充的后端单测与 Flutter widget 回归也已通过。
