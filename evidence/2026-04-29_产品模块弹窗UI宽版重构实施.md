# 2026-04-29_产品模块弹窗UI宽版重构实施

## 1. 任务目标
将 `product_management_page.dart` 中原本以代码段形式（inline）散落的所有涉及新建、编辑、交互确认的弹窗，统一提取为独立高保真组件，并采用双栏宽版 (860px) 或符合系统一致性的 `MesDialog` 和 `MesActionDialog` 规范。同时解决表单组件被 `_MesLockedDialogScope` 错误双层嵌套的 Bug。

## 2. 变更详情
- **提取产品表单**：
  将“新建产品”与“编辑产品”逻辑提取为独立的 `ProductFormDialog`（`product_form_dialog.dart`）。
  - 使用双栏布局，宽度 `860px`，左侧为基础信息（产品名称、产品分类），右侧为附加信息（备注）与提示文本。
  - 组件内部完成网络请求与错误处理收口，有效缩减主页面的业务逻辑混杂。
  - 支持 `wrapMesDialog: false`，避免由于 `StatefulBuilder` 导致的 UI 套娃。
- **提取操作确认弹窗**：
  将停用原因采集（`showInactiveReasonDialog`）、影响分析确认（`showConfirmImpactDialog`）、删除确认包含密码输入（`showConfirmDeleteProductDialog`）统一提取到 `product_action_dialogs.dart` 中。
- **修复版本弹窗嵌套**：
  在主页中为 `_showVersionDialog` 追加了 `wrapMesDialog: false`，解决了内部 `ProductVersionDialog` 因被 `StatefulBuilder` 包裹而错误生成外层白底框的问题。
- **依赖消除**：
  移除主页面中已无人使用的各种内置 `_validateXXX` 方法，代码更加精简（`product_management_page.dart` 从原本 63KB 缩小至 49KB 左右）。
- **静态检查分析**：
  通过 `flutter analyze` 修复了诸如 `value` 在 `DropdownButtonFormField` 弃用、`withOpacity` 替换为 `withAlpha` 等新版 Flutter/Dart 告警，确保主页面和新提取组件零额外告警。

## 3. 验收标准
- [x] 所有弹窗采用独立的组件文件维护。
- [x] 表单弹窗采用 `860px` 宽的双栏流式布局，右侧具备毛玻璃灰底视觉提示区。
- [x] `_MesLockedDialogScope` 双层白边问题由于精准切断不再复现。
- [x] `flutter analyze` 在 `features/product/presentation` 目录下干净无告警。

## 4. 残余风险与后续
- 产品参数（Product Parameter）页面的关联编辑或弹窗若涉及类似套娃问题，将随接下来的组件优化继续处理。
- 迁移为无缝热重载兼容，未引入任何可能破坏生产逻辑的业务改动。
