# 前端全站 UI 差异审计表

## 审计范围
- 模块：auth / craft / equipment / message / misc / plugin_host / product / production / quality / settings / shell / time_sync / user
- 文件类型：`*_page.dart` / `*_dialog.dart` / `*_panel.dart` / `*_drawer.dart`
- 维度：页面头部 / 筛选区 / 操作条 / 内容容器 / 列表与分页 / 状态反馈 / 浮层 / 响应式与密度
- 策略：直接套壳 / 局部重排 / 结构重组 / 无需调整

## 视觉基线
- 锚点：用户模块用户管理页（`frontend/lib/features/user/presentation/user_management_page.dart`）
- 公共件：`core/ui/foundation`、`core/ui/primitives`、`core/ui/patterns`
- 锚点现状：`MesCrudPageScaffold` + `CrudPageHeader`（旧件经 wrapper） + `UserModuleFilterPanel`（内含 `MesFilterBar`） + `SimplePaginationBar`（旧件） + `showLockedFormDialog`（旧件） + `CrudListTableSection`

## 总结

### 总体统计

- **审计文件数**：60 个（page 60 + dialog 10 + panel 8 + drawer 1）
- **关涉差距行总数**：约 145 行有差距（非"无需调整"）
- **"直接套壳"总数**：约 65 行——最常见策略，低风险
- **"局部重排"总数**：约 45 行——中等风险
- **"结构重组"总数**：约 20 行——最高风险，涉及页面骨架替换

### 最高频差距项

1. **SimplePaginationBar（旧件）仍在使用**：共 25+ 处文件引用，遍及 user/equipment/production/quality/misc 全部 CRUD 页面。这是全站最广泛的单一旧件。
2. **CrudPageHeader（旧件）仍在使用**：共 18 处 import，其中 16 处页面文件 + 2 处 wrapper widget。部分页面的 wrapper 已迁移到 `MesPageHeader`，但 wrapper 内部仍对接 `CrudPageHeader`。
3. **缺 MesCrudPageScaffold**：约 20 个 CRUD 型页面仍手写 `Padding + Column` 骨架，未使用统一 scaffold。
4. **缺 MesFilterBar**：约 18 个页面手写 `Row/Wrap + TextField + Dropdown` 筛选区，未包裹 `MesFilterBar`。
5. **showLockedFormDialog（旧件）**：约 25+ 处调用，主要集中在 equipment/production/user/craft/product 模块。虽然行为统一（锁定 dismissible），但属于旧件。
6. **裸 CircularProgressIndicator**：约 35 处使用，未封装为标准加载状态件。
7. **手写 Text(error) 代替 MesInlineBanner**：约 12 处页面用手写 `Text` + `errorContainer` 样式代替标准 banner。

### 模块成熟度排序（由高到低）

1. **message**——几乎全部达标（MesCrudPageScaffold + MesPageHeader + MesFilterBar + MesPaginationBar + MesDetailPanel）
2. **product**——大部分达标（MesCrudPageScaffold + MesPageHeader + MesFilterBar + MesInlineBanner + MesToolbar），仅 SimplePaginationBar 和个别 dialog 为旧件
3. **shell / settings**——达标（dashboard 场景用 MesSectionCard/MesEmptyState/MesPageHeader）
4. **user**——半迁移状态：scaffold 已升级，但 2 个 header wrapper 仍用 CrudPageHeader，全部用 SimplePaginationBar，多处 showLockedFormDialog
5. **craft**——混合态：kanban/reference 已用 MesCrudPageScaffold，但 process_management/process_configuration 仍为旧结构
6. **quality**——混合态：trend/scrap_statistics 已用 MesCrudPageScaffold，但 data/defect_analysis/supplier 仍为旧结构
7. **production**——最低成熟度：14 个子页面中仅 3 个用 MesCrudPageScaffold（data/pipeline_instances 部分），其余全为旧件骨架
8. **equipment**——最低成熟度：6 个子页面全部为旧件骨架（CrudPageHeader + Padding+Column + SimplePaginationBar）

### 迁移优先级建议

- **P0（高优先 / 覆盖面最广）**：全站 `SimplePaginationBar` -> `MesPaginationBar`（25+ 处，直接套壳）
- **P1**：equipment 全模块 + production 全模块结构重组（`Padding+Column` -> `MesCrudPageScaffold`）
- **P2**：`CrudPageHeader` -> `MesPageHeader` 迁移（18 处）
- **P3**：筛选区 `MesFilterBar` 包裹（18 处）
- **P4**：`showLockedFormDialog` 标准化 / 浮层统一
