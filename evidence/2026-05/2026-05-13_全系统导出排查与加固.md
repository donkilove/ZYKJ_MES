## 2. 范围与目标
- 目标：修复产品域导出按钮因“后端回 CSV 流、前端按 JSON 解码”导致的失败与乱码，并新增前端公共下载解析能力，复用于用户导出任务下载。
- 非目标：不统一全系统后端导出契约，不改质量/生产/工艺等仍采用 `content_base64` 的导出协议。
- 影响范围：
  - 产品导出列表
  - 产品版本参数导出
  - 产品参数查询导出
  - 用户导出任务下载

## 3. 导出契约矩阵
| 页面/入口 | 前端服务 | 后端接口 | 当前契约 | 本轮动作 |
| --- | --- | --- | --- | --- |
| 产品管理-导出产品 | `ProductService.exportProducts()` | `GET /api/v1/products/export/list` | 原始 CSV 流 | 修复前端解析 |
| 产品版本管理/参数管理-导出参数 | `ProductService.exportProductVersionParameters()` | `GET /api/v1/products/{product_id}/versions/{version}/export` | 原始 CSV 流 | 修复前端解析 |
| 产品参数查询-导出 | `ProductService.exportProductParameters()` | `GET /api/v1/products/parameters/export` | 原始 CSV 流 | 修复前端解析 |
| 用户导出任务下载 | `UserService.downloadUserExportTask()` | `GET /api/v1/users/export-tasks/{task_id}/download` | 原始文件流 | 接入公共解析守卫 |
| 工艺看板导出 | `CraftService.exportCraftKanbanProcessMetrics()` | `GET /api/v1/craft/kanban/process-metrics/export` | JSON + `content_base64` | 只回归，不改协议 |
| 生产工单/报废/返修导出 | `ProductionService.*export*()` | `/api/v1/production/*/export` | JSON + `content_base64` | 只回归，不改协议 |
| 质量统计/趋势/不良/首件导出 | `QualityService.*export*()` | `/api/v1/quality/*/export` | JSON + `content_base64` | 只回归，不改协议 |

## 4. 执行记录
- 新增 `frontend/lib/core/network/download_response_parser.dart`
  - 统一解析原始文件流响应
  - 支持 `filename*=` UTF-8 文件名与普通 `filename=`
  - 非 200 时优先解析 JSON 错误体，否则安全按 UTF-8 解码纯文本并转成可读 `ApiException`
- 修改 `frontend/lib/features/product/services/product_service.dart`
  - 产品导出 3 个方法统一改走公共文件流解析
  - 维持页面层返回 `ProductExportFile`
  - 使用 `base64Encode(response.bodyBytes)` 兼容现有页面保存逻辑
- 修改 `frontend/lib/features/user/services/user_service.dart`
  - 用户导出任务下载复用公共文件流解析
  - 移除服务内重复的 `content-disposition` 解析逻辑
- 修改 `frontend/test/services/product_service_test.dart`
  - 产品导出测试改为真实 CSV 字节流
  - 补 UTF-8 文件名、BOM 字节、非 JSON 错误响应断言
- 修改 `frontend/test/services/user_service_test.dart`
  - 用户导出任务下载测试切到公共文件流解析路径
  - 补非 JSON 失败响应断言
- 工具降级：
  - 当前终端 `dart` / `flutter` 不在 PATH
  - 通过仓库启动脚本线索定位 `C:/flutter/bin/flutter.bat`
