# 2026-05-13 工序首件预设改造

## 用户目标

- 工序设置新增 `首件内容`、`首件检验值` 两项预设，保留空格与换行。
- 开始首件页直接使用当前工序预设，不再依赖首件模板入口。
- 生成/刷新二维码前，如果两项内容与工序预设完全一致，则阻止生成并提示错误。
- 扫码复核详情展示需要保留原始格式。

## 主要改动

- 后端
  - `mes_process` 模型新增 `first_article_check_content`、`first_article_test_value`。
  - 新增 Alembic 迁移 `backend/alembic/versions/c1d2e3f4a5b6_add_process_first_article_presets.py`。
  - 工艺接口 `CraftProcessCreate/Update/Item` 与 `craft` 端点支持读写两项预设。
  - 生产上下文 `MyOrderItem` 新增当前工序首件预设字段。
  - 扫码复核服务改为：
    - 必填判空使用 `strip()`，但保留原始文本。
    - 创建/刷新复核会话时校验是否仍与当前工序预设完全一致。
    - 通过扫码复核落库首件记录时保留原始文本格式。
  - 扫码复核 HTML 页面为详情值增加 `white-space: pre-wrap`。

- 前端
  - 工序编辑弹窗新增 `首件内容`、`首件检验值` 多行输入框。
  - `CraftProcessItem`、`CraftService.createProcess/updateProcess` 同步两项字段。
  - 开始首件页：
    - 页面初始化直接带入当前工序预设。
    - 移除“首件模板”按钮与页面内模板选择流程。
    - 新增“刷新首件内容”“刷新首件检验值”按钮，分别恢复对应预设。
    - 生成/刷新二维码前本地校验：
      - 两项都必须包含非空白字符。
      - 两项不能同时与预设完全一致。
      - 请求发送原始文本，不做 `.trim()`。

- 测试
  - 后端新增/调整：
    - 工序预设 round-trip 集成用例。
    - 我的工单上下文返回当前工序预设用例。
    - 扫码复核创建/刷新命中“未修改预设”拦截用例。
    - 扫码复核详情保留多行/空格格式用例。
    - 复核网页包含 `white-space: pre-wrap` 断言。
  - 前端新增/调整：
    - 开始首件页不再展示模板按钮，初始化取工序预设。
    - 未修改预设时不发起扫码复核。
    - 两个刷新按钮恢复各自预设。
    - 工序管理页新增工序测试覆盖两项预设提交。

## 当前阻塞与降级

- 当前数据库尚未应用本次迁移；直接运行依赖 `mes_process.first_article_check_content` / `mes_process.first_article_test_value` 的后端集成用例时，会报 PostgreSQL `UndefinedColumn`。
- 当前终端环境中未找到可执行的 `dart` / `flutter`，因此本轮无法执行 `dart format`、`flutter format` 或 `flutter test`。

## 后续建议

1. 在目标数据库执行 Alembic 迁移后，再运行本次新增的后端集成用例。
2. 在具备 Flutter/Dart 工具链的环境里执行 `dart format` / `flutter test`，完成前端真实验证。
