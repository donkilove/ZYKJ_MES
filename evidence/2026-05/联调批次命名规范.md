# 联调批次与账号规范

> 日期：2026-05-02
> 适用计划：`docs/superpowers/plans/2026-05-02-cross-module-integration-test-plan.md`

## 1. 批次命名约定

本轮联调统一使用以下前缀：

```text
EVAL-USR-<batch>     — 用户模块场景
EVAL-PRD-<batch>     — 产品模块场景
EVAL-CRAFT-<batch>   — 工艺模块场景
EVAL-QUAL-<batch>    — 质量模块场景
EVAL-PIPE-<batch>    — 生产/流水线模块场景
```

- `<batch>` 使用 `HHmmss` 格式时戳，如 `142721`。
- 对应数据库工单编码前缀、返修单编码等遵循当前系统自动生成规则。

## 2. 工单类型定义

| 类型 | 编码示例 | 用途 |
|------|---------|------|
| `normal-complete` | `EVAL-NORM-<batch>` | 正常模式完整完工 |
| `quality-failed` | `EVAL-QUAL-<batch>` | 首件失败/返修/报废 |
| `pipeline-complete` | `EVAL-PIPE-<batch>` | 流水线完整完工 |
| `sync-impact` | `EVAL-SYNC-<batch>` | 产品/工艺版本变更中的在制工单 |
| `edge-case` | `EVAL-EDGE-<batch>` | 权限、换人、解绑、禁用等异常流 |

## 3. 账号用途约定

| 账号 | 角色 | 用途 |
|------|------|------|
| `ltadm1` | `system_admin` | 系统管理员，全局操作 |
| `ltprd1` | `production_admin` | 生产管理员，建单/流水线配置 |
| `ltqua1` | `quality_admin` | 质量管理员，首件/返修/报废 |
| `ltopr1` | `operator` | 首工序操作员（绑定 `PERF-PROCESS-STD-01`） |
| `ltopr2` | `operator` | 次工序操作员（绑定 `PERF-PROCESS-STD-02`） |
| `ltopr3` | `operator` | 代班 helper（不绑定 PERF 工序） |
| `ltopr4` | `operator` | 未绑定/边界校验账号 |

## 4. 基线样本复用

| 资源 | 编码 | 说明 |
|------|------|------|
| 产品 | `PERF-PRODUCT-STD-01` | perf 标准产品 |
| 工艺模板 | `PERF-TEMPLATE-STD-01` | perf 标准模板（`v1`） |
| 工序 | `PERF-PROCESS-STD-01` | 首道工序 |
| 工序 | `PERF-PROCESS-STD-02` | 末道工序 |
| 供应商 | `PERF-SUPPLIER-STD-01` | perf 标准供应商 |
| 验证码 | `VC-654321` | 当日验证码 |

## 5. 清理策略

- 联调期间不对已完结工单做物理删除，保留现场以便事后回溯。
- 重复验证时新建独立工单，避免串扰已有样本。
- 遇到接口返回与数据库事实不一致时，保留现场工单和 `run_id`，不立即清理。
