# 指挥官任务日志模板

## 使用说明

- 本模板用于记录“主 agent 指挥、子 agent 执行、子 agent 验证”的完整闭环。
- 新建任务日志时，建议复制本文件并按日期重命名，例如：`evidence/commander_execution_YYYYMMDD.md`。
- 模板中的尖括号占位内容必须替换为真实信息，禁止保留空泛描述。
- 若无迁移需求，统一写“无迁移，直接替换”。
- 若存在硬阻塞，必须记录“已尝试动作、当前影响、建议动作”，不得只写“阻塞中”。
- 若发生工具降级、只读调研或 evidence 代记，必须记录原因、时间、责任人与补偿措施。
